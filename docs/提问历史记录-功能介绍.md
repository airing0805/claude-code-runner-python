# 提问历史记录功能介绍

本文档介绍 Claude Code Runner v8.0.0 版本新增的提问历史记录功能。

## 功能概述

提问历史记录功能让用户能够快速查看之前向 Claude 提出的问题，方便重复使用类似的问题。系统会从所有会话记录中提取首次用户提问，按项目维度组织，帮助用户高效管理和复用历史提问。

## 核心特性

| 特性 | 说明 |
|------|------|
| 项目隔离 | 按项目维度组织提问记录，方便按项目查找 |
| 时间排序 | 提问按时间倒序排列，最新提问优先显示 |
| 完整内容 | 提取完整的提问文本，不限长度 |
| 智能过滤 | 自动过滤 IDE 标签和敏感信息 |
| 高效缓存 | 基于文件修改时间自动缓存，避免重复计算 |

## 使用方法

### 访问提问历史记录

1. 在左侧导航栏中找到「提问历史记录」菜单项
2. 点击进入提问历史记录视图

### 查看项目列表

进入提问历史记录后，首先显示所有有会话记录的项目列表：

- 每个项目显示项目路径和会话数量
- 按会话数量降序排列
- 点击项目进入该项目的提问列表

### 查看提问列表

选择项目后，显示该项目的所有历史提问：

- 每条提问显示完整提问内容和时间
- 按时间倒序排列，最新提问优先
- 支持分页浏览（每页 20 条）

### 交互操作

#### 展开/收起提问

- 提问文本超过 200 字符时自动截断
- 点击「展开」按钮查看完整内容
- 展开后按钮变为「收起」

#### 复制提问

- 点击提问项右侧的复制按钮
- 提问内容复制到剪贴板
- 显示「已复制」成功提示（2 秒后消失）

#### 继续会话

- 点击提问项进入对应会话
- 系统加载历史会话内容
- 用户可以继续之前的对话

## 时间显示规则

| 时间范围 | 显示格式 | 示例 |
|----------|----------|------|
| < 1 分钟 | 刚刚 | 刚刚 |

## 数据提取规则

### 提取逻辑

系统从会话 JSONL 文件中提取首次用户提问：

1. **定位首次用户消息**：寻找 `type == "user"` 的第一条记录
2. **提取提问内容**：从消息内容中提取所有文本
3. **过滤特殊标签**：排除 IDE 相关标签
4. **敏感信息脱敏**：自动过滤 API Keys、Tokens 等

### 过滤的内容

以下内容会被自动过滤，不会出现在提问历史中：

- `<ide_selection>...</ide_selection>` - IDE 选中的代码
- `<ide_opened_file>...</ide_opened_file>` - IDE 打开的文件
- `<command-message>...</command-message>` - 命令消息标识
- `<command-name>...</command-name>` - 命令名称
- `<command-args>...</command-args>` - 命令参数

### 敏感信息过滤

以下敏感信息会被自动脱敏：

- API Keys（以 `sk-` 开头的密钥）
- GitHub Tokens（包含 `ghp_`, `gho_` 等前缀）
- Passwords（包含 password, pwd 字段）
- AWS Keys（包含 `AKIA` 前缀）

## 缓存机制

### 工作原理

系统使用元数据缓存提高性能：

- **缓存位置**：项目目录下的 `.metadata.json` 文件
- **缓存内容**：会话标志位（是否存在有效提问、提问时间戳）
- **失效策略**：当会话文件修改时，缓存自动失效

### 性能指标

| 指标 | 目标值 |
|------|--------|
| 缓存读取时间 | < 10ms |
| 二次访问响应 | < 100ms |
| 首次提取时间 | < 500ms/文件 |

## 移动端适配

提问历史记录功能支持响应式布局：

- 768px 断点：调整列表项样式
- 480px 断点：优化小屏幕显示
- 触摸友好：按钮尺寸适合触摸操作

## 常见问题

### Q: 为什么有些会话没有出现在提问列表中？

A: 系统只提取有效的首次用户提问。如果会话中没有用户消息，或用户消息只包含 IDE 标签（如 `<ide_selection>`），则该会话不会显示在提问列表中。

### Q: 提问内容会自动更新吗？

A: 是的，系统会检测会话文件的变化。当文件修改时，缓存自动失效，下次访问时会重新提取提问内容。

### Q: 可以搜索特定的提问吗？

A: 当前版本暂不支持搜索功能，用户需要通过浏览项目列表和提问列表来查找历史提问。

### Q: 复制功能在移动端可用吗？

A: 是的，移动端也支持点击复制按钮复制提问内容。
