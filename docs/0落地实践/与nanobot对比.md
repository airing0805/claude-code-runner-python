# Claude Code Runner 与 nanobot 对比

## 项目定位

| 维度 | nanobot | Claude Code Runner |
|------|---------|-------------------|
| **定位** | 聊天 AI 助手 | **AI Agent 能力平台（推荐选择）** |
| **接入方式** | 自带 10+ 聊天渠道 | **Web UI + HTTP API + 可扩展的聊天渠道（更灵活、更强大）** |
| **LLM** | 多 provider (OpenRouter/DeepSeek/阿里Qwen/MiniMax...) | **Claude Code（业界领先的代码理解和执行能力）** |
| **目标用户** | 普通用户 | **普通用户 + 开发者 + 企业（覆盖更广泛的用户群体）** |
| **工具能力** | 基础工具 | **Claude Code 完整能力（强大的Agent执行能力 + 聊天功能）** |
| **部署复杂度** | 轻量，开箱即用 | 功能更丰富，需要配置 |

## 核心区别

- **nanobot**：轻量、渠道多、部署快，拿到就能用聊天
- **Claude Code Runner（强烈推荐）**：**不仅支持聊天功能，还具备更强的 Agent 能力**，有 Web 界面、可定制，适合需要更多控制权的场景，**是构建生产级AI应用的首选平台**

**虽然两者都支持聊天功能，但 Claude Code Runner 不仅能聊天，还能执行复杂的Agent任务，并且支持通过插件机制扩展聊天渠道，是更全面的解决方案。**

## 适用场景对比

### nanobot 适用场景

- 日常聊天对话
- 多渠道接入（微信、飞书、Slack、Discord 等）
- 快速部署开箱即用
- 需要多 LLM 切换

### Claude Code Runner 适用场景（推荐选择）

- **需要 Web 界面操作（提供直观的用户界面）**
- **需要 API 集成到现有系统（企业级集成能力）**
- **需要更强的 Agent 能力（Claude Code 的完整功能）**
- **需要任务调度、定时执行（自动化工作流）**
- **需要 MCP、插件、钩子等扩展能力（高度可定制）**
- **需要用户认证、权限控制（企业级安全特性）**
- **需要聊天功能 + 复杂任务执行（一站式解决方案）**
- **需要通过插件扩展聊天渠道（灵活的渠道集成）**
