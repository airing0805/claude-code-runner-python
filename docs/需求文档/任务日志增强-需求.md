## 一、需求背景

### 1.1 业务背景

Claude Code Runner 的任务调度模块已实现核心功能，包括任务队列管理、定时任务执行、任务状态查询等。在任务执行过程中，会产生大量的日志输出，包括正常运行日志和错误日志。

### 1.2 用户痛点

| 痛点 | 描述 | 影响程度 |
|------|------|----------|
| 日志混杂 | 正常日志和错误日志混合显示，难以快速定位问题 | 高 |
| 无法实时查看 | 任务运行中无法实时查看日志输出 | 中 |
| 历史日志难查 | 历史任务日志只能一次性加载全部，无法分页 | 中 |
| 搜索困难 | 大日志量时无法快速搜索关键字 | 中 |

### 1.3 解决方案价值

- **提升排查效率**：分离正常日志和错误日志，快速定位问题
- **实时监控**：SSE 实时推送日志，监控任务执行状态
- **历史追溯**：分页加载历史日志，支持大日志量场景
- **快速定位**：关键字搜索，快速找到目标日志行

## 二、产品定位

### 2.1 功能概述

在现有任务日志功能基础上，增强日志展示能力，提供两个独立的日志标签页（正常日志/错误日志），支持实时日志滚动显示、历史日志分页加载和日志搜索功能。

### 2.2 目标用户

- 使用任务调度功能的开发者
- 需要监控定时任务执行状态的用户
- 需要排查任务执行问题的运维人员

### 2.3 使用场景

| 场景 | 描述 | 适用功能 |
|------|------|----------|
| 场景一 | 任务运行中，实时查看日志输出 | 实时日志（SSE） |
| 场景二 | 任务完成后，查看历史日志 | 历史日志分页 |
| 场景三 | 大日志量时搜索关键字 | 日志搜索 |
| 场景四 | 排查错误问题 | 错误日志标签页 |

### 2.4 核心价值

1. **分类展示**：分离正常日志和错误日志，聚焦关键信息
2. **实时监控**：SSE 实时推送，实时掌握任务状态
3. **高效查阅**：分页加载 + 关键字搜索，快速定位目标

## 三、功能需求

### 3.1 功能优先级 (MoSCoW)

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **Must** | 实时日志推送 | SSE 实时推送日志到前端 |
| **Must** | 历史日志分页 | 支持 offset/limit 分页加载 |
| **Should** | 日志搜索 | 支持关键字搜索 |
| **Should** | 搜索高亮 | 搜索结果关键字高亮显示 |
| **Could** | 正则搜索 | 支持正则表达式搜索（可选） |

### 3.2 详细功能需求

#### FR-036: 日志标签页容器

**描述**: 创建日志标签页容器，支持"正常日志"和"错误日志"两个标签页切换

**输入**:
- 无（从系统获取）

**处理逻辑**:
1. 创建两个标签页：正常日志、错误日志
2. 记录当前选中的标签页状态
3. 切换标签页时加载对应的日志数据
4. 默认显示上次选中的标签页（如有）

**输出**:
- 标签页容器 UI
- 标签页切换按钮
- 当前标签页状态指示

**异常处理**:
- 加载失败显示错误信息

#### FR-037: 正常日志标签页

**描述**: 显示任务的标准输出日志（stdout）

**输入**:
- 任务 ID
- 分页参数（可选）

**处理逻辑**:
1. 调用 `GET /api/scheduler/tasks/{id}/logs/normal` API
2. 获取 stdout 类型日志
3. 按时间正序排列显示

**输出**:
- 正常日志列表 UI
- 每行显示：时间戳、日志内容

#### FR-038: 错误日志标签页

**描述**: 显示任务的错误输出日志（stderr）

**输入**:
- 任务 ID
- 分页参数（可选）

**处理逻辑**:
1. 调用 `GET /api/scheduler/tasks/{id}/logs/error` API
2. 获取 stderr 类型日志
3. 按时间正序排列显示

**输出**:
- 错误日志列表 UI
- 每行显示：时间戳、日志内容
- 错误日志使用特殊样式（红色/粉色背景）

#### FR-039: 标签页状态保持

**描述**: 记住用户选择的标签页状态，页面刷新后保持

**输入**:
- 上次选中的标签页

**处理逻辑**:
1. 切换标签页时保存状态到 localStorage
2. 页面加载时读取 localStorage 恢复状态

**输出**:
- 标签页状态持久化

#### FR-040: 标签页切换快捷键

**描述**: 支持快捷键切换日志标签页（可选功能）

**输入**:
- 键盘事件（Alt+1, Alt+2）- 使用 Alt 避免与浏览器快捷键冲突

**处理逻辑**:
1. Alt+1 切换到正常日志
2. Alt+2 切换到错误日志

**输出**:
- 快捷键响应

**注意**: 此功能为 Could 级别，可根据开发时间灵活实现

---

#### FR-041: SSE 实时日志连接

**描述**: 建立 SSE 连接，实时接收任务日志

**输入**:
- 任务 ID

**处理逻辑**:
1. 建立 SSE 连接 `/api/scheduler/tasks/{id}/logs/stream`
2. 接收实时日志事件
3. 根据日志类型（stdout/stderr）分发到对应标签页
4. 自动滚动到最新日志

**输出**:
- SSE 连接建立
- 实时日志接收

**异常处理**:
- 连接断开：显示断开状态，提供重连按钮
- 任务结束：显示"任务已结束"提示

#### FR-042: 日志类型过滤

**描述**: SSE 连接支持按日志类型过滤

**输入**:
- 日志类型参数（stdout/stderr/all）

**处理逻辑**:
1. 前端发送 type 参数
2. 后端只推送对应类型的日志

**输出**:
- 过滤后的实时日志

#### FR-043: 断线重连机制

**描述**: SSE 连接断开后自动重连

**输入**:
- 连接断开事件

**处理逻辑**:
1. 检测连接断开
2. 显示断开状态
3. 3 秒后自动重连
4. 最大重试次数：5 次
5. 超过最大重试后显示"连接失败"提示

**输出**:
- 自动重连行为
- 重连状态提示

#### FR-044: 连接生命周期管理

**描述**: 管理 SSE 连接的生命周期

**输入**:
- 任务状态变化事件

**处理逻辑**:
1. 任务开始：建立 SSE 连接
2. 任务运行中：保持连接，接收日志
3. 任务结束：接收完成事件，关闭连接
4. 任务失败：接收失败事件，关闭连接

**输出**:
- 连接状态管理

## 四、API 接口需求

### 4.1 日志分类接口

**与现有接口关系**:
- 现有接口 `GET /api/scheduler/logs` 和 `GET /api/scheduler/logs/{task_id}` 返回所有日志
- 新增接口 `GET /api/scheduler/tasks/{id}/logs/normal` 和 `/error` 提供分类日志
- 两者互补，现有接口用于日志列表总览，新接口用于详细日志查看

#### 获取正常日志

**接口**: `GET /api/scheduler/tasks/{id}/logs/normal`

**请求参数**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | path | 是 | - | 任务 ID |
| page | query | 否 | 1 | 页码 |
| limit | query | 否 | 100 | 每页数量，最大 500 |
| start_time | query | 否 | - | 开始时间（ISO 格式） |
| end_time | query | 否 | - | 结束时间（ISO 格式） |

**响应格式**:

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "log-uuid",
        "task_id": "task-uuid",
        "type": "stdout",
        "content": "执行任务步骤 1...",
        "timestamp": "2026-03-10T10:30:00.000Z"
      }
    ],
    "total": 150,
    "page": 1,
    "limit": 100,
    "pages": 2
  }
}
```

#### 获取错误日志

**接口**: `GET /api/scheduler/tasks/{id}/logs/error`

**请求参数**: 同正常日志

**响应格式**: 同正常日志

### 4.2 实时日志接口

**接口**: `GET /api/scheduler/tasks/{id}/logs/stream`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | path | 是 | 任务 ID |
| type | query | 否 | 日志类型（stdout/stderr/all），默认 all |

**响应格式** (SSE):

```
data: {"type": "stdout", "content": "执行中...", "timestamp": "..."}

event: complete
data: {"message": "任务已完成"}
```

**事件类型**:
| 事件 | 说明 |
|------|------|
| log | 日志数据 |
| complete | 任务完成 |
| error | 任务失败 |
| close | 连接关闭 |

### 4.3 日志搜索接口

**接口**: `GET /api/scheduler/tasks/{id}/logs/search`

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | path | 是 | 任务 ID |
| keyword | query | 是 | 搜索关键字 |
| regex | query | 否 | 是否使用正则表达式（默认 false） |
| type | query | 否 | 日志类型过滤（stdout/stderr/all） |
| page | query | 否 | 页码 |
| limit | query | 否 | 每页数量 |

**响应格式**:

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "log-uuid",
        "task_id": "task-uuid",
        "type": "stderr",
        "content": "Error: 连接失败",
        "timestamp": "2026-03-10T10:30:00.000Z",
        "match_positions": [[7, 14]]
      }
    ],
    "total": 5,
    "keyword": "error",
    "page": 1,
    "limit": 100,
    "pages": 1
  }
}
```

**match_positions**: 关键字在内容中的位置数组，用于前端高亮显示

## 八、风险与对策

| 风险 | 影响 | 可能性 | 应对策略 |
|------|------|--------|----------|
| SSE 连接不稳定 | 实时日志中断 | 中 | 断线重连机制 |

| 日志数据不一致 | 显示错误 | 低 | 实时刷新机制 |
