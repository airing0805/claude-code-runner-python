## 一、市场分析

### 1.1 需求背景

Claude Code Runner 是 Claude Code CLI 的 Web 服务封装，用户通过该工具与 Claude 进行交互式编程。在长期使用过程中，用户会产生大量的历史会话和提问记录。

### 1.2 用户痛点

| 痛点 | 描述 | 影响程度 |
|------|------|----------|
| 重复输入相似问题 | 用户经常需要向 Claude 提出类似的问题 | 高 |
| 难以回溯历史思路 | 无法快速找到之前的提问内容 | 中 |
| 效率低下 | 每次都要重新输入完整的问题描述 | 中 |

### 1.3 解决方案价值

- **提升效率**：用户可以快速查看和复用历史提问
- **降低重复劳动**：避免重复输入相同或相似的问题
- **增强用户体验**：提供便捷的历史记录浏览功能

---

## 二、产品定位

### 2.1 功能概述

添加提问历史记录功能，让用户能够快速查看之前向 Claude 提出的问题，方便重复使用类似的问题。参考现有历史记录功能的设计模式，实现项目列表和提问列表的二级导航。

### 2.2 目标用户

- Claude Code Runner 的日常使用者
- 需要频繁与 Claude 交互进行编程任务的开发者

### 2.3 核心价值

1. **快速访问**：一键查看所有历史提问
2. **项目隔离**：按项目维度组织提问记录
3. **便捷复用**：点击提问可继续相关会话

## 三、功能需求

### 3.1 功能优先级 (MoSCoW)

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **Must** | 项目列表展示 | 显示所有有会话的项目 |
| **Must** | 提问列表展示 | 按项目显示历史提问 |
| **Must** | 提问数据提取 | 从会话文件提取首次用户提问 |
| **Should** | 点击继续会话 | 点击提问进入对应会话 |
| **Should** | 缓存机制 | 缓存提取结果避免重复计算 |
| **Should** | 复制提问 | 一键复制提问内容到剪贴板 |
| **Could** | 搜索功能 | 搜索历史提问内容 |
| **Won't** | 编辑功能 | 本版本不支持编辑历史提问 |

### 3.2 详细功能需求

#### FR-001: 项目列表展示

**描述**: 显示用户所有有会话记录的项目列表

**输入**:
- 无（从系统获取）

**处理逻辑**:
1. 调用现有的 `/api/projects` API
2. 获取项目路径、会话数量等信息
3. 按会话数量排序显示

**输出**:
- 项目列表 UI
- 每项显示：项目路径、会话数量

**异常处理**:
- 无项目时显示空状态提示

#### FR-002: 提问列表展示

**描述**: 显示选中项目的所有历史提问

**输入**:
- 项目路径（encoded_name）

**处理逻辑**:
1. 调用新增的 `/api/projects/{project_name}/questions` API
2. 获取该项目的所有提问记录
3. 按时间倒序排列
4. 使用虚拟滚动优化长列表性能

**输出**:
- 提问列表 UI
- 每项显示：提问内容、时间戳

**异常处理**:
- 无提问时显示空状态提示
- 加载失败显示错误信息

#### FR-003: 提问数据提取

**描述**: 从会话 JSONL 文件中提取首次用户提问

**输入**:
- 会话文件路径

**提取规则**:

```python
# 1. 定位首次用户消息
# 寻找 type == "user" 的第一条记录

# 2. 提取提问内容
# 从 message.content 数组中提取所有 text 类型的内容

# 3. 过滤特殊标签
# 排除 <ide_selection>...</ide_selection> 内容
# 排除 <ide_opened_file>...</ide_opened_file> 内容
# 排除 <command-message>...</command-message> 内容
# 排除 <command-name>...</command-name> 内容
# 排除 <command-args>...</command-args> 内容

# 4. 拼接有效内容
# 将多个 text 内容按顺序拼接
```

**输出数据格式**:

```json
{
  "id": "session_uuid",
  "project_name": "E--workspaces-2026-project",
  "question_text": "帮我实现 xxx 功能",
  "session_id": "d9eb61b6-5eb1-42f7-974c-832e54859911",
  "timestamp": "2026-02-18T09:45:09.554Z"
}
```

**异常处理**:
- 文件读取失败：返回空提问
- JSON 解析失败：跳过该行
- 无用户消息：返回空提问

#### FR-004: 点击继续会话

**描述**: 点击提问项后跳转到对应会话

**输入**:
- 会话 ID（session_id）

**处理逻辑**:
1. 获取提问项关联的 session_id
2. 调用会话加载逻辑
3. 进入会话视图继续对话

**输出**:
- 切换到会话视图
- 加载历史会话内容

#### FR-005: 复制提问功能

**描述**: 一键复制提问内容到剪贴板

**输入**:
- 提问文本（question_text）

**处理逻辑**:
1. 用户点击复制按钮
2. 调用 `navigator.clipboard.writeText()` API
3. 显示"已复制"成功提示（Toast，2秒后消失）

**输出**:
- 提问内容复制到系统剪贴板
- 成功提示信息

**异常处理**:
- 复制失败：显示"复制失败，请手动选择复制"

## 四、数据提取规则详细说明

### 4.1 提取位置

| 字段 | 来源 | 路径 |
|------|------|------|
| session_id | 会话记录 | `sessionId` 或 `uuid` |
| timestamp | 用户消息 | `timestamp` |
| content | 用户消息 | `message.content` |

### 4.2 提取时机

**推荐方案**: 懒加载 + 缓存

1. **首次请求时提取**：用户访问项目提问列表时触发
2. **缓存结果**：提取后缓存到 `.metadata.json`
3. **失效策略**：基于文件修改时间（mtime）

### 4.3 过滤规则详解

**需要过滤的标签内容**:

| 标签 | 说明 | 过滤原因 |
|------|------|----------|
| `<ide_selection>` | IDE 选中的代码 | 非用户主动输入 |
| `<ide_opened_file>` | IDE 打开的文件 | 环境信息 |
| `<command-message>` | 命令消息标识 | 系统生成 |
| `<command-name>` | 命令名称 | 系统生成 |
| `<command-args>` | 命令参数 | 系统生成 |

**过滤逻辑伪代码**:

```python
import re

def extract_question_text(content_array: list) -> str:
    """提取有效提问文本"""
    texts = []
    
    for item in content_array:
        if item.get("type") != "text":
            continue
        
        text = item.get("text", "")
        
        # 过滤特殊标签
        patterns = [
            r'<ide_selection>.*?</ide_selection>',
            r'<ide_opened_file>.*?</ide_opened_file>',
            r'<command-message>.*?</command-message>',
            r'<command-name>.*?</command-name>',
            r'<command-args>.*?</command-args>',
        ]
        
        for pattern in patterns:
            text = re.sub(pattern, '', text, flags=re.DOTALL)
        
        # 清理空白
        text = text.strip()
        
        if text:
            texts.append(text)
    
    return '\n'.join(texts)
```

### 4.4 数据格式规范

**QuestionItem 数据模型**:

```python
@dataclass
class QuestionItem:
    """提问记录项"""
    id: str                      # 会话 UUID
    project_name: str            # 项目编码名称
    question_text: str           # 提问内容（完整文本，不限长度）
    session_id: str              # 会话 ID（与 id 相同）
    timestamp: str               # ISO 格式时间戳
    
    def to_dict(self) -> dict:
        return {
            "id": self.id,
            "project_name": self.project_name,
            "question_text": self.question_text,
            "session_id": self.session_id,
            "timestamp": self.timestamp,
        }
```

## 五、缓存机制需求

### 5.1 缓存架构

**复用现有 MetadataCache 类**，扩展支持提问内容缓存。

### 5.2 缓存数据结构

**优化策略**: 缓存仅存储标志位，不存储完整提问文本

```json
{
  "session_uuid.jsonl": {
    "mtime": 1708123456.789,
    "title": "会话标题",
    "message_count": 10,
    "tools": ["Read", "Write"],
    "has_question": true,
    "question_timestamp": "2026-02-18T09:45:09.554Z"
  }
}
```

**说明**:
- `has_question`: 是否存在有效用户提问
- `question_timestamp`: 提问时间戳（用于排序）
- 完整 `question_text` 实时从文件提取

### 5.3 缓存策略

| 策略 | 配置 | 说明 |
|------|------|------|
| 缓存位置 | `.metadata.json` | 项目目录下 |
| 失效条件 | mtime 变化 | 文件修改时缓存失效 |
| 最大条目 | 无限制 | 按项目隔离存储 |

### 5.4 性能要求

| 指标 | 目标值 |
|------|--------|
| 缓存命中率 | > 95% |
| 缓存读取时间 | < 10ms |
| 首次提取时间 | < 500ms/文件 |

## 六、API 接口需求

### 6.1 获取项目提问列表

**接口**: `GET /api/projects/{project_name}/questions`

**请求参数**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| project_name | path | 是 | - | 项目编码名称 |
| page | query | 否 | 1 | 页码，从 1 开始 |
| limit | query | 否 | 20 | 每页数量，最大 100 |

**响应格式**:

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "d9eb61b6-5eb1-42f7-974c-832e54859911",
        "project_name": "E--workspaces-2026-project",
        "question_text": "帮我实现 xxx 功能",
        "question_text_truncated": "帮我实现 xxx 功...",
        "session_id": "d9eb61b6-5eb1-42f7-974c-832e54859911",
        "timestamp": "2026-02-18T09:45:09.554Z",
        "time_display": "昨天 10:30"
      }
    ],
    "total": 50,
    "page": 1,
    "limit": 20,
    "pages": 3
  }
}
```

**错误响应**:

| 状态码 | 错误码 | 说明 |
|--------|--------|------|
| 404 | PROJECT_NOT_FOUND | 项目不存在 |
| 500 | INTERNAL_ERROR | 内部错误 |

- 展开后按钮变为"收起"
- 复制时复制完整内容（非截断内容）

| < 24 小时 | N 小时前 | 3 小时前 |
| 昨天 | 昨天 HH:mm | 昨天 10:30 |
| < 7 天 | N 天前 | 3 天前 |
| >= 7 天 | YYYY-MM-DD HH:mm | 2026-02-28 14:30 |
