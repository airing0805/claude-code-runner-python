# 任务调度 - 需求规格说明书

## 1. 概述

### 1.1 目的
本文档定义了Claude Code Runner任务调度模块的功能需求、非功能需求和技术约束，为开发、测试和验收提供依据。

### 1.2 范围
任务调度模块提供以下核心能力：
- 任务队列管理：手动添加、删除、清空任务
- 定时任务管理：基于Cron表达式的自动化任务调度
- 任务状态跟踪：实时监控任务执行状态
- 调度器控制：启动、停止、刷新调度服务
- 任务历史查询：查看已完成和失败的任务记录
- **任务日志管理：独立的正常日志和错误日志标签页，支持实时滚动和分页加载**

### 1.3 目标用户
- **开发团队**：需要批量执行AI编程任务
- **DevOps工程师**：需要自动化代码审查、测试生成等任务
- **平台管理员**：需要监控和管理任务执行情况
- **普通用户**：需要定时执行重复性AI任务

## 2. 功能需求

### 2.1 任务队列管理

> **队列设计核心原则**（重要）
> - **队列积压是正常的**：同一个定时任务在队列中多次出现是正常现象。Cron 每次触发都会添加任务到队列，如果任务执行慢，队列会积压，这是正常的计划安排。
> - **禁止去重**：不应该基于 prompt 进行去重。不同的定时任务可能有相同的 prompt，但它们是独立的任务，不应该互相干扰。
> - **队列反映真实状态**：队列应该真实反映任务状态，用户可以看到有多少任务在等待。如果去除重复，用户就看不到任务堆积，无法判断任务执行是否有问题。

#### 2.1.1 添加任务到队列
- **FR-001**: 用户可以提交任务描述（prompt）到任务队列
- **FR-002**: 支持指定工作目录（workspace），默认为当前工作目录
- **FR-003**: 支持设置任务超时时间（1秒-1小时，默认10分钟）
- **FR-004**: 支持配置允许使用的工具列表（Read/Write/Edit/Glob/Grep/Bash/Task/TodoWrite/WebFetch/WebSearch/NotebookEdit）
- **FR-005**: 支持启用自动批准模式，跳过工具操作确认
- **FR-006**: 任务添加后立即返回任务ID和基本信息

#### 2.1.2 队列操作
- **FR-007**: 用户可以查看当前队列中的所有任务
- **FR-008**: 用户可以从队列中删除指定任务
- **FR-009**: 用户可以清空整个任务队列
- **FR-010**: 队列操作应提供实时反馈和错误处理

### 2.2 定时任务管理

#### 2.2.1 创建定时任务
- **FR-011**: 用户可以创建基于Cron表达式的定时任务
- **FR-012**: 定时任务必须包含名称、描述、Cron表达式等基本信息
- **FR-013**: 支持与普通任务相同的配置选项（工作目录、超时、工具限制、自动批准）
- **FR-014**: 系统应验证Cron表达式的有效性并预览下次执行时间

#### 2.2.2 定时任务操作
- **FR-015**: 用户可以查看所有定时任务列表
- **FR-016**: 用户可以编辑现有的定时任务配置
- **FR-017**: 用户可以启用/禁用定时任务（不影响任务定义，仅控制执行）
- **FR-018**: 用户可以立即执行指定的定时任务（不等待Cron触发）
- **FR-019**: 用户可以删除不再需要的定时任务

### 2.3 任务状态跟踪

#### 2.3.1 状态分类
- **FR-020**: 任务状态包括：待执行（pending）、运行中（running）、已完成（completed）、失败（failed）
- **FR-021**: 系统应实时更新任务状态变化
- **FR-022**: 运行中任务应显示已执行时长

#### 2.3.2 状态查询
- **FR-023**: 用户可以分别查看不同状态的任务列表
- **FR-024**: 用户可以查看指定任务的详细信息
- **FR-025**: 已完成和失败任务支持分页查询（每页1-100条，默认20条）

### 2.4 调度器控制

#### 2.4.1 基本控制
- **FR-026**: 用户可以启动任务调度器服务
- **FR-027**: 用户可以停止任务调度器服务
  - 优雅停止：等待当前运行任务完成后停止调度器
  - 强制停止：立即停止调度器，当前运行任务继续执行但不再启动新任务
- **FR-028**: 用户可以手动刷新所有数据

#### 2.4.2 任务控制
- **FR-028.1**: 用户可以取消运行中的任务
  - 取消后任务状态变为 CANCELLED
  - 取消操作需要用户二次确认
  - 取消后的任务不触发重试机制
  - 取消后的任务移动到 failed.json 文件

#### 2.4.3 状态监控
- **FR-029**: 系统应显示调度器当前状态（运行中/已停止）
- **FR-030**: 系统应显示各类任务的统计数量（队列、定时、运行中）

### 2.5 任务详情与日志

#### 2.5.1 任务详情
- **FR-031**: 用户可以查看任务的完整执行信息
- **FR-032**: 详情包括：任务描述、配置参数、执行结果、文件变更列表、工具使用记录、执行耗时、费用等

#### 2.5.2 执行日志
- **FR-033**: 系统应记录任务执行过程中的详细日志
- **FR-034**: 日志应包含时间戳、日志级别、消息内容和上下文信息
- **FR-035**: 用户可以查看指定任务的完整执行日志

### 2.6 任务日志展示（新增）

#### 2.6.1 独立日志标签页
- **FR-036**: 系统必须提供两个独立的主标签页用于展示任务执行日志：**正常日志**和**错误日志**
- **FR-037**: 正常日志标签页应位于任务调度界面的主标签页区域，与其他标签页（队列、定时、运行、完成、失败）并列
- **FR-038**: 错误日志标签页应位于任务调度界面的主标签页区域，与其他标签页（队列、定时、运行、完成、失败）并列
- **FR-039**: 正常日志标签页显示所有任务的标准输出流（stdout）和成功执行日志
- **FR-040**: 错误日志标签页显示所有任务的错误输出流（stderr）和失败执行日志

#### 2.6.2 实时日志显示
- **FR-041**: 任务执行过程中，正常日志应实时滚动显示在正常日志标签页
- **FR-042**: 任务执行过程中，错误日志应实时滚动显示在错误日志标签页
- **FR-043**: 系统应自动滚动到底部，跟随最新日志行
- **FR-044**: 提供暂停/继续自动滚动功能，方便用户查看特定日志段

#### 2.6.3 历史日志显示
- **FR-045**: 任务完成后，默认在对应日志标签页显示最近100行日志
- **FR-046**: 支持向上滚动加载更多历史日志（无限滚动或分页加载）
- **FR-047**: 日志区域应提供复制功能，允许用户复制全部或选中的日志内容

#### 2.6.4 日志样式与分类
- **FR-048**: 正常日志使用常规样式（黑色文字，白色背景）
- **FR-049**: 错误日志需高亮显示（红色背景或红色边框）

#### 2.6.5 日志数据映射
- **FR-048**: 前端必须正确映射后端日志字段，从`task.result.message`获取完整的执行日志
- **FR-049**: 禁止使用不存在的字段（如`task.output`），确保前后端数据一致性

## 3. 非功能需求

### 3.1 性能需求
- **NFR-001**: 任务添加响应时间 ≤ 100ms
- **NFR-002**: 任务列表查询响应时间 ≤ 500ms（1000个任务以内）
- **NFR-003**: 调度器轮询间隔默认10秒，可配置
- **NFR-004**: 支持同时运行多个任务（受限于系统资源）
- **NFR-005**: 日志实时显示延迟 ≤ 500ms

### 3.2 安全需求
- **NFR-006**: 工作目录路径必须进行安全验证，防止路径遍历攻击
  - 禁止路径遍历：不允许使用 `..` 跳出项目根目录
  - 禁止访问系统目录：不允许访问 `/etc`, `/root`, `C:\\Windows` 等系统敏感目录
  - 白名单机制：仅允许访问配置文件中定义的允许目录
  - 路径规范化：统一处理相对路径和绝对路径，防止绕过验证
- **NFR-007**: 工具使用必须限制在白名单范围内
- **NFR-008**: 任务执行应受超时限制，防止无限执行
- **NFR-009**: 敏感操作（如删除、清空、取消）需要用户确认
- **NFR-010**: API 访问安全（当前版本为单机应用，无认证要求）
  - 未来扩展：支持 API Token 认证
  - 未来扩展：支持 IP 白名单访问控制
- **NFR-011**: 敏感数据保护
  - 日志中不应包含 API Key、密码等敏感信息
  - 响应中不应返回敏感配置信息
  - 环境变量敏感信息需要脱敏处理

### 3.3 可用性需求
- **NFR-012**: Web界面应直观易用，提供清晰的状态指示
- **NFR-013**: 系统应提供实时反馈和错误提示
- **NFR-014**: 支持自动刷新机制（默认5秒）
- **NFR-015**: 提供Cron表达式帮助和示例
- **NFR-016**: 日志区域应支持搜索功能，便于查找特定内容

### 3.4 可靠性需求
- **NFR-017**: 数据存储应保证原子性和一致性
- **NFR-018**: 并发操作应通过文件锁机制保证数据安全
- **NFR-019**: 系统崩溃后应能恢复未完成的任务状态

### 3.5 可维护性需求
- **NFR-020**: 代码应有充分的单元测试和集成测试覆盖，覆盖率 ≥ 80%
  - 使用 pytest-cov 工具验证覆盖率
  - 在 CI/CD 流程中自动执行覆盖率检查
  - 核心模块（scheduler, storage, executor）覆盖率要求 ≥ 90%
- **NFR-021**: 配置参数应集中管理，便于调整
- **NFR-022**: 日志记录应详细且结构化，便于问题排查

## 4. 技术约束

### 4.1 存储约束
- **TC-001**: 使用JSON文件存储任务数据（data/目录下）
- **TC-002**: 历史记录最多保留1000条
- **TC-003**: 日志采用JSONL格式（每行一个JSON对象）

### 4.2 API约束
- **TC-004**: REST API前缀为`/api/scheduler`
- **TC-005**: 所有API响应遵循统一格式：`{success: boolean, data: any, message: string}`
- **TC-006**: 错误响应包含错误码和详细信息

### 4.3 前端约束
- **TC-007**: 使用原生JavaScript实现，无外部框架依赖
- **TC-008**: 界面采用七标签页布局（队列、定时、运行、完成、失败、**正常日志**、**错误日志**）
- **TC-009**: 支持响应式设计，适配不同屏幕尺寸
- **TC-010**: 日志区域必须支持实时滚动和暂停功能

## 5. 数据模型

### 5.1 Task（任务）
```python
class Task:
    id: str                    # 任务唯一标识 (UUID)
    prompt: str               # 任务描述/提示词
    workspace: str            # 工作目录，默认"."
    timeout: int              # 超时时间（毫秒），默认600000
    auto_approve: bool        # 是否自动批准工具操作
    allowed_tools: list[str]  # 允许使用的工具列表
    created_at: str           # 创建时间 (ISO 8601)
    started_at: str           # 开始执行时间
    finished_at: str          # 结束时间
    retries: int              # 当前重试次数
    status: TaskStatus        # 任务状态
    source: TaskSource        # 任务来源
    scheduled_id: str         # 定时任务ID（如果来自定时任务）
    scheduled_name: str       # 定时任务名称
    result: dict              # 执行结果，包含message字段存储完整日志
    error: str                # 错误信息
    files_changed: list[str]  # 变更的文件列表
    tools_used: list[str]     # 使用过的工具列表
    cost_usd: float           # 消耗费用（美元）
    duration_ms: int          # 执行耗时（毫秒）
```

### 5.2 ScheduledTask（定时任务）
```python
class ScheduledTask:
    id: str                   # 任务唯一标识 (UUID)
    name: str                 # 任务名称
    prompt: str               # 任务描述/提示词
    cron: str                 # Cron表达式
    workspace: str            # 工作目录，默认"."
    timeout: int              # 超时时间（毫秒），默认600000
    auto_approve: bool        # 是否自动批准工具操作
    allowed_tools: list[str]  # 允许使用的工具列表
    enabled: bool             # 是否启用
    last_run: str             # 上次执行时间
    next_run: str             # 下次执行时间
    created_at: str           # 创建时间
    updated_at: str           # 更新时间
    run_count: int            # 已执行次数
```

### 5.3 枚举类型
- **TaskStatus**: PENDING, RUNNING, COMPLETED, FAILED, CANCELLED
- **TaskSource**: MANUAL, SCHEDULED, IMMEDIATE

## 6. API接口规范

### 6.1 任务队列管理
- `POST /api/scheduler/tasks` - 添加任务到队列
- `GET /api/scheduler/tasks` - 获取队列列表
- `DELETE /api/scheduler/tasks/{task_id}` - 删除队列任务
- `DELETE /api/scheduler/tasks/clear` - 清空队列

### 6.2 定时任务管理
- `POST /api/scheduler/scheduled-tasks` - 创建定时任务
- `GET /api/scheduler/scheduled-tasks` - 获取定时任务列表
- `PATCH /api/scheduler/scheduled-tasks/{task_id}` - 更新定时任务
- `DELETE /api/scheduler/scheduled-tasks/{task_id}` - 删除定时任务
- `POST /api/scheduler/scheduled-tasks/{task_id}/toggle` - 启用/禁用定时任务
- `POST /api/scheduler/scheduled-tasks/{task_id}/run` - 立即执行定时任务

### 6.3 任务状态查询
- `GET /api/scheduler/tasks/running` - 获取运行中任务
- `GET /api/scheduler/tasks/completed` - 获取已完成任务（分页）
- `GET /api/scheduler/tasks/failed` - 获取失败任务（分页）
- `GET /api/scheduler/tasks/{task_id}` - 获取任务详情

### 6.4 调度器控制
- `GET /api/scheduler/status` - 获取调度器状态
- `POST /api/scheduler/start` - 启动调度器
- `POST /api/scheduler/stop` - 停止调度器

### 6.5 Cron表达式验证
- `POST /api/scheduler/validate-cron` - 验证Cron表达式
- `GET /api/scheduler/cron-examples` - 获取Cron示例

### 6.6 任务日志查询
- `GET /api/scheduler/logs` - 获取任务日志
- `GET /api/scheduler/logs/{task_id}` - 获取指定任务日志
- `DELETE /api/scheduler/logs` - 清空日志

## 8. 附录

### 8.1 Cron表达式格式
支持标准Cron格式：`minute hour day month weekday`
- 分钟（0-59）
- 小时（0-23）
- 日期（1-31）
- 月份（1-12）
- 星期（0-6，0为周日）

支持特殊字符：
- `*` - 任意值
- `,` - 值列表
- `-` - 范围
- `/` - 步长
- `@hourly`, `@daily` - 别名

### 8.2 工具白名单
- Read - 读取文件内容
- Write - 写入文件内容  
- Edit - 编辑文件内容
- Glob - 文件路径匹配
- Grep - 文本搜索
- Bash - 执行shell命令
- Task - 创建子任务
- TodoWrite - 写入TODO列表
- WebFetch - 获取网页内容
- WebSearch - 网络搜索
- NotebookEdit - Jupyter notebook编辑
