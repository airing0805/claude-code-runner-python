## 1. 需求概述

### 1.1 模块目标

任务状态跟踪模块负责管理和展示任务的执行状态，包括待执行、运行中、已完成、失败、已取消等状态的定义、转换和展示。

### 1.2 设计原则

- 任务状态必须实时更新
- 用户可以查看不同状态的任务列表
- 状态转换必须有明确的规则
- 运行中的任务应显示执行时长

## 2. 功能需求分析

### 2.1 FR-020: 任务状态枚举定义

#### 需求描述
任务状态包括：待执行（pending）、运行中（running）、已完成（completed）、失败（failed）

#### 状态定义

| 状态值 | 英文名 | 说明 | 适用场景 |
|--------|--------|------|----------|
| pending | PENDING | 待执行 | 任务刚加入队列，等待执行 |
| running | RUNNING | 运行中 | 任务正在执行 |
| completed | COMPLETED | 已完成 | 任务成功执行完成 |
| failed | FAILED | 失败 | 任务执行失败 |
| cancelled | CANCELLED | 已取消 | 任务被用户取消 |

#### 状态转换图

```
                    +---------+
                    | PENDING |
                    +---------+
                         |
           +-------------+-------------+
           |             |             |
           v             v             v
      +----------+  +----------+  +----------+
      | RUNNING  |  | FAILED   |  | CANCELLED|
      +----------+  +----------+  +----------+
           |
           v
      +-----------+
      | COMPLETED |
      +-----------+
```

#### 状态详细说明

##### PENDING（待执行）
- 任务已创建，等待调度器取出执行
- 可被用户删除或取消
- 状态字段：`status = "pending"`

##### RUNNING（运行中）
- 任务已被调度器取出，正在执行
- 记录开始执行时间 `started_at`
- 不可被删除，可被取消
- 状态字段：`status = "running"`

##### COMPLETED（已完成）
- 任务成功执行完成
- 记录结束时间 `finished_at`
- 记录执行结果 `result`
- 记录工具使用 `tools_used`
- 记录变更文件 `files_changed`
- 记录执行费用 `cost_usd`
- 记录执行耗时 `duration_ms`
- 状态字段：`status = "completed"`

##### FAILED（失败）
- 任务执行失败
- 记录结束时间 `finished_at`
- 记录错误信息 `error`
- 记录失败原因
- 可能触发重试机制
- 状态字段：`status = "failed"`

##### CANCELLED（已取消）
- 任务被用户主动取消
- 记录取消时间
- 不触发重试机制
- 状态字段：`status = "cancelled"`

#### 验收标准

- [ ] 定义所有5种状态
- [ ] 状态值正确映射
- [ ] 状态转换规则明确

### 2.2 FR-021: 实时状态更新

#### 需求描述
系统应实时更新任务状态变化

#### 用户场景

| 场景 | 描述 |
|------|------|
| 场景1 | 队列中的任务被取出执行，状态变为running |
| 场景2 | 任务执行完成，状态变为completed |
| 场景3 | 任务执行失败，状态变为failed |

#### 实现方式

| 机制 | 说明 |
|------|------|
| 自动刷新 | 前端默认5秒刷新一次 |
| 手动刷新 | 用户点击刷新按钮立即更新 |
| 实时推送 | 未来版本考虑使用WebSocket |

#### 更新时机

| 触发事件 | 状态变更 |
|----------|----------|
| 调度器取出任务 | pending -> running |
| 任务执行成功 | running -> completed |
| 任务执行失败 | running -> failed |
| 用户取消任务 | pending/running -> cancelled |

#### 验收标准

- [ ] 状态自动更新
- [ ] 更新延迟小于5秒
- [ ] 状态显示正确

### 2.3 FR-022: 显示已执行时长

#### 需求描述
运行中任务应显示已执行时长

#### 用户场景

| 场景 | 描述 |
|------|------|
| 场景1 | 用户查看任务执行进度 |
| 场景2 | 判断任务是否卡住 |

#### 计算规则

```
已执行时长 = 当前时间 - 开始执行时间 (started_at)
```

#### 显示格式

| 时长范围 | 显示格式 | 示例 |
|----------|----------|------|
| < 1分钟 | X秒 | 45秒 |
| < 1小时 | X分X秒 | 5分30秒 |
| >= 1小时 | X小时X分 | 2小时15分 |

#### 数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| started_at | string | 开始执行时间（ISO 8601） |
| duration_ms | integer | 已执行时长（毫秒，仅完成后有值） |

#### 验收标准

- [ ] 实时显示已执行时长
- [ ] 格式正确美观
- [ ] 超过超时时间有提示

### 2.4 FR-023: 查看不同状态的任务

#### 用户场景

#### 失败任务（Failed）

### 2.6 FR-025: 分页查询

#### 需求描述
已完成和失败任务支持分页查询（每页1-100条，默认20条）

#### 用户场景

| 场景 | 描述 |
|------|------|
| 场景1 | 查看大量已完成任务 |
| 场景2 | 查看失败任务历史 |

#### 输入

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | integer | 否 | 1 | 页码，从1开始 |
| limit | integer | 否 | 20 | 每页条数，范围1-100 |

#### 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| items | array[Task] | 任务列表 |
| total | integer | 总条数 |
| page | integer | 当前页码 |
| limit | integer | 每页条数 |
| pages | integer | 总页数 |

#### 示例响应

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "uuid-xxxx",
        "prompt": "任务描述",
        "status": "completed",
        "finished_at": "2026-03-07T10:00:00Z",
        "duration_ms": 120000,
        "cost_usd": 0.05
      }
    ],
    "total": 150,
    "page": 1,
    "limit": 20,
    "pages": 8
  },
  "message": ""
}
```

#### 业务规则

1. page 最小值为 1
2. limit 最小值为 1，最大值为 100
3. 当 page 大于总页数时，返回最后一页
4. 任务按完成时间倒序排列

#### 验收标准

- [ ] 支持分页参数
- [ ] 分页参数范围正确
- [ ] 返回正确的分页信息
- [ ] 超过范围有正确处理

## 3. 状态转换规则

### 3.1 状态转换矩阵

| 当前状态 | 目标状态 | 触发条件 | 备注 |
|----------|----------|----------|------|
| PENDING | RUNNING | 调度器取出任务执行 | |
| PENDING | CANCELLED | 用户取消任务 | |
| PENDING | FAILED | 系统错误 | 极少情况 |
| RUNNING | COMPLETED | 任务执行成功 | 记录结果 |
| RUNNING | FAILED | 任务执行失败 | 记录错误 |
| RUNNING | CANCELLED | 用户取消任务 | 不触发重试 |
| COMPLETED | - | 终止状态 | 不可转换 |
| FAILED | RUNNING | 重试机制 | 最多N次 |
| FAILED | - | 终止状态 | 不可转换 |
| CANCELLED | - | 终止状态 | 不可转换 |

### 3.2 状态字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| status | TaskStatus | 当前状态 |
| created_at | string | 创建时间（PENDING时设置） |
| started_at | string | 开始执行时间（转换为RUNNING时设置） |
| finished_at | string | 结束时间（转换为COMPLETED/FAILED/CANCELLED时设置） |

## 5. 数据模型

### 5.1 TaskStatus 枚举

```python
class TaskStatus(str, Enum):
    PENDING = "pending"      # 待执行
    RUNNING = "running"     # 运行中
    COMPLETED = "completed"  # 已完成
    FAILED = "failed"       # 失败
    CANCELLED = "cancelled" # 已取消
```

### 5.2 TaskSource 枚举

```python
class TaskSource(str, Enum):
    MANUAL = "manual"     # 手动添加
    SCHEDULED = "scheduled"  # 定时任务触发
    IMMEDIATE = "immediate"   # 立即执行定时任务
```

### 5.3 任务时间线

```
时间线：
|--------|----------------|----------------|
创建    开始执行         结束
  |        |                |
  v        v                v
PENDING  RUNNING      COMPLETED/FAILED/CANCELLED
         (duration_ms)    (duration_ms = finished_at - started_at)
```
