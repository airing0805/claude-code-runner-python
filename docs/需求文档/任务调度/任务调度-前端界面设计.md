# 任务调度-前端界面设计

## 1. 整体布局架构

### 1.1 页面结构
主布局采用侧边栏 + 主内容区模式，包含任务调度、定时任务、执行监控、调度器状态、**正常日志查看**、**错误日志查看**六个主要功能模块。

### 1.2 响应式设计
- 桌面端(≥1200px): 完整侧边栏显示
- 平板端(768px-1199px): 折叠侧边栏（仅图标）
- 移动端(<768px): 隐藏侧边栏，通过汉堡菜单访问

## 2. 任务队列页面设计

### 2.1 页面布局
- 顶部操作栏：添加任务按钮、批量操作下拉菜单、搜索框
- 中部任务列表：表格形式展示任务信息
- 底部分页控件：支持分页浏览

### 2.2 任务列表字段
| 字段 | 显示内容 | 交互 |
|------|---------|------|
| 状态 | 状态徽章（待执行/运行中/已完成/失败） | 点击筛选 |
| 描述 | 任务prompt前50字符 | 点击查看详情 |
| 工作目录 | workspace路径 | - |
| 创建时间 | ISO 8601格式时间 | - |
| 超时 | 超时时间（分钟） | - |
| 操作 | 删除/重试/查看详情按钮 | 对应操作 |

### 2.3 任务详情对话框
- 任务基本信息（完整prompt、workspace、timeout等）
- 执行日志面板（实时滚动显示）
- 结果信息（文件变更、工具使用记录）
- 操作按钮（关闭、重试）

## 3. 定时任务页面设计

### 3.1 页面布局
- 顶部操作栏：添加定时任务按钮、批量启用/禁用
- 中部定时任务列表：表格展示定时任务配置
- Cron表达式可视化编辑器（单独对话框）

### 3.2 定时任务列表字段
| 字段 | 显示内容 | 交互 |
|------|---------|------|
| 名称 | 任务名称 | - |
| 状态 | 启用/禁用开关 | 切换状态 |
| Cron表达式 | 格式化显示的Cron表达式 | 点击编辑 |
| 描述 | 任务描述 | - |
| 下次执行 | 下次预计执行时间 | - |
| 统计 | 执行次数/成功率 | - |
| 操作 | 编辑/删除/立即执行 | 对应操作 |

### 3.3 Cron表达式编辑器
- 预设模板选择（每小时/每天/每周等）
- 可视化表单（分钟、小时、日、月、周选择器）
- 手动输入框（支持5位和6位格式）
- 实时预览下次执行时间

## 4. 执行监控页面设计

### 4.1 实时监控面板
- 运行中任务卡片（显示进度、耗时、当前状态）
- 任务日志实时输出（带自动滚动和暂停功能）
- 资源使用情况（CPU、内存占用）

### 4.2 任务控制功能
- 取消任务按钮
- 暂停/继续日志输出
- 清空日志
- 导出日志

## 5. 调度器状态页面设计

### 5.1 状态概览
- 调度器运行状态（运行中/已停止）
- 任务统计（待执行、运行中、已完成、失败数量）
- 系统资源使用情况
- 最近活动时间

### 5.2 控制面板
- 启动/停止调度器按钮
- 配置查看/编辑链接
- 日志查看入口
- 系统健康检查

## 6. 日志查看页面设计（新增）

### 6.1 页面布局
- **正常日志主标签页**：在任务调度界面中添加"正常日志"主标签页，与其他标签页（队列、定时、运行、完成、失败）并列
- **错误日志主标签页**：在任务调度界面中添加"错误日志"主标签页，与其他标签页（队列、定时、运行、完成、失败）并列
- **正常日志显示区域**：占据页面主要区域，显示所有任务的标准输出和成功日志
- **错误日志显示区域**：占据页面主要区域，显示所有任务的错误输出和失败日志
- **控制工具栏**：每个日志标签页都包含暂停/继续自动滚动、复制日志、清空日志等按钮

### 6.2 日志显示功能
- **实时滚动**：任务执行过程中，正常日志在正常日志标签页自动滚动，错误日志在错误日志标签页自动滚动
- **暂停滚动**：用户可点击暂停按钮停止自动滚动，便于查看特定日志段
- **继续滚动**：用户可点击继续按钮恢复自动滚动
- **历史加载**：任务完成后，默认显示最近100行日志，支持向上滚动加载更多
- **搜索功能**：支持在日志中搜索特定关键词

### 6.3 日志样式规范
- **正常日志**：黑色文字，白色背景，常规字体
- **错误日志**：红色文字，浅红色背景（#fee2e2），或红色边框突出显示
- **时间戳**：每行日志前显示时间戳（HH:mm:ss格式）
- **代码高亮**：支持语法高亮显示代码片段

### 6.4 交互功能
- **复制日志**：提供"复制全部"和"复制选中"功能
- **导出日志**：支持将日志导出为文本文件
- **清除日志**：清空当前日志显示区域
- **自动聚焦**：切换到日志tab时自动聚焦到日志区域

## 7. UI组件规范

### 7.1 颜色规范
- 主色调: #3b82f6 (蓝色)

- 警告: #f59e0b (橙色)
- 错误: #ef4444 (红色)

- 主要按钮: 蓝色背景，白色文字
- 次要按钮: 白色背景，蓝色边框
- 危险按钮: 红色背景，白色文字
- 禁用状态: 灰色背景，不可点击

## 8. 交互规范

### 8.1 表单验证
- 实时验证（失去焦点时）
- 错误提示显示在字段下方
- 必填字段标记星号(*)

### 8.2 加载状态
- 按钮加载: 显示旋转图标
- 页面加载: 全屏loading遮罩
- 数据加载: 表格骨架屏

### 8.3 确认对话框
- 删除操作: "确定要删除此任务吗？"
- 取消操作: "确定要取消此任务吗？"
- 批量操作: "确定要对选中的{count}个任务执行此操作吗？"

## 9. 数据绑定规范

### 9.1 任务状态映射
- pending → 待执行
- running → 运行中  
- completed → 已完成
- failed → 已失败
- cancelled → 已取消

### 9.2 时间格式化
- 创建时间: YYYY-MM-DD HH:mm:ss
- 耗时显示: X分Y秒 或 X小时Y分

### 9.3 数字格式化
- 文件大小: KB/MB/GB 自动转换
- 执行次数: 千分位分隔

### 9.4 日志数据映射
- **日志源**：从`task.result.message`字段获取完整执行日志
- **字段验证**：确保不使用不存在的字段（如`task.output`）
- **数据同步**：前后端日志字段保持一致
