## 1. 任务对象 (Task)

```python
class Task:
    # 必填字段
    id: str                          # 任务唯一标识，UUID格式
    prompt: str                      # 任务描述/指令

    # 可选字段
    workspace: str = "."             # 工作目录，默认项目根目录
    timeout: int = 600000            # 超时时间(ms)，默认 600000
    auto_approve: bool = False        # 自动批准，默认 false
    allowed_tools: list[str] | None = None  # 可用工具列表，None表示不限制
    created_at: str                  # 创建时间，ISO 8601 格式

    # 定时任务字段
    source: str = "manual"           # 任务来源: manual(手动)/scheduled(定时触发)/immediate(立即执行)
    scheduled_id: str | None = None  # 对应的定时任务 ID
    scheduled_name: str | None = None  # 定时任务名称

    # 执行状态字段
    retries: int = 0                # 已重试次数
    started_at: str | None = None   # 开始执行时间
    finished_at: str | None = None  # 完成/失败时间
    status: TaskStatus = TaskStatus.PENDING  # 执行状态
    error: str | None = None        # 错误信息

    # 执行结果字段
    result: dict | None = None      # 执行结果数据
    files_changed: list[str] = []    # 变更的文件列表
    tools_used: list[str] = []       # 使用过的工具列表
    cost_usd: float | None = None   # API 消耗费用（美元）
    duration_ms: int | None = None   # 执行耗时（毫秒）
```

## 2. 定时任务对象 (ScheduledTask)

```python
class ScheduledTask:
    # 必填字段
    id: str                          # 定时任务唯一标识
    name: str                        # 任务名称
    cron: str                        # Cron 表达式
    prompt: str                      # 任务描述

    # 可选字段
    workspace: str = "."             # 工作目录
    timeout: int = 600000           # 超时时间
    auto_approve: bool = False       # 自动批准
    allowed_tools: list[str] | None = None  # 可用工具列表
    enabled: bool = True             # 是否启用，默认 true
    created_at: str                  # 创建时间
    updated_at: str                  # 更新时间

    # 状态字段
    last_run: str | None = None     # 上次执行时间（ISO 8601 格式）
    next_run: str | None = None     # 下次预计执行时间（ISO 8601 格式）
    run_count: int = 0               # 已执行次数

    # 统计字段
    success_count: int = 0           # 成功执行次数
    failed_count: int = 0            # 失败执行次数
    last_error: str | None = None    # 最后一次错误信息
    last_success: str | None = None  # 最后一次成功时间
```

## 3. 任务状态枚举 (TaskStatus)

```python
class TaskStatus(Enum):
    PENDING = "pending"       # 待执行
    RUNNING = "running"       # 运行中
    COMPLETED = "completed"   # 已完成
    FAILED = "failed"         # 失败
    CANCELLED = "cancelled"   # 已取消
```

## 4. 任务来源枚举 (TaskSource)

```python
class TaskSource(Enum):
    MANUAL = "manual"         # 手动添加
    SCHEDULED = "scheduled"  # 定时任务自动触发
    IMMEDIATE = "immediate"  # 立即执行定时任务
```

## 5. 文件存储结构

### 5.1 JSON文件结构

```
data/
├── queue.json       # 待执行任务队列 { "tasks": [Task] }
├── running.json     # 当前运行任务 { "tasks": [Task] }
├── completed.json   # 已完成任务 { "tasks": [Task] }
├── failed.json      # 失败任务 { "tasks": [Task] }
├── cancelled.json   # 已取消任务 { "tasks": [Task] }
├── scheduled.json   # 定时任务配置 { "tasks": [ScheduledTask] }
└── logs.jsonl       # 任务执行日志
```

### 5.2 文件格式规范

**queue.json 格式**:
```json
{
  "tasks": [
    {
      "id": "task-uuid",
      "prompt": "任务描述",
      "workspace": ".",
      "timeout": 600000,
      "auto_approve": false,
      "allowed_tools": ["Read", "created_at": "Write"],
      "2026-03-04T10:00:00.000Z",
      "source": "manual",
      "status": "pending"
    }
  ]
}
```

**scheduled.json 格式**:
```json
{
  "tasks": [
    {
      "id": "scheduled-uuid",
      "name": "每日报告",
      "cron": "0 9 * * *",
      "prompt": "生成每日开发报告",
      "workspace": ".",
      "timeout": 1800000,
      "auto_approve": true,
      "enabled": true,
      "created_at": "2026-03-04T10:00:00.000Z",
      "updated_at": "2026-03-04T10:00:00.000Z",
      "last_run": null,
      "next_run": "2026-03-05T09:00:00.000Z",
      "run_count": 0,
      "success_count": 0,
      "failed_count": 0
    }
  ]
}
```

**logs.jsonl 格式**:
```jsonl
{"task_id": "task-uuid", "timestamp": "2026-03-04T10:00:00.000Z", "level": "INFO", "message": "任务开始执行"}
{"task_id": "task-uuid", "timestamp": "2026-03-04T10:00:05.000Z", "level": "INFO", "message": "使用工具: Read", "context": {"tool": "Read", "file_path": "/test.py"}}
```

**注意**：
- `next_run` 可以为 `null`，表示未计算下次执行时间
- 当 `next_run` 为 `null` 时，调度器不会触发该任务
- 首次创建或禁用后重新启用时，应计算并设置 `next_run`
