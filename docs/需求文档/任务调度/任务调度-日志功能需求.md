# 任务调度 - 日志功能需求

## 1. 概述

### 1.1 目的
本文档专门定义任务调度模块中日志功能的详细需求，包括日志展示、实时监控、历史查看等核心能力。

### 1.2 范围
- 独立日志标签页设计
- 实时日志滚动显示
- 历史日志分页加载
- 正常输出与错误输出分离
- 日志交互功能（复制、搜索、导出）

### 1.3 背景
当前任务调度系统将日志显示在弹出对话框中，用户体验不佳。用户需要在独立的界面中查看完整的执行日志，特别是实时监控长时间运行的任务。

## 2. 功能需求

### 2.1 独立日志主标签页
- **LF-001**: 在任务调度主界面添加"正常日志"主标签页，与其他标签页（队列、定时、运行、完成、失败）并列
- **LF-002**: 在任务调度主界面添加"错误日志"主标签页，与其他标签页（队列、定时、运行、完成、失败）并列
- **LF-003**: 正常日志标签页显示所有任务的标准输出流（stdout）和成功执行日志
- **LF-004**: 错误日志标签页显示所有任务的错误输出流（stderr）和失败执行日志
- **LF-005**: 切换到任一日志标签页时自动聚焦到日志显示区域

### 2.2 实时日志显示
- **LF-006**: 任务执行过程中，正常日志实时显示在正常日志标签页
- **LF-007**: 任务执行过程中，错误日志实时显示在错误日志标签页
- **LF-008**: 系统自动滚动到底部，跟随最新日志行
- **LF-009**: 提供"暂停滚动"按钮，停止自动滚动以便查看特定日志段
- **LF-010**: 提供"继续滚动"按钮，恢复自动滚动功能

### 2.3 历史日志显示
- **LF-008**: 任务完成后，默认显示最近100行日志
- **LF-009**: 支持向上滚动加载更多历史日志（无限滚动）
- **LF-010**: 日志区域显示总行数统计

### 2.4 日志样式与分类
- **LF-011**: 正常输出使用黑色文字，白色背景
- **LF-012**: 错误输出使用红色文字，浅红色背景（#fee2e2）或红色边框
- **LF-013**: 每行日志前显示时间戳（HH:mm:ss格式）
- **LF-014**: 支持代码语法高亮

### 2.5 交互功能
- **LF-015**: 提供"复制全部"按钮，复制整个日志内容
- **LF-016**: 支持选中文本后右键复制
- **LF-017**: 提供搜索功能，在日志中查找关键词
- **LF-018**: 提供"清空日志"按钮，清空当前显示区域

### 2.6 数据映射
- **LF-019**: 前端从`task.result.message`字段获取完整执行日志
- **LF-020**: 禁止使用不存在的字段（如`task.output`）
- **LF-021**: 确保前后端日志字段一致性

## 3. 非功能需求

### 3.1 性能需求
- **LNFR-001**: 日志实时显示延迟 ≤ 500ms
- **LNFR-002**: 日志区域支持至少10,000行日志的流畅显示
- **LNFR-003**: 搜索功能响应时间 ≤ 200ms

### 3.2 可用性需求
- **LNFR-004**: 日志区域支持鼠标滚轮和键盘导航
- **LNFR-005**: 自动滚动功能默认开启，可手动控制
- **LNFR-006**: 错误日志必须明显区分于正常日志

### 3.3 可靠性需求
- **LNFR-007**: 日志数据丢失率 = 0%
- **LNFR-008**: 系统崩溃后能恢复未保存的日志

## 4. 技术约束

### 4.1 前端约束
- **LTC-001**: 使用原生JavaScript实现，无外部框架依赖
- **LTC-002**: 日志区域使用`<pre>`标签保持格式
- **LTC-003**: 支持响应式设计，适配不同屏幕尺寸

### 4.2 后端约束
- **LTC-004**: 日志数据通过现有API接口提供（`/api/scheduler/tasks/{task_id}`）
- **LTC-005**: 不需要新增API接口，复用现有任务详情接口
