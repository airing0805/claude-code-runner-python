## 1. 需求概述

### 1.1 模块目标

调度器控制模块负责管理任务调度服务的启动、停止、刷新等操作，提供调度器的状态监控和任务统计功能。

### 1.2 设计原则

- 调度器的启停操作需要优雅处理
- 支持强制停止和优雅停止两种模式
- 状态变化需要实时反馈
- 敏感操作需要用户确认

## 2. 功能需求分析

### 2.1 FR-026: 启动任务调度器

#### 需求描述
用户可以启动任务调度器服务

#### 用户场景

| 场景 | 描述 |
|------|------|
| 场景1 | 系统启动后首次启动调度器 |
| 场景2 | 调度器停止后重新启动 |

#### 启动流程

```
1. 检查是否已运行
2. 初始化调度器组件
3. 加载定时任务配置
4. 启动轮询机制
5. 更新调度器状态为 RUNNING
6. 返回启动结果
```

#### 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 操作是否成功 |
| data | object | 调度器状态信息 |
| message | string | 操作消息 |

#### 业务规则

1. 如果调度器已在运行，返回错误
2. 启动时加载所有启用的定时任务
3. 立即开始执行队列中的任务
4. 启动后状态变为 RUNNING

#### 验收标准

- [ ] 可以成功启动调度器
- [ ] 已运行时返回错误
- [ ] 返回正确的状态信息

### 2.2 FR-027: 停止任务调度器

#### 需求描述
用户可以停止任务调度器服务，支持优雅停止和强制停止两种模式

#### 用户场景

| 场景 | 描述 |
|------|------|
| 场景1 | 维护期间停止调度器 |
| 场景2 | 紧急情况下快速停止 |

#### 停止模式

##### 优雅停止（Graceful Stop）
- 等待当前运行任务完成后停止调度器
- 不再启动新任务
- 队列中的任务保持 pending 状态

##### 强制停止（Force Stop）
- 立即停止调度器
- 当前运行任务继续执行完成（但不再受调度器管理）
- 不再启动新任务

#### 输入

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| force | boolean | 否 | 是否强制停止，默认false |

#### 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 操作是否成功 |
| data | object | 调度器状态信息 |
| message | string | 操作消息 |

#### 业务规则

1. 如果调度器未运行，返回错误
2. 优雅停止等待任务完成（可设置超时时间）
3. 强制停止立即返回
4. 停止后状态变为 STOPPED

#### 停止流程（优雅模式）

```
1. 接收停止请求
2. 设置停止标志
3. 等待当前任务完成（最多等待 N 分钟）
4. 停止轮询机制
5. 更新调度器状态为 STOPPED
6. 返回停止结果
```

#### 停止流程（强制模式）

```
1. 接收强制停止请求
2. 立即设置停止标志
3. 停止轮询机制
4. 当前运行任务继续执行（不受调度器控制）
5. 更新调度器状态为 STOPPED
6. 返回停止结果
```

#### 验收标准

- [ ] 可以优雅停止调度器
- [ ] 可以强制停止调度器
- [ ] 未运行时返回错误
- [ ] 队列任务不受影响

#### 验收标准

### 2.4 FR-028.1: 取消运行中的任务

#### 需求描述
用户可以取消运行中的任务

#### 用户场景

| 场景 | 描述 |
|------|------|
| 场景1 | 任务执行异常需要取消 |
| 场景2 | 用户改变主意停止任务 |

#### 输入

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task_id | string | 是 | 要取消的任务ID |

#### 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 操作是否成功 |
| message | string | 操作消息 |

#### 业务规则

1. 只能取消 RUNNING 状态的任务
2. 取消后任务状态变为 CANCELLED
3. 取消操作需要用户二次确认
4. 取消后的任务不触发重试机制
5. 取消后的任务移动到 failed.json 文件

#### 状态转换

```
RUNNING -> CANCELLED
```

#### 二次确认

- 取消操作是高风险操作
- 弹出确认对话框
- 明确告知取消后任务将无法恢复

#### 验收标准

- [ ] 可以取消运行中的任务
- [ ] 状态正确变为 CANCELLED
- [ ] 需要二次确认
- [ ] 不触发重试机制
- [ ] 任务正确移动到历史记录

### 2.5 FR-029: 调度器状态监控

#### 需求描述

#### 状态定义

| 状态值 | 说明 |
|--------|------|
| running | 调度器运行中 |
| stopped | 调度器已停止 |
| starting | 调度器启动中 |

#### 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | 调度器状态 |
| started_at | string | 启动时间 |
| queue_count | integer | 队列任务数 |
| running_count | integer | 运行任务数 |
| scheduled_count | integer | 定时任务数 |

#### 业务规则

1. 状态实时更新
2. 显示启动时间
3. 显示任务统计

#### 验收标准

- [ ] 显示正确的调度器状态
- [ ] 状态实时更新
- [ ] 显示启动时间

## 3. API 接口规范

### 3.1 获取调度器状态

```
GET /api/scheduler/status
```

**响应：**
```json
{
  "success": true,
  "data": {
    "status": "running",
    "started_at": "2026-03-07T10:00:00Z",
    "tasks": {
      "queue": 5,
      "running": 2,
      "completed_today": 10,
      "failed_today": 1
    },
    "scheduled": {
      "total": 8,
      "enabled": 5
    }
  },
  "message": ""
}
```

### 3.2 启动调度器

```
POST /api/scheduler/start
```

**响应：**
```json
{
  "success": true,
  "data": {
    "status": "running",
    "started_at": "2026-03-07T10:00:00Z"
  },
  "message": "调度器已启动"
}
```

### 3.3 停止调度器

```
POST /api/scheduler/stop
```

**参数：**
| 参数 | 类型 | 说明 |
|------|------|------|
| force | boolean | 是否强制停止，默认false |

**响应：**
```json
{
  "success": true,
  "data": {
    "status": "stopped"
  },
  "message": "调度器已停止"
}
```

### 3.4 取消任务

```
POST /api/scheduler/tasks/{task_id}/cancel
```

**响应：**
```json
{
  "success": true,
  "data": null,
  "message": "任务已取消"
}
```

### 3.5 刷新数据

```
POST /api/scheduler/refresh
```

**响应：**
```json
{
  "success": true,
  "data": {
    "queue_count": 5,
    "running_count": 2
  },
  "message": "数据已刷新"
}
```

## 4. 状态机设计

### 4.1 调度器状态

```
     +---------+
     | STOPPED |
     +---------+
          |
          | start()
          v
   +-------------+
   |   STARTING  |
   +-------------+
          |
          | (启动完成)
          v
    +----------+
    | RUNNING  |
    +----------+
          |
          | stop()
          v
   +-------------+
   |   STOPPING |
   +-------------+
          |
          | (停止完成)
          v
     +---------+
     | STOPPED |
     +---------+
```

### 4.2 状态转换规则

| 状态 | 可转换到 | 触发操作 |
|------|----------|----------|
| STOPPED | STARTING | start() |
| STARTING | RUNNING | 启动完成 |
| RUNNING | STOPPING | stop() |
| STOPPING | STOPPED | 停止完成 |

## 8. 安全控制

### 8.1 敏感操作确认

| 操作 | 需要确认 | 确认方式 |
|------|----------|----------|
| 启动调度器 | 否 | - |
| 优雅停止 | 是 | 弹窗确认 |
| 强制停止 | 是 | 弹窗确认 |
| 取消任务 | 是 | 弹窗确认 |
| 清空队列 | 是 | 弹窗确认 |
| 删除定时任务 | 否 | - |

### 8.2 确认对话框内容

**停止调度器确认：**
```
确认要停止调度器吗？

- 优雅停止：等待当前任务完成后停止
- 强制停止：立即停止，新的任务将不会执行

[取消] [优雅停止] [强制停止]
```

**取消任务确认：**
```
确认要取消此任务吗？

任务名称：xxx
当前状态：运行中

取消后任务将无法恢复。

[取消] [确认取消]
```

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 强制停止后任务状态不一致 | 任务状态异常 | 重新启动时检查并修复 |
| 优雅停止超时 | 长时间等待 | 设置最大等待时间 |
| 并发操作 | 状态冲突 | 使用文件锁 |
