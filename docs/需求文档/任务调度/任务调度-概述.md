# 任务调度-概述

## 概述

任务调度功能提供 24/7 自动化任务执行能力，支持立即执行任务队列和基于 Cron 表达式的定时任务调度，适用于自动化代码检查、文档更新、测试运行等重复性任务。

> **来源**: 基于 `claude-code-24h-integration` 项目功能迁移
> **版本**: v1.6 (2026-03-05 更新)
> **架构说明**: Python FastAPI 架构（非 Node.js/PM2）

---

## 1. 功能概述

### 1.1 系统架构

**调度器 (Scheduler)**
- 定时轮询 (默认 10 秒)
- 检查定时任务是否到期
- 从队列中获取任务执行
- 管理任务状态

**任务存储 (JSON 文件)**
- queue.json         # 待执行任务队列
- scheduled.json     # 定时任务配置
- running.json       # 当前运行任务
- completed.json     # 已完成任务历史
- failed.json        # 失败任务记录
- cancelled.json     # 已取消任务历史
- logs.jsonl         # 任务执行日志

**任务执行器 (Executor)**
- 调用 ClaudeCodeClient 执行任务
- 支持超时控制
- 自动重试 (最多 2 次)

## 2. Web界面操作

### 2.1 任务调度Web界面功能

**任务队列页面**
- 任务列表显示（包含状态、描述、创建时间等）
- 任务筛选和搜索功能
- 任务排序（按创建时间、优先级等）
- 任务操作（删除、重试、查看详情）

**定时任务页面**
- 定时任务列表（包含下次执行时间、Cron表达式等）
- Cron表达式可视化编辑器
- 定时任务状态切换（启用/禁用）
- 立即执行定时任务功能

**任务执行监控页面**
- 运行中任务实时监控
- 任务执行进度显示
- 任务日志查看
- 任务取消功能

**调度器状态面板**
- 调度器运行状态指示
- 任务统计信息（待执行、运行中、已完成、失败）
- 系统资源使用情况
- 调度器控制按钮（启动/停止）

### 2.2 Web界面交互流程

**创建任务流程**
1. 用户点击"添加任务"按钮
2. 弹出任务创建对话框
3. 用户填写任务描述、工作目录、超时时间等参数
4. 用户选择自动批准和工具限制选项
5. 提交后任务加入队列并显示在任务列表中

**创建定时任务流程**
1. 用户切换到"定时任务"标签页
2. 点击"添加定时任务"按钮
3. 填写任务名称、描述、Cron表达式等
4. 配置执行参数（工作目录、超时、自动批准等）
5. 保存后定时任务显示在列表中，显示下次执行时间

**任务执行监控流程**
1. 调度器自动从队列获取任务执行
2. Web界面实时更新任务状态为"运行中"
3. 用户可查看任务执行日志和进度
4. 任务完成后状态更新为"已完成"或"失败"
5. 用户可查看任务结果详情

### 2.3 Web界面安全控制

- **权限验证**: 所有操作需要用户认证
- **输入验证**: 任务参数进行前端和后端双重验证
- **工作目录限制**: 禁止访问未授权的目录
- **敏感操作确认**: 删除、取消等操作需要二次确认
- **错误处理**: 友好的错误提示和恢复机制
