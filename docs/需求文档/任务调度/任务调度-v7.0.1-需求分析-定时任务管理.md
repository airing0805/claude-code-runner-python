## 1. 需求概述

### 1.1 模块目标

定时任务管理模块允许用户创建基于Cron表达式的自动化任务调度，支持定时触发、立即执行、启用/禁用等功能。

### 1.2 设计原则

- 定时任务是独立的任务定义，每次触发都会向队列添加新任务
- 定时任务与队列任务是分离的，定时任务管理界面和队列管理界面独立
- 支持灵活的Cron表达式配置
- 定时任务的状态（enabled/disabled）独立于任务定义

## 2. 功能需求分析

### 2.1 FR-011: 创建基于Cron表达式的定时任务

#### 需求描述
用户可以创建基于Cron表达式的定时任务

#### 用户场景

| 场景 | 描述 |
|------|------|
| 场景1 | 每天凌晨2点执行代码审查 |
| 场景2 | 每周一自动运行测试套件 |
| 场景3 | 每小时同步外部数据 |

#### 输入

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 定时任务名称 |
| prompt | string | 是 | 任务描述/提示词 |
| cron | string | 是 | Cron表达式 |
| workspace | string | 否 | 工作目录，默认为当前目录 |
| timeout | integer | 否 | 超时时间（秒），默认600秒 |
| auto_approve | boolean | 否 | 是否启用自动批准，默认false |
| allowed_tools | array[string] | 否 | 允许使用的工具列表 |

#### 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 操作是否成功 |
| data | object | 创建的定时任务信息 |
| message | string | 操作消息 |

#### 业务规则

1. name 不能为空且不能重复
2. prompt 不能为空
3. cron 表达式必须有效
4. 自动计算并保存下次执行时间

#### 验收标准

- [ ] 可以创建定时任务
- [ ] 名称不能为空
- [ ] 名称不能重复
- [ ] Cron表达式必须有效
- [ ] 返回任务信息

## 3. Cron表达式规范

### 3.1 表达式格式

标准Cron格式：`minute hour day month weekday`

| 字段 | 允许值 | 特殊字符 |
|------|--------|----------|
| minute | 0-59 | * , - / |
| hour | 0-23 | * , - / |
| day | 1-31 | * , - / ? L W |
| month | 1-12 | * , - / |
| weekday | 0-6 | * , - / ? L # |

### 3.2 特殊字符说明

| 字符 | 说明 | 示例 |
|------|------|------|
| * | 任意值 | * * * * * 每分钟 |
| , | 值列表 | 0,30 * * * * 每小时第0和30分钟 |
| - | 范围 | 0 9-17 * * * 每天9点到17点 |
| / | 步长 | */5 * * * * 每5分钟 |
| ? | 任意值（日期和星期互斥） | 0 0 1 * ? 每月1号 |
| L | 最后 | 0 0 L * * 每月最后一天 |
| W | 最近工作日 | 0 0 15W * * 每月15号最近的工作日 |

### 3.3 常用示例

| 表达式 | 说明 |
|--------|------|
| @yearly (0 0 1 1 *) | 每年1月1日午夜 |
| @monthly (0 0 1 * *) | 每月1日午夜 |
| @weekly (0 0 * * 0) | 每周日午夜 |
| @daily (0 0 * * *) | 每天午夜 |
| @hourly (0 * * * *) | 每小时 |
| */5 * * * * | 每5分钟 |
| 0 2 * * * | 每天凌晨2点 |
| 0 9 * * 1-5 | 工作日上午9点 |
| 0 0 * * 0,6 | 周末午夜 |

## 4. API 接口规范

### 4.1 创建定时任务

```
POST /api/scheduler/scheduled-tasks
```

**请求体：**
```json
{
  "name": "每日代码审查",
  "prompt": "请审查所有新增代码",
  "cron": "0 2 * * *",
  "workspace": "/project",
  "timeout": 600,
  "auto_approve": true,
  "allowed_tools": ["Read", "Glob", "Grep"]
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "id": "uuid-xxxx-xxxx",
    "name": "每日代码审查",
    "prompt": "请审查所有新增代码",
    "cron": "0 2 * * *",
    "workspace": "/project",
    "timeout": 600,
    "enabled": true,
    "next_run": "2026-03-08T02:00:00Z",
    "created_at": "2026-03-07T10:00:00Z"
  },
  "message": "定时任务已创建"
}
```

### 4.2 获取定时任务列表

```
GET /api/scheduler/scheduled-tasks
```

**响应：**
```json
{
  "success": true,
  "data": {
    "tasks": [...],
    "total": 5
  },
  "message": ""
}
```

### 4.3 更新定时任务

```
PATCH /api/scheduler/scheduled-tasks/{task_id}
```

**请求体：**
```json
{
  "name": "每日代码审查（更新）",
  "cron": "0 3 * * *"
}
```

### 4.4 删除定时任务

```
DELETE /api/scheduler/scheduled-tasks/{task_id}
```

### 4.5 启用/禁用定时任务

```
POST /api/scheduler/scheduled-tasks/{task_id}/toggle
```

### 4.6 立即执行定时任务

```
POST /api/scheduler/scheduled-tasks/{task_id}/run
```
