## 1. 概念定义

### 1.1 多轮会话

**定义**: 指用户和 AI 在**单个会话**中的多次消息交换。

**核心特征**:
- 都在同一个 `session_id` 下
- 消息通过 `parentUuid` 链接成链式结构
- 上下文会累积传递，每次回复都能看到之前的对话历史
- 用户**主动**发起多次输入

**示例场景**:
```
用户: "分析项目结构"
  → AI: [分析结果]
用户: "那 utils 包含什么?"  ← 同一会话内的第 2 轮
  → AI: [utils 包含...]
用户: "帮我优化一下"        ← 同一会话内的第 3 轮
  → AI: [优化建议]
```

---

### 1.2 继续会话

**定义**: 指从**历史记录**中恢复一个**之前已经结束**的会话，在其基础上继续对话。

**核心特征**:
- 在**会话之间**跳转，不是会话内部轮次
- 通过 `resume` 参数或 `continue_conversation` 参数触发
- 恢复后会加载历史消息作为上下文
- 工作目录通常锁定为原会话目录

**两种恢复方式**:

| 方式 | 参数 | 数据来源 | 触发方式 |
|------|------|---------|---------|
| **continue_conversation** | 无，复选框勾选 | 最近一次会话 | 勾选"继续会话"复选框 |
| **resume** | 指定会话 ID | 用户选择的指定会话 | 从历史记录选择会话 |

**示例场景**:
```
会话 A (session_id=abc123) - 已结束
├── 用户: "分析项目结构"
└── AI: [分析结果]

用户关闭页面...

数天后，用户回来，点击"继续会话"
  ↓
恢复到会话 A，历史消息重新显示
  ↓
用户: "帮我优化一下"  ← 新的消息轮次
  ↓
AI: [优化建议...]
```

---

### 1.3 Agent Loop (多轮对话)

**定义**: 指 AI 代理在**执行单个用户请求的过程中**，需要多次交互来完成任务的机制。

**核心特征**:
- **阻塞机制**: 碰到问题后停止，等待用户答案
- **状态管理**: PENDING → SHOWING → ANSWERED → PROCESSING → COMPLETED/ERROR/TIMEOUT
- **消息链**: `tool_use` → `tool_result` → 下一步...
- AI **主动**发起多次提问，用户被动回答

**典型场景** (AskUserQuestion):
```
用户: "帮我配置 OAuth"  ← 单次请求
  ↓
AI: 调用 AskUserQuestion("选择 OAuth 方式?")  ← 第 1 次交互
  ↓ [阻塞等待]
用户: 选择 "oauth2"  ← 答案提交
  ↓
AI: 继续执行下一步...
  ↓
AI: 调用 AskUserQuestion("选择提供商?")  ← 第 2 次交互
  ↓ [再次阻塞]
用户: 选择 "google"  ← 答案提交
  ↓
AI: 完成配置
```

**SDK 支持状态**: 当前版本 (claude-agent-sdk v0.0.25) **不支持** Agent Loop 阻塞机制。

---

### 1.4 tool_use 消息

**定义**: Claude SDK 返回的一种**消息类型**，表示 AI 决定调用某个工具。

**核心特征**:
- 是**消息类型**的一种，不是独立功能
- 包含工具名称 (`tool_name`) 和工具参数 (`tool_input`)
- 是 Agent Loop 中问答交互的**载体**
- 通过 `tool_use_id` 与 `tool_result` 消息配对

**消息类型枚举**:
```python
class MessageType(Enum):
    TEXT = "text"           # 文本输出
    TOOL_USE = "tool_use"   # 工具调用 ← 本概念
    TOOL_RESULT = "tool_result"  # 工具结果
    THINKING = "thinking"   # 思考过程
    ERROR = "error"         # 错误
    COMPLETE = "complete"   # 任务完成
```

## 2. 区别对比表

| 维度 | 多轮会话 | 继续会话 | Agent Loop | tool_use |
|------|----------|----------|------------|----------|
| **概念类型** | 交互模式 | 会话恢复机制 | 执行机制 | 消息类型 |
| **范围** | 单会话内 | 会话之间 | 单请求内 | 消息级别 |
| **发起方** | 用户主动 | 用户主动 | AI 主动 | AI 决定 |
| **触发方式** | 用户多次输入 | resume / continue_conversation | AskUserQuestion 等工具调用 | AI 工具调用决策 |
| **会话 ID** | 同一 `session_id` | 切换 `session_id` | 同一 `session_id` | 同一 `session_id` |
| **消息类型** | user/assistant | - | tool_use/tool_result | tool_use |
| **阻塞机制** | 无 | 无 | 有 | 无 |
| **状态管理** | - | - | PENDING → ... | - |

## 3. 常见混淆点

### 3.1 "多轮会话" vs "Agent Loop"

**错误理解**: 认为两者都是"多次对话"，没有区别。

**正确理解**:
- **多轮会话**: 用户**主动**多次输入，每次输入都是新的对话轮次
- **Agent Loop**: AI **主动**多次提问，等待用户答案后继续

**记忆口诀**:
> 用户多次问 = 多轮会话
> AI 多次问 = Agent Loop

---

### 3.2 "tool_use" vs "AskUserQuestion"

**错误理解**: 认为 `tool_use` 就是问答交互。

**正确理解**:
- `tool_use`: **消息类型**，所有工具调用都产生此消息
- `AskUserQuestion`: **特定工具**，用于问答交互

**类比**:
> tool_use ≈ 快递包裹（任何物品）
> AskUserQuestion ≈ 询问单（需要签收的包裹）

---

### 3.3 "继续会话" vs "会话内的多轮对话"

**错误理解**: 认为"继续会话"就是在会话内再多问一轮。

**正确理解**:
- **继续会话**: 恢复**不同会话** (resume 参数)
- **会话内多轮**: **同一会话**内多次消息交换

**图示**:
```
会话 A (session_id=abc123)
├── 用户: "分析项目"
└── AI: [结果]
  ↓ (会话结束)

    ──[继续会话]──→

会话 B (session_id=abc123)  ← 恢复会话 A
├── 用户: "帮我优化一下"
└── AI: [优化建议]
```

---

### 3.4 消息链的误解

**错误理解**: 认为 `tool_use` 之后的 `tool_result` 就是用户主动输入。

**正确理解**:
- `tool_result` 可以是**系统自动生成**（如 Read 工具的结果）
- `tool_result` 也可以是**用户提交的答案**（如 AskUserQuestion 的答案）

需要根据 `tool_use.tool_name` 判断来源：
- `AskUserQuestion` → `tool_result` 是用户答案
- `Read/Write/Edit` → `tool_result` 是系统结果

## 记忆口诀

| 概念 | 记忆口诀 |
|------|----------|
| 多轮会话 | 用户主动，同一会话 |
| 继续会话 | 恢复历史，切换会话 |
| Agent Loop | AI 主动提问，阻塞等待 |
| tool_use | 消息类型，不是功能 |
