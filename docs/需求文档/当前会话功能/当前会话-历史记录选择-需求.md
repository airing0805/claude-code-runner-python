## 1. 功能描述

**历史记录下拉选择**用于快速选择历史会话，继续之前的对话上下文。

> ⚠️ **重要**: 历史记录下拉选择后的业务逻辑，与"历史记录功能→项目列表→历史记录列表→继续此会话"的逻辑**完全相同**。

---

## 2. 界面设计

```
历史会话: [ 会话1 - "分析代码结构"        ▼ ]
            ├── 📋 会话1 - "分析代码结构" (2小时前)
            ├── 📋 会话2 - "修复Bug" (昨天)
            ├── 📋 会话3 - "添加功能" (3天前)
            ├── 📋 会话4 - "代码审查" (上周)
            └── 📋 会话5 - "性能优化" (2周前)
```

**组件说明**:
- 下拉框显示当前工作空间下的历史会话列表
- 每条显示会话标题、预览和时间
- 按创建时间倒序排列

---

## 3. 交互流程

```
1. 用户点击历史会话下拉框
   ↓
2. 显示该工作空间的历史会话列表（按时间倒序）
   ↓
3. 用户选择具体会话
   ↓
4. 复用"历史记录功能→继续此会话"的业务逻辑：
   - 恢复该会话的工作空间
   - 恢复会话消息历史
   - 更新会话ID为选中会话
```

**用户反馈**:
| 时刻 | 反馈 |
|------|------|
| 会话加载中 | 下拉框内显示加载指示器 |
| 加载成功 | 界面显示恢复的会话上下文 |
| 加载失败 | 红色提示："会话加载失败" |

---

## 4. 业务逻辑

> ⚠️ **重要**: 历史记录下拉选择后的业务逻辑，与"历史记录功能→项目列表→历史记录列表→继续此会话"的逻辑**完全相同**。

详细的数据回填规则、消息解析规则、会话状态限制等，请参考 [当前会话-继续会话-需求.md](当前会话-继续会话-需求.md)。

### 4.1 数据回填规则（简化版）

| 数据项 | 来源 | 回填行为 |
|--------|------|----------|
| 会话ID (resume) | 选择的会话文件 | 填充到会话ID输入框，设为只读 |
| 工作目录 (working_dir) | 会话创建时的项目路径 | 填充并锁定，不可修改 |
| 历史消息 | 完整会话消息历史 | 显示在输出区域 |

### 4.2 状态限制规则

恢复历史会话后，以下操作被禁用或限制：

| 操作 | 限制说明 |
|------|----------|
| 工作目录 | 锁定，不可修改 |
| 继续会话复选框 | 禁用（因为已通过 resume 指定会话） |
| 工具权限 | 可调整 |
| 新建会话按钮 | 正常可用 |

### 4.3 与继续会话功能的关系

| 功能 | 说明 |
|------|------|
| 历史记录下拉选择 | v12 新增的快捷入口，与继续会话复选框功能合并 |
| 继续会话复选框 | v11 旧版功能，v12 由下拉选择替代 |
| 继续此会话 | 历史记录页面中的操作，功能相同 |

---

## 5. 显示规则

| 规则 | 说明 |
|------|------|
| 列表数量 | 最多显示10个会话 |
| 排序方式 | 按创建时间倒序 |
| 预览文本 | 显示首条用户消息的前30个字符 |
| 时间显示 | 相对时间（如"2小时前"、"昨天"） |
| 空状态 | 无历史时显示"暂无历史会话" |

---

## 6. 验收标准

- [ ] 历史记录下拉框显示该工作空间的历史会话列表
- [ ] 选择具体会话后复用"历史记录功能-继续此会话"逻辑恢复上下文
- [ ] 列表按时间倒序排列
- [ ] 选中的会话信息正确显示在界面
- [ ] 与历史记录功能的业务逻辑完全一致
