# 当前会话 - 继续会话需求

## 1. 概述

本文档定义"继续会话"功能的详细需求，包括延续最近会话（continue_conversation）和恢复指定会话（resume）两种方式。

## 2. 功能状态

| 功能 | 编号 | 优先级 | 状态 | 说明 |
|------|------|--------|------|------|
| 会话恢复 | FR-003 | P0 | ✅ 已实现 | resume 参数支持 |
| 继续会话 | FR-003-A | P0 | ⚠️ 部分实现 | 前端逻辑需完善 |
| 最近会话 ID 获取 | FR-003-B | P1 | 📋 待实现 | 需前端实现 |

## 3. 两种延续方式

### 3.1 continue_conversation（延续最近会话）

| 属性 | 说明 |
|------|------|
| 触发方式 | 勾选"继续会话"复选框 |
| 数据来源 | 最近一次会话 |
| 工作目录 | 可修改 |
| 消息历史 | 自动续接最近会话 |
| 会话ID显示 | 可选显示 |

### 3.2 resume（恢复指定会话）

| 属性 | 说明 |
|------|------|
| 触发方式 | 从历史记录选择会话 |
| 数据来源 | 用户选择的指定会话 |
| 工作目录 | 锁定为原会话目录（来自 cwd 字段） |
| 消息历史 | 显示所选会话历史 |
| 会话ID显示 | 必须显示 |

| 会话ID | 文本输入框(只读) | 显示当前会话ID，可用于恢复会话 |
| 新会话按钮 | 按钮 | 创建全新的会话 |

## 5. 交互规则

### 5.1 会话管理操作

| 操作 | 触发方式 | 响应 |
|------|----------|------|
| 新建会话 | 点击"新会话"按钮 | 清空输入框，会话ID设为"新会话"，取消"继续会话"勾选 |
| 继续会话 | 勾选"继续会话" | 自动使用最近会话ID填充 resume 字段 |
| 恢复历史会话 | 从历史记录点击会话 | 在标签页打开历史会话，可查看并继续执行 |

### 5.2 历史会话交互限制

恢复历史会话后，以下操作被禁用或限制：

| 操作 | 限制说明 |
|------|----------|
| 继续会话复选框 | 禁用（因为已通过 resume 指定会话） |
| 工作目录 | 锁定，不可修改 |
| 工具权限 | 可调整 |
| 新建会话按钮 | 正常可用 |

## 6. 数据回填规则

当用户从历史记录继续会话时，系统自动回填以下数据：

| 数据项 | 来源 | 回填行为 |
|--------|------|----------|
| 会话ID (resume) | 选择的会话文件 `{sessionId}.jsonl` | 填充到会话ID输入框，设为只读 |
| 工作目录 (working_dir) | 会话创建时的项目路径（首条 user 消息的 `cwd` 字段） | 填充并锁定，不可修改 |
| 历史消息 | 完整会话消息历史（解析 jsonl 文件中所有 user/assistant 消息） | 显示在输出区域 |
| 会话标题 | 首条用户消息内容（首条 user 消息的 `message.content[0].text`） | 显示在标签页标题 |

## 7. 会话数据格式

### 7.1 数据来源

会话延续功能依赖于 Claude Code 的会话文件格式：

- **存储位置**: `~/.claude/projects/{encoded-path}/{sessionId}.jsonl`
- **文件格式**: 每行 JSON 对应一条消息，按时间顺序存储
- **消息链**: 通过 `parentUuid` 链式关系重新构建对话历史

### 7.2 数据结构

```typescript
Session {
  id: string (UUID)           // 会话唯一标识（对应会话文件名的 sessionId）
  title: string               // 会话标题（首条用户消息摘要）
  project_path: string        // 关联的项目路径（对应会话消息的 cwd 字段）
  timestamp: string           // 创建时间 ISO 格式（对应首条 user 消息的 timestamp）
  message_count: number       // 消息总数
  tools_used: string[]       // 使用的工具列表（从 assistant 消息的 tool_use 提取）
  files_changed: string[]    // 变更的文件列表（从 file-history-snapshot 跟踪）
}

Message {
  role: "user" | "assistant"
  content: string | ContentBlock[]  // 消息内容
  timestamp: string           // 发送时间
  uuid: string               // 消息唯一标识（用于构建消息链）
  parentUuid: string | null  // 父消息 UUID（用于会话消息链关系）
}
```

## 8. 消息解析规则

### 8.1 消息链关系

- 每条消息通过 `uuid` 和 `parentUuid` 构成链式结构
- `user` 消息的 `parentUuid` 为 `null` 表示新会话开始
- `permissionMode` 字段存在时表示新任务发起（最可靠的会话边界判断依据）

### 8.2 多轮对话判断

| 条件 | 含义 |
|------|------|
| `permissionMode` 字段存在 | 新会话开始 |
| `parentUuid` 为 `null` | 首条用户消息 |
| 内容包含 `tool_result` | 继续当前对话（工具结果返回） |

### 8.3 用户输入类型区分

| 类型 | 识别特征 | 业务含义 |
|------|---------|----------|
| ide_selection | 包含 `<ide_selection>` 标签 | 用户选中代码后提问 |
| ide_opened_file | 包含 `<ide_opened_file>` 标签 | 用户打开文件后操作 |
| tool_result | content 包含 `tool_result` 类型 | 工具执行结果返回 |
| 普通文本 | content 只包含 `text` 类型 | 直接输入的任务描述 |

### 8.4 消息 content 字段类型

| 消息类型 | content 字段类型 | 说明 |
|----------|-----------------|------|
| 第一条用户消息 | `string` | 简单文本消息，直接作为 text 类型 |
| 后续用户消息 | `array` | 复杂消息格式，包含多种内容块 |
| Assistant 消息 | `array` | 包含 thinking、text、tool_use 等内容块 |

### 8.5 解析逻辑

1. 如果 content 是字符串类型，直接转换为 `{"type": "text", "text": "..."}`
2. 如果 content 是数组类型，遍历内容块数组解析各类型：
   - `text` → 文本内容
   - `thinking` → 思考过程
   - `tool_use` → 工具调用（提取 name、input、id）
   - `tool_result` → 工具结果（通过 tool_use_id 关联 tool_use 的名称）

### 8.6 过滤规则

- 解析会话标题时，过滤包含 `<ide_selection>` 或 `<ide_opened_file>` 标签的内容块
- 仅返回有实际内容的消息（空 content 或空字符串不返回）

## 9. 验证规则

## 12. 继续会话实现需求（评审改进）

> 本节记录 PQ-004 问题的改进需求，详见 [当前会话-需求.md](当前会话-需求.md) 评审问题章节。

### 12.1 最近会话 ID 获取机制

**问题描述**：
- 前端没有实现"最近会话 ID"的获取和存储机制
- "继续会话"复选框勾选后无法正确获取最近会话

**实现要求**：

| 需求项 | 说明 | 优先级 |
|--------|------|--------|
| 最近会话获取 API | 调用 `/api/sessions?limit=1&sort_by=timestamp&order=desc` 获取最近会话 | P1 |
| 会话 ID 缓存 | 前端缓存最近会话 ID，避免重复请求 | P2 |
| 复选框状态同步 | 勾选时自动填充 resume 字段，取消时清空 | P1 |
| 空会话处理 | 无历史会话时禁用复选框并提示 | P2 |

### 12.2 复选框逻辑规范

```
继续会话复选框行为：

1. 初始状态
   - 检查是否有历史会话
   - 无历史会话：禁用复选框，显示提示 "无历史会话"
   - 有历史会话：启用复选框

2. 勾选时
   - 获取当前工作目录的最近会话 ID
   - 自动填充到 resume 参数
   - 会话 ID 输入框显示最近会话 ID（只读）

3. 取消勾选时
   - 清空 resume 参数
   - 会话 ID 输入框恢复可编辑

4. 工作目录变更时
   - 如复选框已勾选，重新获取新目录的最近会话
   - 如无会话，取消勾选并提示
```

### 12.3 前端实现示例

```javascript
// 继续会话复选框逻辑
class ContinueSessionHandler {
  constructor(runner) {
    this.runner = runner;
    this.latestSessionId = null;
  }

  // 初始化：获取最近会话
  async init() {
    await this.fetchLatestSession();
    this.updateCheckboxState();
  }

  // 获取最近会话 ID
  async fetchLatestSession() {
    const workingDir = this.runner.workingDirSelect.value;
    try {
      const response = await fetch(
        `/api/sessions?working_dir=${encodeURIComponent(workingDir)}&limit=1&sort_by=timestamp&order=desc`
      );
      const data = await response.json();
      this.latestSessionId = data.sessions?.[0]?.id || null;
    } catch (error) {
      console.error('获取最近会话失败:', error);
      this.latestSessionId = null;
    }
  }

  // 更新复选框状态
  updateCheckboxState() {
    const checkbox = this.runner.continueConversationCheckbox;
    if (!this.latestSessionId) {
      checkbox.disabled = true;
      checkbox.title = '无历史会话';
    } else {
      checkbox.disabled = false;
      checkbox.title = `继续会话: ${this.latestSessionId.slice(0, 8)}...`;
    }
  }

  // 处理勾选事件
  async handleCheck(checked) {
    if (checked) {
      if (!this.latestSessionId) {
        await this.fetchLatestSession();
      }
      if (this.latestSessionId) {
        this.runner.sessionIdInput.value = this.latestSessionId;
        this.runner.sessionIdInput.readOnly = true;
      } else {
        // 无会话，取消勾选
        this.runner.continueConversationCheckbox.checked = false;
        this.runner.showNotification('无历史会话可继续');
      }
    } else {
      this.runner.sessionIdInput.value = '';
      this.runner.sessionIdInput.readOnly = false;
    }
  }
}
```
