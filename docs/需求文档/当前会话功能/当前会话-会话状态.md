## 1. 概述

### 1.1 定义

会话状态管理是指从用户发起任务开始到会话结束的完整生命周期管理，包括状态转换、资源分配、超时控制和并发限制。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| **状态一致性** | 确保会话状态在所有组件间保持一致 |
| **资源管理** | 及时释放不再使用的会话资源 |
| **超时保护** | 防止会话无限期占用资源 |
| **并发控制** | 限制同时活跃的会话数量 |
| **用户体验** | 提供清晰的会话状态反馈 |

## 3. 会话模型

### 3.1 会话数据结构

```python
@dataclass
class SessionInfo:
    session_id: str              # 会话唯一标识
    client: ClaudeCodeClient    # Claude 客户端实例
    is_waiting: bool            # 是否在等待用户答案
    pending_question_id: str | None  # 待处理的问题 ID
    created_at: float           # 创建时间戳
    last_activity: float       # 最后活动时间
    metadata: dict             # 附加元数据
```

### 3.2 会话状态定义

| 状态 | 说明 | 可操作 |
|------|------|--------|
| `idle` | 会话空闲，无活动 | ❌ 无需管理 |
| `executing` | 任务执行中 | ✅ 可接收答案 |
| `waiting` | 等待用户答案 | ⚠️ 只允许提交答案 |
| `completed` | 任务已完成 | ❌ 无需管理 |
| `error` | 会话错误 | ⚠️ 需要清理 |

### 3.3 状态优先级

```
executing > waiting > completed > error > idle
```

## 4. 生命周期管理

### 4.1 生命周期阶段

```
┌─────────────────────────────────────────────────┐
│              会话生命周期                           │
└─────────────────────────────────────────────────┘
                        │
                        ▼
    创建 (created)
                        │
                        ▼
    初始化 (initializing)
   ┌─────────────────────┴─────────────────────┐
   │                                     │
   ▼                                     ▼
执行中 (executing)  ←─→  等待 (waiting)
   │         │         │
   │         │         │
   │         │         ▼
   │         │    答案提交
   │         │         │
   │         │         ▼
   │         │    继续执行
   │         │         │
   │         │         ▼
   │         │    完成/错误
   │         │         │
   └─────────────┴─────────────────────┘
                        │
                        ▼
    终止 (terminated)
                        │
                        ▼
    清理 (cleaned)
```

### 4.2 阶段转换条件

| 从状态 | 到状态 | 触发条件 |
|--------|--------|---------|
| 创建 → 初始化 | 会话 ID 生成完成 | 立即 |
| 初始化 → 执行中 | 客户端创建完成 | 任务启动 |
| 执行中 → 等待 | AI 调用 AskUserQuestion tool_use 消息 |
| 等待 → 执行中 | 用户提交答案 | /api/task/answer 调用 |
| 执行中 → 完成 | 任务完成 | complete 消息 |
| 执行中 → 错误 | 执行出错 | error 消息 |
| 完成/错误 → 终止 | 用户操作或超时 | 新会话/页面关闭 |
| 终止 → 清理 | 清理条件满足 | 超时/资源回收 |

## 6. 超时处理

### 6.1 超时类型

| 超时类型 | 时间限制 | 处理方式 |
|---------|---------|---------|
| **单次答案超时** | 300 秒 (5 分钟) | 超时后自动选择默认选项或继续 |
| **会话总超时** | 3600 秒 (1 小时) | 自动结束会话并清理资源 |
| **活动超时** | 1800 秒 (30 分钟) | 最后活动后清理 |

### 6.2 超时处理流程

```
会话创建
  ↓
设置超时时间戳 (created_at + 3600 秒)
  ↓
定期检查超时
  ↓
超时检测 → 会话状态: error → 清理资源
  ↓
发送超时通知 → 前端显示"会话已超时"
```

## 7. 并发控制

### 7.1 并发限制

| 嵄源类型 | 默认限制 | 可配置 | 说明 |
|---------|---------|--------|------|
| **最大会话数** | 10 | 是 | 同时活跃的最大会话数 |
| **最大等待数** | 5 | 是 | 同时等待答案的最大问题数 |
| **会话队列** | 20 | 是 | 超过限制后队列等待 |
