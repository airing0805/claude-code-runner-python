## 1. 概述

### 1.1 定义

会话状态管理是指从用户发起任务开始到会话结束的完整生命周期管理，包括状态转换、资源分配、超时控制和并发限制。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| **状态一致性** | 确保会话状态在所有组件间保持一致 |
| **资源管理** | 及时释放不再使用的会话资源 |
| **超时保护** | 防止会话无限期占用资源 |
| **并发控制** | 限制同时活跃的会话数量 |
| **用户体验** | 提供清晰的会话状态反馈 |

## 2. 会话模型

### 2.1 会话数据结构

```python
@dataclass
class SessionState:
    """内存中的会话状态（包含客户端实例）"""
    session_id: str                          # 会话唯一标识
    client: ClaudeCodeClient                 # Claude 客户端实例
    is_waiting: bool = False                # 是否在等待用户答案
    pending_question_id: Optional[str] = None  # 待处理的问题 ID
    created_at: float = field(default_factory=time.time)  # 创建时间戳
    last_activity: float = field(default_factory=time.time)  # 最后活动时间
    cwd: Optional[str] = None                # 工作目录

@dataclass
class SessionInfo:
    """会话信息（不含客户端，用于 API 返回）"""
    session_id: str                         # 会话唯一标识
    is_waiting: bool                        # 是否在等待用户答案
    pending_question_id: Optional[str]      # 待处理的问题 ID
    created_at: float                       # 创建时间戳
    last_activity: float                    # 最后活动时间
    cwd: Optional[str]                       # 工作目录
```

### 2.2 会话状态定义

| 状态 | 说明 | 可操作 |
|------|------|--------|
| `pending` | 任务已提交，等待执行 | ❌ 无需管理 |
| `idle` | 会话空闲，无活动 | ❌ 无需管理 |
| `running` | 任务执行中 | ✅ 可接收答案 |
| `waiting` | 等待用户答案 | ⚠️ 只允许提交答案 |
| `completed` | 任务已完成 | ❌ 无需管理 |
| `failed` | 会话错误 | ⚠️ 需要清理 |

> **状态说明**：
> - `pending`：任务已提交到执行队列，尚未开始执行
> - `idle`：会话空闲，无活动任务
> - `running`：任务正在执行中（等同于旧版 `executing`）
> - `waiting`：等待用户提交答案
> - `completed`：任务正常完成
> - `failed`：任务执行失败或出错（等同于旧版 `error`）

### 2.3 状态优先级

```
running > waiting > pending > completed > failed > idle
```

## 3. 生命周期管理

### 3.1 生命周期阶段

```
┌─────────────────────────────────────────────────┐
│              会话生命周期                           │
└─────────────────────────────────────────────────┘
                        │
                        ▼
    创建 (created)
                        │
                        ▼
    初始化 (initializing)
   ┌─────────────────────┴─────────────────────┐
   │                                     │
   ▼                                     ▼
执行中 (executing)  ←─→  等待 (waiting)
   │         │         │
   │         │         │
   │         │         ▼
   │         │    答案提交
   │         │         │
   │         │         ▼
   │         │    继续执行
   │         │         │
   │         │         ▼
   │         │    完成/错误
   │         │         │
   └─────────────┴─────────────────────┘
                        │
                        ▼
    终止 (terminated)
                        │
                        ▼
    清理 (cleaned)
```

### 3.2 阶段转换条件

| 从状态 | 到状态 | 触发条件 |
|--------|--------|---------|
| 创建 → 初始化 | 会话 ID 生成完成 | 立即 |
| 初始化 → pending | 任务已提交到执行队列 | API 调用成功 |
| pending → running | 客户端开始执行 | 任务真正开始执行 |
| running → waiting | AI 调用 AskUserQuestion tool_use 消息 |
| waiting → running | 用户提交答案 | /api/task/answer 调用 |
| running → completed | 任务完成 | complete 消息 |
| running → failed | 执行出错 | error 消息 |
| completed/failed → 终止 | 用户操作或超时 | 新会话/页面关闭 |
| 终止 → 清理 | 清理条件满足 | 超时/资源回收 |

## 5. 超时处理

### 5.1 超时类型

| 超时类型 | 时间限制 | 处理方式 |
|---------|---------|---------|
| **单次答案超时** | 300 秒 (5 分钟) | 超时后自动取消等待状态，前端提示"问题已超时"，发送 error 消息关闭 SSE 连接 |
| **会话总超时** | 3600 秒 (1 小时) | 自动结束会话，发送 error 消息关闭 SSE 连接，清理内存中的会话状态 |
| **活动超时** | 1800 秒 (30 分钟) | 无活动后自动结束会话，清理会话资源，发送 error 消息关闭 SSE 连接 |

### 5.2 超时处理流程

```
会话创建
  ↓
设置超时时间戳 (created_at + 3600 秒)
  ↓
定期检查超时
  ↓
超时检测 → 会话状态: error → 清理资源
  ↓
发送超时通知 → 前端显示"会话已超时"
```

## 6. 并发控制

### 6.1 并发限制

| 资源类型 | 默认限制 | 可配置 | 说明 |
|---------|---------|--------|------|
| **最大会话数** | 10 | 是 | 同时活跃的最大会话数 |
| **最大等待数** | 5 | 是 | 同时等待答案的最大问题数 |
| **会话队列** | 20 | 是 | 超过限制后队列等待 |

## 7. 清理策略

### 7.1 清理触发条件

| 触发条件 | 清理行为 |
|---------|---------|
| **任务完成 (complete)** | 保留会话文件，支持多轮对话，清理内存中的执行状态 |
| **任务错误 (error)** | 清理会话资源，关闭 SSE 连接 |
| **用户点击"新会话"** | 清理旧会话资源，创建新会话 |
| **会话总超时 (3600 秒)** | 自动结束会话，发送 error 消息关闭 SSE 连接，清理内存中的会话状态 |
| **活动超时 (1800 秒)** | 无活动后自动结束会话，清理会话资源，发送 error 消息关闭 SSE 连接 |
| **服务重启** | 内存会话丢失，但 jsonl 文件会话可恢复 |
| **页面关闭** | 清理内存会话资源，jsonl 文件保留（由后台清理任务处理超期文件） |

### 7.2 资源清理规则

| 资源类型 | 清理时机 | 清理方式 |
|---------|---------|---------|
| **内存会话状态** | 任务完成/错误/超时/新会话/服务重启 | 释放 SessionState 对象 |
| **SSE 连接** | 任务完成/错误/超时/连接断开 | 关闭连接，发送最终消息 |
| **jsonl 文件** | 会话超期（30 天） | 后台清理任务删除 |
| **客户端实例 (client)** | 会话结束时 | 释放 ClaudeCodeClient 资源 |
