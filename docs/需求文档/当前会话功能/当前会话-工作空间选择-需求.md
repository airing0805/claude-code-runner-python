## 1. 功能描述

**工作空间下拉选择**用于选择工作目录，支持选择历史记录或手动输入新路径。

选择历史工作空间后创建新会话，使用该工作空间。

---

## 2. 界面设计

```
工作空间: [ 项目A                           ▼ ]
            ├── 📁 E:\workspaces\project-a
            ├── 📁 E:\workspaces\project-b
            ├── 📁 C:\Users\admin\docs
            ─────────────────────
            ✏️ 输入新工作空间...
```

**组件说明**:
- 下拉框显示最近使用的工作空间列表
- 最下方提供手动输入选项
- 工作空间按最后使用时间倒序排列

---

## 3. 交互流程

```
1. 用户点击工作空间下拉框
   ↓
2. 显示工作空间列表（按最近使用排序）
   ↓
3. 用户选择历史工作空间
   ↓
4. 创建新会话，使用该工作空间
   ↓
5. 或用户手动输入新路径
   ↓
6. 验证路径，创建新会话
```

**用户反馈**:
| 时刻 | 反馈 |
|------|------|
| 路径验证失败 | 红色错误提示："工作目录不存在" |
| 新会话创建成功 | 绿色提示："已切换到项目A" |
| 会话ID更新 | 显示新生成的会话ID |

---

## 4. 选择行为规则

| 选择场景 | 行为 |
|---------|------|
| 选择历史工作空间 | 创建新会话，使用该工作空间 |
| 输入新路径 | 验证路径，创建新会话 |
| 清空选择 | 显示空白，可输入新路径 |

**显示规则**:
| 规则 | 说明 |
|------|------|
| 列表数量 | 最多显示10个工作空间 |
| 排序方式 | 按最后使用时间倒序 |
| 空状态 | 无历史时显示"暂无历史工作空间" |

---

## 5. 验收标准

- [ ] 下拉框显示工作空间列表（按最近使用排序）
- [ ] 选择历史工作空间后创建新会话
- [ ] 手动输入新路径后验证并创建新会话
- [ ] 无效路径显示错误提示

---

## 6. 业务规则

### 6.1 路径验证规则

| 验证项 | 规则 | 错误处理 |
|--------|------|----------|
| 路径存在性 | 必须是存在的目录 | 显示错误："工作目录不存在" |
| 路径可访问性 | 必须有读取权限 | 显示错误："无权限访问该目录" |
| 路径格式 | 支持绝对路径和相对路径 | 自动转换为绝对路径 |

### 6.2 会话状态规则

| 场景 | 行为 |
|------|------|
| 选择历史工作空间 | 清空当前会话，创建新会话 |
| 输入新路径并验证成功 | 清空当前会话，创建新会话 |
| 输入新路径验证失败 | 保持当前会话不变，提示错误 |
| 工作空间变更 | 重置会话ID为"新会话" |

### 6.3 数据存储规则

| 数据项 | 存储位置 | 说明 |
|--------|----------|------|
| 工作空间历史 | 前端 localStorage | 保存最近使用的10个工作空间 |
| 最后使用时间 | 前端 localStorage | 用于排序显示 |
