# 界面重构 - 主需求

本文档定义 v12 版本的界面重构需求，主要包括：移除 Tab 标签页、统一工作空间与会话管理、优化历史记录交互流程。

## 1. 概述

### 1.1 定义

界面重构是对当前会话功能的重大调整，旨在简化界面交互、统一会话管理流程、提升用户体验。

### 1.2 重构目标

| 目标 | 说明 |
|------|------|
| **简化交互** | 移除 Tab 标签页，降低认知负担 |
| **统一管理** | 工作空间切换 = 新会话，概念统一 |
| **优化流程** | 历史记录选择直接继续会话，减少操作步骤 |
| **保持兼容** | 保证现有功能不受影响（除 Tab 之外）|

### 1.3 影响范围

| 组件 | 影响 | 说明 |
|------|------|------|
| Tab 标签页系统 | 🗑️ 移除 | v9 已实现的功能将被移除 |
| 工作空间输入控件 | ✏️ 增强 | 支持下拉选择 + 手动输入 |
| 历史记录组件 | 🔄 调整 | 点击即可继续会话 |
| 继续会话复选框 | 🗑️ 移除 | 由工作空间切换替代 |
| 新会话按钮 | 🔄 调整 | 点击清空表单，而不是新开 Tab |

## 4. 交互设计

### 4.1 工作空间切换 = 新会话

**场景描述：** 用户切换工作空间时，应创建新的会话。

**交互流程：**

```
1. 用户在工作空间下拉框中选择 "项目A"
   ↓
2. 系统验证路径有效性
   ↓
3. 创建新会话
   - 生成新的会话ID
   - 清空输出显示区
   - 清空输入区
   - 更新会话状态为"新会话"
   ↓
4. 用户可立即开始执行任务
```

**用户反馈：**

| 时刻 | 反馈 |
|------|------|
| 路径验证失败 | 红色错误提示："工作目录不存在" |
| 新会话创建成功 | 绿色提示："已切换到项目A" |
| 会话ID更新 | 显示新生成的会话ID |

---

### 4.2 历史记录选择 = 继续会话

**场景描述：** 用户从历史记录中选择一个会话，应直接加载到当前界面继续。

**交互流程：**

```
1. 用户点击"历史记录"菜单项
   ↓
2. 侧边抽屉从右侧滑出
   ↓
3. 用户点击某个会话项
   ↓
4. 系统加载会话数据
   - 恢复工作空间路径
   - 恢复消息历史
   - 恢复工具配置
   - 更新会话ID
   ↓
5. 关闭抽屉，界面显示恢复的会话
   ↓
6. 用户可立即继续执行任务
```

**用户反馈：**

| 时刻 | 反馈 |
|------|------|
| 抽屉打开 | 平滑动画，遮罩层变暗 |
| 会话加载中 | 抽屉内显示加载指示器 |
| 加载成功 | 绿色提示："已恢复会话"，抽屉关闭 |
| 加载失败 | 红色提示："会话加载失败"，保留在抽屉中 |

---

### 4.3 新会话 = 清空表单

**场景描述：** 用户点击"新会话"按钮时，应清空当前界面的所有输入和输出。

**交互流程：**

```
1. 用户点击"新会话"按钮
   ↓
2. 系统确认操作（可选）
   - 提示："当前会话未保存，是否确定新建会话？"
   ↓
3. 清空界面
   - 清空输出显示区
   - 清空输入区
   - 重置工具配置到默认值
   - 清空会话ID（显示"新会话"）
   ↓
4. 用户可开始新的任务
```

**用户反馈：**

| 时刻 | 反馈 |
|------|------|
| 点击按钮 | 按钮按下效果 |
| 确认提示 | 对话框显示 |
| 表单清空成功 | 绿色提示："已创建新会话" |

## 5. 核心功能需求

### FR-001: 移除 Tab 标签页

**需求描述：** 彻底移除 Tab 标签页系统及相关代码。

**验收标准：**

- [ ] 前端界面不再显示 Tab 标签页
- [ ] 不再支持多个会话并行显示
- [ ] 移除相关的状态管理代码
- [ ] 移除 Tab 拖拽、排序等交互功能

---

### FR-002: 工作空间组合控件

**需求描述：** 工作空间输入控件支持下拉选择历史工作空间，或手动输入新路径。

**验收标准：**

- [ ] 控件支持两种模式：下拉选择、手动输入
- [ ] 下拉列表显示最近使用的工作空间（最多 10 个）
- [ ] 支持手动输入新路径
- [ ] 输入路径后自动验证有效性
- [ ] 无效路径显示错误提示

---

### FR-003: 工作空间切换 = 新会话

**需求描述：** 切换工作空间时自动创建新会话，清空当前界面。

**验收标准：**

- [ ] 选择工作空间后立即创建新会话
- [ ] 生成新的会话 ID
- [ ] 清空输出显示区
- [ ] 清空输入区
- [ ] 显示切换成功的提示

---

### FR-004: 历史记录继续会话

**需求描述：** 从历史记录中选择会话后，直接加载到当前界面继续。

**验收标准：**

- [ ] 点击历史记录项即可继续会话
- [ ] 自动恢复工作空间路径
- [ ] 自动恢复消息历史
- [ ] 自动恢复工具配置
- [ ] 更新会话 ID

---

### FR-005: 新会话清空表单

**需求描述：** 点击"新会话"按钮后，清空当前界面的所有输入和输出。

**验收标准：**

- [ ] 清空输出显示区
- [ ] 清空输入区
- [ ] 重置工具配置
- [ ] 会话 ID 显示为"新会话"
- [ ] 可选：未保存会话时提示确认

---

### FR-006: 工作空间历史记录

**需求描述：** 保存并显示最近使用的工作空间。

**验收标准：**

- [ ] 使用过的路径自动保存到本地存储
- [ ] 工作空间下拉框显示历史记录
- [ ] 历史记录按使用时间倒序排列
- [ ] 最多保存 10 个历史路径

---

## 6. 业务规则

### BR-001: 会话唯一性

**规则描述：** 每次切换工作空间都生成新的会话 ID，确保会话唯一。

**实现方式：**
- 使用 UUID 生成会话 ID
- 存储会话信息到本地

---

### BR-002: 历史记录显示顺序

**规则描述：** 历史记录按最后使用时间倒序显示。

**实现方式：**
- 每次使用会话后更新时间戳
- 查询时按时间倒序排序

---

### BR-003: 工作空间路径验证

**规则描述：** 用户输入的工作空间路径必须有效才能切换。

**验证逻辑：**
- 路径是否存在
- 路径是否可读
- 路径是否在允许范围内

---

## 7. 验收标准

### 7.1 功能验收

| 验收项 | 标准 |
|--------|------|
| Tab 移除 | 界面无 Tab 标签页，代码无 Tab 相关逻辑 |
| 工作空间控件 | 支持下拉选择和手动输入，历史记录正确显示 |
| 工作空间切换 | 切换后创建新会话，界面清空 |
| 历史记录继续 | 点击历史记录后正确加载会话 |
| 新会话清空 | 点击新会话后表单正确清空 |

### 7.2 性能验收

| 验收项 | 标准 |
|--------|------|
| 界面响应 | 工作空间切换响应时间 < 200ms |
| 历史记录加载 | 会话加载时间 < 500ms |
| 表单清空 | 新会话清空响应时间 < 100ms |

### 7.3 兼容性验收

| 验收项 | 标准 |
|--------|------|
| 现有功能 | 除 Tab 外，其他功能不受影响 |
| 数据兼容 | 历史会话数据可正常加载 |
