## 1. 消息类型定义

| 类型 | 说明 | 样式 | CSS 类名 | 状态 |
|------|------|------|----------|------|
| text | 文本内容 | 蓝色边框 | `.message-text` | ✅ 已实现 |
| thinking | 思考过程 (v0.2.11+) | 黄色背景，可折叠 | `.message-thinking` | ✅ 已实现 |
| tool_use | 工具调用 | 绿色边框 | `.message-tool_use` | ✅ 已实现 |
| tool_result | 工具结果 | 蓝色边框 | `.message-tool_result` | ✅ 已实现 |
| error | 错误信息 | 红色边框，红色文字 | `.message-error` | ✅ 已实现 |
| complete | 任务完成 | 绿色边框，成功提示样式 | `.message-complete` | ✅ 已实现 |
| info | 提示信息 | 天蓝色边框 | `.message-info` | ⚠️ 规划中（未实现） |
| ask_user_question | 用户问答 (v0.x.x+) | 紫色边框，交互式选项 | `.message-ask-question` | ✅ 已实现（SDK 不支持，占位） |

> **说明**: `info` 类型在早期版本规划中使用，当前版本（v14）尚未实现。如有需要可在后续版本中添加。

### 样式一致性要求

| 消息类型 | 边框颜色 | 背景色 | 文字色 | 示例 |
|----------|----------|--------|--------|
| text | `#3b82f6` (blue-500) | 透明 | 默认 | 蓝色边框卡片 |
| thinking | `#fbbf24` (amber-400) | `#fef3c7` (amber-100) | `#92400e` (amber-800) | 黄色背景可折叠 |
| tool_use | `#22c55e` (green-500) | 透明 | 默认 | 绿色边框卡片 |
| tool_result | `#3b82f6` (blue-500) | 透明 | 默认 | 蓝色边框卡片 |
| error | `#ef4444` (red-500) | `#fef2f2` (red-50) | `#dc2626` (red-600) | 红色边框+背景 |
| complete | `#22c55e` (green-500) | `#f0fdf4` (green-50) | `#15803d` (green-700) | 绿色边框+背景 |
| info | `#0ea5e9` (sky-500) | `#f0f9ff` (sky-50) | `#0369a1` (sky-700) | 天蓝色边框+背景 |
| ask_user_question | `#a855f7` (purple-500) | `#faf5ff` (purple-50) | `#7e22ce` (purple-700) | 紫色边框+背景 |

### 2.2 消息内容渲染

| 消息类型 | 内容渲染规则 |
|----------|------------|
| text | 普通文本，代码块使用等宽字体和语法高亮 |
| thinking | 浅灰色背景，带思考图标，可折叠 |
| tool_use | 绿色边框卡片，显示工具名称和参数 |
| tool_result | 蓝色边框卡片，显示执行结果 |
| error | 红色边框和文字，带错误图标 |
| complete | 绿色成功提示，显示统计摘要 |
| info | 天蓝色边框卡片，提示信息 |
| ask_user_question | 紫色边框卡片，交互式选项 |

### 2.3 消息时间戳显示

| 消息类型 | 时间戳显示 | 格式 | 位置 |
|----------|----------|------|------|
| text | 可选显示 | HH:mm:ss | 消息右侧 |
| thinking | 不显示 | - | - |
| tool_use | ✅ 必须显示 | HH:mm:ss | 消息右侧 |
| tool_result | ✅ 必须显示 | HH:mm:ss | 消息右侧 |
| error | ✅ 必须显示 | HH:mm:ss | 消息右侧 |
| complete | ✅ 必须显示 | HH:mm:ss | 消息右侧 |
| info | 可选显示 | HH:mm:ss | 消息右侧 |
| ask_user_question | ✅ 必须显示 | HH:mm:ss | 消息右侧 |

> **说明**：
> - 时间戳使用 24 小时制，格式为 `HH:mm:ss`（如 `14:30:05`）
> - 时间戳显示在消息卡片右侧，与消息内容垂直居中对齐
> - 实时流式消息（text、thinking）默认不显示时间戳，避免频繁更新造成视觉干扰
> - tool_use、tool_result、error、complete、ask_user_question 必须显示时间戳，便于用户追踪操作时序
> - ask_user_question 的时间戳显示在问答对话框右上角

### 2.4 加载状态与进度显示

#### 2.4.1 流式消息加载指示器

| 状态 | 显示内容 | 样式 |
|------|---------|------|
| 正在连接 | "正在连接..." | 灰色文字，带旋转图标 |
| 正在接收 | "正在接收消息..." | 灰色文字，带旋转图标 |
| 消息计数 | "已接收 N 条消息" | 灰色文字，位于底部状态栏 |

#### 2.4.2 执行按钮加载状态

| 状态 | 按钮文字 | 样式 |
|------|---------|------|
| 默认 | "➤ 发送" | 蓝色背景 #3b82f6 |
| 加载中 | "⏳ 执行中" | 禁用状态，显示旋转图标 |
| 停止中 | "⏹ 停止中" | 红色背景 #ef4444，禁用状态 |

#### 2.4.3 问答对话框加载状态

| 状态 | 显示内容 | 样式 |
|------|---------|------|
| 等待答案 | 问答对话框正常显示 | 紫色边框 #a855f7 |
| 提交中 | "提交中..." | 确认按钮禁用，显示加载指示器 |
| 提交失败 | "提交失败，请重试" | 红色文字提示 |

## 3. tool_use 消息

### 3.1 消息结构

```python
@dataclass
class ToolUseMessage:
    type: MessageType           # MessageType.TOOL_USE
    content: str             # 工具调用描述文本
    timestamp: str           # ISO 8601 时间戳
    tool_name: str           # 工具名称，如 "Read", "Write"
    tool_input: dict         # 工具参数
    tool_use_id: str         # 工具调用 ID
```

### 3.2 JSON 格式

```json
{
  "type": "tool_use",
  "content": "调用工具: Read",
  "timestamp": "2026-03-04T10:00:00.000Z",
  "tool_name": "Read",
  "tool_input": {
    "file_path": "/path/to/file.py"
  },
  "tool_use_id": "call_abc123def456"
}
```

### 3.3 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | ✅ | 固定为 "tool_use" |
| `content` | string | ✅ | 工具调用描述文本，供前端展示 |
| `timestamp` | string | ✅ | ISO 8601 格式时间戳 |
| `tool_name` | string | ✅ | 工具名称，如 "Read", "Write", "Edit" 等 |
| `tool_input` | object | ✅ | 工具参数，结构由具体工具定义 |
| `tool_use_id` | string | ✅ | 唯一标识，用于与 tool_result 配对 |

### 3.4 常用工具名称

| 工具名 | 说明 | 图标 |
|-------|-----|------|
| Read | 读取文件 | 📖 |
| Write | 写入文件 | 📝 |
| Edit | 编辑文件 | 📝 |
| Bash | 执行命令 | 💻 |
| Glob | 文件模式匹配 | 🔍 |
| Grep | 内容搜索 | 🔎 |
| Task | 启动子代理 | 🤖 |

### 3.5 前端展示规则

| 工具 | 展示方式 |
|------|---------|
| Read | 显示文件路径 + 读取图标 | `📖 Read: src/main.py` |
| Write | 显示文件路径 + 写入图标 | `✏️ Write: src/utils.py` |
| Edit | 显示文件路径 + 编辑图标 | `📝 Edit: src/main.py` |
| Bash | 显示命令内容 + 执行图标 | `💻 Bash: pytest tests/` |
| Glob | 显示模式 + 搜索图标 | `🔍 Glob: *.py` |
| Grep | 显示模式 + 搜索图标 | `🔎 Grep: import` |

## 4. tool_result 消息

### 4.1 消息结构

```python
@dataclass
class ToolResultMessage:
    type: MessageType           # MessageType.TOOL_RESULT
    content: str             # 结果内容或错误信息
    timestamp: str           # ISO 8601 时间戳
    tool_use_id: str         # 关联的 tool_use ID
    is_error: bool           # 是否错误
    error_type: str | None   # 错误类型
    metadata: dict          # 附加元数据
```

### 4.2 JSON 格式

```json
{
  "type": "tool_result",
  "content": "文件内容或错误信息",
  "timestamp": "2026-03-04T10:00:00.000Z",
  "tool_use_id": "call_abc123def456",
  "is_error": false,
  "error_type": null,
  "metadata": {
    "exit_code": 0,
    "lines_read": 150
  }
}
```

### 4.3 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | ✅ | 固定为 "tool_result" |
| `content` | string | ✅ | 结果内容（成功时）或错误信息（失败时） |
| `timestamp` | string | ✅ | ISO 8601 格式时间戳 |
| `tool_use_id` | string | ✅ | 关联的 `tool_use.tool_use_id`，必须匹配 |
| `is_error` | boolean | ✅ | 是否执行失败 |
| `error_type` | string | ❌ | 错误类型（失败时） |
| `metadata` | object | ❌ | 附加元数据，根据工具类型而不同 |

### 4.4 与 tool_use 的关系

```
┌─────────────────────────────────────────────┐
│              工具调用完整流程                  │
└─────────────────────────────────────────────┘
                        │
                        ▼
[tool_use 消息]
  tool_name: "Read"
  tool_input: {"file_path": "src/main.py"}
  tool_use_id: "call_abc123"
                        │
                        ▼
                    工具执行
                        │
                        ▼
[tool_result 消息]
  tool_use_id: "call_abc123"  ← 配对 ID
  content: [文件内容]
  is_error: false
```

## 5. ask_user_question 消息

### 5.1 功能概述

当 Claude Code 在执行任务过程中需要用户确认、选择或输入信息时，会暂停执行并向用户提问。用户回答后，任务继续执行。

### 5.2 功能说明

**当前可用替代方案：**
- 用户可以在任务描述中直接回答问题
- 使用 continue_conversation 或 resume 参数管理会话上下文

### 5.3 问答组件设计

#### 5.3.1 组件属性

| 属性 | 说明 |
|------|------|
| 标题 | 显示 header 字段内容 |
| 问题描述 | 显示 question 字段内容 |
| 选项列表 | 根据 options 数组渲染选项 |
| 多选支持 | multiSelect = true 时显示复选框 |
| 单选支持 | multiSelect = false 时显示单选按钮 |
| 确认按钮 | 提交用户选择 |
| 取消按钮 | 取消问答（可选） |

#### 5.3.2 组件样式


### 5.4 答案格式

**单选答案**:
```json
{
  "请选择一个功能": "背唐诗"
}
```

**多选答案**:
```json
{
  "请选择一个功能": ["背唐诗", "讲笑话"]
}
```

**文本输入答案**:
```json
{
  "自定义端口号": "8080"
}
```
