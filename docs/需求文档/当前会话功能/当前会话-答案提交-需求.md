## 1. 概述

### 1.1 定义

答案提交 API 是用户在前端问答对话框中选择答案后，调用后端接口将答案传递给 AI，使 Agent Loop 继续执行的机制。

### 1.2 核心特征

| 特征 | 说明 |
|------|------|
| 触发时机 | 用户在问答对话框中提交答案 |
| 目的 | 将用户答案作为 `tool_result` 发送给 AI |
| 会话状态 | 提交时会话必须处于等待状态 |
| 阻塞解除 | 答案提交后 AI 恢复执行 |

### 1.3 业务流程

```
用户发起任务
  ↓
AI 执行中...
  ↓
AI 调用 AskUserQuestion 工具
  ↓ [tool_use 消息]
前端显示问答对话框
  ↓ [会话状态: waiting]
用户选择并提交答案
  ↓ [调用 /api/task/answer API]
后端验证会话状态和答案
  ↓
将答案作为 tool_result 发送给 AI
  ↓ [会话状态: executing]
AI 继续执行任务
  ↓
完成或下一轮问题
```

## 3. API 定义

### 3.1 端点信息

| 属性 | 值 |
|------|-----|
| **URL** | `/api/task/answer` |
| **方法** | `POST` |
| **认证** | 需要认证（如用户认证） |
| **内容类型** | `application/json` |

### 3.2 请求模型

```typescript
interface QuestionAnswerRequest {
  // 会话 ID（必填）
  session_id: string;

  // 问题 ID（必填）
  question_id: string;

  // 答案（必填，类型取决于问题类型）
  answer: string | string[] | boolean | null;

  // 追问答案（可选）
  follow_up_answers?: {
    [questionId: string]: string | string[] | boolean | null
  };

  // 原始问题数据（可选，用于构建 toolUseResult）
  raw_question_data?: object;
}
```

### 3.3 响应模型

```typescript
interface QuestionAnswerResponse {
  // 是否成功
  success: boolean;

  // 响应消息
  message: string;

  // 错误代码（失败时）
  error_code?: string;
}
```

### 3.4 请求示例

**单选答案**:
```json
{
  "session_id": "abc123def456789",
  "question_id": "auth_strategy_01",
  "answer": "oauth2"
}
```

**多选答案**:
```json
{
  "session_id": "abc123def456789",
  "question_id": "oauth_providers",
  "answer": ["google", "github"]
}
```

**文本答案**:
```json
{
  "session_id": "abc123def456789",
  "question_id": "api_endpoint",
  "answer": "https://api.example.com/v2"
}
```

**带追问的答案**:
```json
{
  "session_id": "abc123def456789",
  "question_id": "auth_strategy_01",
  "answer": "oauth2",
  "follow_up_answers": {
    "oauth_providers": ["google"],
    "api_scope": ["read", "write"]
  }
}
```

### 3.5 响应示例

**成功响应**:
```json
{
  "success": true,
  "message": "答案已提交，任务继续执行"
}
```

**失败响应**:
```json
{
  "success": false,
  "message": "当前会话不在等待回答状态",
  "error_code": "session_not_waiting"
}
```

## 4. 会话状态管理

### 4.1 会话状态定义

| 状态 | 说明 | 可提交答案 |
|------|------|-------------|
| `idle` | 会话空闲，无活动 | ❌ 否 |
| `executing` | 任务执行中，未等待答案 | ❌ 否 |
| `waiting` | 正在等待用户答案 | ✅ 是 |
| `completed` | 任务已完成 | ❌ 否 |
| `error` | 会话处于错误状态 | ❌ 否 |

### 4.2 状态转换规则

```
任务启动 → executing
  ↓
AI 调用 AskUserQuestion → waiting
  ↓
用户提交答案 → executing
  ↓
AI 继续执行...
  ↓
任务完成或出错 → completed / error
```

### 4.3 状态检查流程

```python
# app/routers/task.py:503-547
@router.post("/answer")
async def submit_answer(answer: QuestionAnswerRequest):
    # 1. 获取会话
    session = await session_manager.get_session(answer.session_id)
    
    # 2. 验证会话存在
    if not session:
        raise HTTPException(status_code=404, detail="会话不存在或已过期")
    
    # 3. 验证会话在等待状态
    if not session.is_waiting:
        raise HTTPException(status_code=400, detail="当前会话不在等待回答状态")
    
    # 4. 验证 question_id 匹配
    pending_question_id = client.get_pending_question_id()
    if pending_question_id and pending_question_id != answer.question_id:
        raise HTTPException(status_code=400, detail="question_id 不匹配")
    
    # 5. 提交答案到 SDK
    await client.submit_answer({
        "question_id": answer.question_id,
        "answer": answer.answer,
        "follow_up_answers": answer.follow_up_answers,
        "raw_question_data": answer.raw_question_data,
    })
    
    # 6. 更新会话状态
    await session_manager.set_waiting(answer.session_id, False)
    
    return QuestionAnswerResponse(success=True, message="答案已提交，任务继续执行")
```

### 4.4 状态一致性保证

| 规则 | 说明 |
|------|------|
| **单一真理** | SessionManager 维护会话状态，确保一致性 |
| **原子操作** | 状态检查和答案提交应为原子操作 |
| **超时清理** | 超时的等待状态应自动清理 |

## 5. 验证规则

### 5.1 请求验证

| 字段 | 验证规则 | 错误提示 |
|------|---------|---------|
| `session_id` | UUID 格式，会话必须存在 | "会话不存在或已过期" |
| `question_id` | 非空字符串，匹配待处理问题 | "问题 ID 不匹配" |
| `answer` | 不能为空（除非问题允许） | "答案不能为空" |
| `answer` 类型 | 必须匹配问题类型 | "答案格式不正确" |

### 5.2 答案类型验证

| 问题类型 | answer 类型验证 | 验证规则 |
|---------|--------------|---------|
| `multiple_choice` | string[] | 至少选择一项 |
| `checkbox` | string[] | 至少选择一项（如果 required） |
| `text` | string | 不能为空（如果 required） |
| `boolean` | boolean | 必须 true 或 false |

### 5.3 追问答案验证

| 验证项 | 规则 |
|---------|------|
| **结构匹配** | follow_up_answers 的键必须对应问题的 follow_up_questions |
| **类型匹配** | 每个答案必须匹配对应问题类型 |
| **可选性** | 非 required 的问题允许答案为空或 null |

## 6. 错误处理

### 6.1 错误类型定义

| 错误代码 | HTTP 状态 | 说明 |
|---------|-----------|------|
| `session_not_found` | 404 | 会话不存在或已过期 |
| `session_not_waiting` | 400 | 会话不在等待回答状态 |
| `question_not_found` | 404 | 问题不存在（已超时或已回答） |
| `invalid_answer` | 400 | 答案格式无效 |
| `question_id_mismatch` | 400 | question_id 不匹配当前待处理问题 |
| `task_interrupted` | 400 | 任务已被中断 |
| `answer_timeout` | 408 | 答案提交超时 |

### 6.2 错误处理策略

| 错误类型 | 处理策略 | 用户体验 |
|---------|---------|---------|
| `session_not_found` | 提示会话已过期，建议新任务 | 🔴 "会话已过期，请重新开始" |
| `session_not_waiting` | 提示任务已继续或超时 | 🟡 "任务已继续或已超时，请刷新页面" |
| `question_not_found` | 提示问题已失效，等待下一个问题 | 🟡 "问题已失效，请等待下一个问题" |
| `invalid_answer` | 提示答案格式错误，请重新选择 | 🟠 "答案格式不正确，请重新选择" |
| `question_id_mismatch` | 提示问题不匹配，请刷新页面 | 🟡 "问题状态已变化，请刷新页面" |

### 6.3 错误响应格式

```json
{
  "success": false,
  "message": "当前会话不在等待回答状态",
  "error_code": "session_not_waiting"
}
```

## 7. 安全规则

### 7.1 认证授权

| 规则 | 说明 |
|------|------|
| **会话归属** | 用户只能提交自己会话的答案 |
| **答案注入** | 验证答案内容，防止注入攻击 |
| **会话劫持** | 验证会话 ID 有效性，防止跨会话操作 |

### 7.2 输入验证

| 验证项 | 规则 |
|---------|------|
| **长度限制** | 答案长度不超过 10000 字符 |
| **格式验证** | 根据问题类型验证答案格式 |
| **XSS 防护** | 对文本答案进行 HTML 转义 |
| **选项验证** | 单选/多选答案必须在选项列表内 |

### 7.3 并发控制

| 规则 | 说明 |
|------|------|
| **单答案处理** | 同一个问题同一时间只处理一个答案 |
| **重复提交** | 防止短时间内重复提交相同答案 |
| **状态锁定** | 提交答案时锁定会话状态，防止竞态条件 |
