## 三、用户旅程地图

### 3.1 首次使用旅程

```
用户访问网站
    ↓
[欢迎提示引导]
    1. 创建新会话开始任务
    2. 选择工作目录
    3. 输入任务描述
    4. 点击执行
    ↓
[任务执行中]
    实时查看 AI 响应和工具调用
    ↓
[任务完成]
    查看执行结果和文件变更
    ↓
[继续工作]
    勾选"继续会话"快速恢复上下文
    或者选择工作目录切换项目
```

### 3.2 日常使用旅程

```
用户启动应用
    ↓
[决策点]
    ├─ [继续上一次工作] → 勾选"继续会话" → 开始任务
    ├─ [开始新任务] → 点击"新会话" → 选择目录 → 开始任务
    └─ [回顾历史] → 打开历史记录 → 选择会话 → 恢复
    ↓
[任务执行]
    多轮对话，工具调用
    ↓
[完成/中断]
    会话自动保存到历史
```

### 3.3 多项目管理旅程

```
用户项目 A 任务完成
    ↓
切换到项目 B
    ↓
选择项目 B 工作目录
    ↓
查看项目 B 历史会话
    ↓
选择最近的会话恢复
    ↓
继续项目 B 的任务
    ↓
[需要回到项目 A]
    查看项目 A 历史会话
    ↓
恢复项目 A 会话
```

## 四、任务流程图

### 4.1 新建会话流程

```
开始
  ↓
用户点击"新会话"按钮
  ↓
清空所有输入区域
  ↓
生成新会话 ID
  ↓
会话 ID 输入框显示"新会话"
  ↓
任务输入框获得焦点
  ↓
等待用户输入
  ↓
[可选] 用户修改工作目录
  ↓
[可选] 用户展开配置选项
  ↓
用户点击"执行任务"
  ↓
开始任务执行
```

### 4.2 继续会话流程

```
开始
  ↓
用户勾选"继续会话"复选框
  ↓
系统检查是否有历史会话
  ↓
[无历史会话]
  ↓
禁用复选框
  显示提示"无历史会话可继续"
  ↓
等待用户操作
    ↓
[有历史会话]
    ↓
获取当前工作目录的最近会话 ID
  ↓
自动填充到会话 ID 输入框
  ↓
会话 ID 输入框设为只读
  ↓
工作目录自动设置为原会话目录
  ↓
等待用户确认或修改
    ↓
用户点击"执行任务"
  ↓
开始任务执行
```

### 4.3 历史会话恢复流程

```
开始
  ↓
用户打开历史记录页面
  ↓
选择项目
  ↓
查看该项目的会话列表
  ↓
用户点击"恢复"按钮
  ↓
加载会话到当前界面
  ↓
加载会话历史消息
  ↓
显示在输出区域
  ↓
工作目录锁定为原会话目录
  ↓
会话 ID 显示为只读
  ↓
等待用户操作
    ↓
[可选] 用户点击"新会话"取消恢复状态
  ↓
用户点击"执行任务"继续
```

## 五、边缘场景

### 5.1 异常场景

| 场景 | 预期行为 | 用户反馈 |
|------|---------|---------|
| **网络断开** | 显示离线提示，保存输入到本地 | "网络已断开，输入已保存" |
| **会话文件不存在** | 显示"会话已过期或不存在"，建议新建会话 | "会话已过期，请重新开始" |
| **目录不存在** | 显示错误提示，阻止任务执行 | "工作目录不存在，请重新选择" |
| **会话 ID 格式错误** | 显示格式错误提示，允许继续输入 | "请输入有效的会话 ID" |
| **权限不足** | 显示权限错误提示 | "无法访问该目录，权限不足" |
| **会话被锁定** | 显示"会话正被其他用户使用" | "该会话正在使用中，请稍后尝试" |

### 5.2 复杂场景

| 场景 | 处理策略 | 用户引导 |
|------|---------|---------|
| **快速连续点击"新会话"** | 防抖处理，只响应第一次 | 避免创建多个空会话 |
| **继续会话后修改工作目录** | 显示警告提示 | "修改工作目录将清除继续会话状态" |
| **大文件会话恢复** | 分页加载消息，显示加载进度 | "正在加载会话历史 (150/500)" |
| **工具调用过多** | 折叠显示，提供摘要 | "已折叠 50 次工具调用" |

## 六、用户调研计划

### 6.1 用户测试

| 测试类型 | 目标 | 方法 |
|---------|------|------|
| **可用性测试** | 验证界面易用性 | 邀请 5-10 名用户完成指定任务 |
| **A/B 测试** | 比较不同设计方案 | 对比继续会话 vs 手动输入会话 ID 的使用率 |
| **访谈测试** | 了解用户真实感受 | 半结构化访谈，收集反馈 |

### 6.2 反馈收集

| 渠道 | 用途 |
|------|------|
| **应用内反馈** | 一键反馈按钮，快速收集问题 |
| **GitHub Issues** | 功能建议和 Bug 报告 |
| **用户调研问卷** | 季度性满意度调查 |

### 6.3 关键指标 (KPIs)

| 指标 | 计算方式 | 目标值 |
|------|---------|-------|
| **首次使用完成率** | 新用户首次完成任务的比率 | > 80% |
| **继续会话使用率** | 使用继续会话功能的用户比例 | > 30% |
| **历史恢复使用率** | 从历史恢复会话的用户比例 | > 15% |
| **平均任务启动时间** | 从打开页面到开始任务的时间 | < 30s |
| **用户满意度** | 用户反馈评分 | > 4.0/5.0 |
