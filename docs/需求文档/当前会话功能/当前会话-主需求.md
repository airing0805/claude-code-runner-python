## 1. 概述

### 1.1 定义

当前会话功能是 Claude Code Runner 的核心功能，为用户提供与 Claude Code CLI 相似的交互界面，支持任务执行、会话管理和历史记录查看。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 实时流式输出 | 通过 SSE 实时显示任务执行过程 |
| 会话历史管理 | 支持继续最近会话和恢复历史会话 |
| 多轮对话支持 | 用户可在同一会话中进行多次交互 |
| 消息类型丰富 | 支持 text、thinking、tool_use、tool_result、error、complete 等多种消息类型 |
| 工具权限控制 | 可配置允许使用的工具列表 |

#### 3.3.3 信息栏 (配置区域)

| 配置项 | 类型 | 说明 |

| 统计项 | 说明 | 优先级 |

| 耗时 | 毫秒 (如: 12,345 ms) | P0 |
| 费用 | 美元 (如: $0.023) | P0 |
| 会话ID | 点击可复制 | P0 |
| 变更文件列表 | files_changed - 展示本次任务修改的文件路径列表 | P1 |
| 工具使用列表 | tools_used - 展示本次任务使用的工具名称列表 | P1 |


**权限模式说明：**

| 模式 | 说明 |
|------|------|
| default | 默认模式，允许使用 Claude Code 的标准工具集 |
| acceptEdits | 接受编辑模式，允许 Claude Code 修改文件 |
| plan | 计划模式，仅规划操作而不执行 |
| bypassPermissions | 绕过权限模式，允许所有操作（需谨慎使用） |

## 4. 交互设计

### 4.1 任务执行流程

```
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  用户输入任务描述  │────▶│  点击发送按钮     │────▶│  后端处理请求    │
└──────────────────┘     └──────────────────┘     └──────────────────┘
                                                            │
                                                            ▼
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  显示执行结果     │◀────│  实时流式推送    │◀────│  SSE 连接建立    │
└──────────────────┘     └──────────────────┘     └──────────────────┘
```

### 4.2 用户操作

| 操作 | 触发方式 | 响应 |
|------|----------|------|
| 发送任务 | 按 Enter 或点击"发送"按钮 | 输入框禁用，按钮切换为"停止" |
| 停止任务 | 点击"停止"按钮 | 发送 AbortController 请求，任务中断 |
| 清空输出 | 点击工具栏右侧的"清空"按钮 | 清空输出显示区，重置统计信息 |
| 新建会话 | 点击"新会话"按钮 | 清空输入框，会话ID重置为"新会话" |
| 继续会话 | 勾选"继续会话" | 自动使用最近会话ID填充 resume 字段 |

### 4.3 用户问答交互

当任务执行过程中遇到需要用户确认或选择的问题时，显示交互式问答组件：

| 操作 | 触发方式 | 响应 |
|------|----------|------|
| 显示问答 | 收到 ask_user_question 类型 SSE 消息 | 输出区显示紫色边框的问答卡片 |
| 选择选项 | 点击选项 | 高亮选中，可多选（checkbox 类型） |
| 提交回答 | 点击"确认"按钮 | 发送回答给后端 API |
| 取消问答 | 点击"取消"按钮 | 发送空答案或取消信号 |

## 5. 核心功能需求

### 5.1 基本任务执行

- 用户输入任务描述（prompt）
- 设置工作目录（working_dir）
- 配置允许的工具列表（tools）
- 执行任务并显示结果
- **交互式问答响应**: 任务执行过程中可接收用户问答消息（ask_user_question），需用户选择后继续执行

### 5.2 会话与任务管理

#### 5.2.1 会话延续支持

> **业务关联**: 会话的延续和恢复功能依赖于 [当前会话-会话格式.md](当前会话-会话格式.md) 中定义的会话文件格式和消息链结构。

- **continue_conversation**: 延续 CLI 中最近的对话历史
- **resume**: 通过会话 ID 恢复特定会话
- 会话 ID 自动提取和显示

#### 5.2.2 会话数据回填规则

| 数据项 | 来源 | 回填行为 |
|--------|------|----------|
| 会话ID (resume) | 选择的会话文件 `{sessionId}.jsonl` | 填充到会话ID输入框，设为只读 |
| 工作目录 (working_dir) | 会话创建时的项目路径 | 填充并锁定，不可修改 |
| 历史消息 | 完整会话消息历史 | 显示在输出区域 |

### 5.3 多轮会话支持

> **详细需求**: 详见 [当前会话-多轮会话-需求.md](当前会话-多轮会话-需求.md)

多轮会话是指用户和 AI 在单个会话中的多次消息交换：
- 都在同一个 `session_id` 下
- 消息通过 `parentUuid` 链接成链式结构
- 上下文会累积传递，每次回复都能看到之前的对话历史
- 用户主动发起多次输入

### 5.4 会话状态管理

> **详细需求**: 详见 [当前会话-会话状态.md](当前会话-会话状态.md)

会话状态包括：
- `idle` - 会话空闲，无活动
- `executing` - 任务执行中
- `waiting` - 等待用户答案
- `completed` - 任务已完成
- `error` - 会话错误

## 6. 业务规则

### 6.1 输入验证规则

| 字段 | 必填 | 长度/格式限制 | 验证规则 |
|------|------|---------------|----------|
| prompt | 是 | 1-10000 字符 | 不能为空或仅包含空白字符 |
| working_dir | 否 | 有效路径 | 必须是存在的目录路径 |
| tools | 否 | 数组 | 必须是有效的工具名称列表 |
| continue_conversation | 否 | 布尔值 | 默认 false |
| resume | 否 | 会话 ID 格式 | 必须是有效的会话 UUID |

### 6.2 权限与安全规则

| 规则项 | 说明 |
|--------|------|
| API 密钥认证 | 所有任务执行 API 需要有效的 API Key |
| 工作目录限制 | 只能在配置允许的目录范围内执行任务 |
| 目录遍历防护 | 禁止通过 working_dir 访问配置目录外的路径 |
| 工具权限控制 | 可配置允许使用的工具列表 |
| 请求大小限制 | prompt 最大 10000 字符 |

### 6.3 限流规则

| 端点 | 限流策略 | 说明 |
|------|----------|------|
| POST /api/task | 10 次/分钟/IP | 同步任务限流 |
| POST /api/task/stream | 10 次/分钟/IP | 流式任务限流 |

### 6.4 会话管理规则

| 规则 | 说明 |
|------|------|
| 会话 ID 生成 | 每个任务自动生成唯一会话 ID（UUID 格式） |
| 会话存储 | 任务会话数据存储在 `~/.claude/projects/{encoded-path}/{sessionId}.jsonl` |
| 会话延续 | continue_conversation 使用最近会话的上下文 |
| 会话恢复 | resume 通过指定会话 ID 读取对应 jsonl 文件恢复历史会话 |
| 会话保留 | 保留最近 30 天的会话数据 |

### 6.5 响应规则

**SSE 消息类型**:

| 类型 | 说明 | 对应需求 |
|------|------|----------|
| text | 文本输出 | [当前会话-消息类型.md](当前会话-消息类型.md) |
| thinking | 思考过程 | [当前会话-消息类型.md](当前会话-消息类型.md) |
| tool_use | 工具调用开始 | [当前会话-消息类型.md](当前会话-消息类型.md) |
| tool_result | 工具执行结果 | [当前会话-消息类型.md](当前会话-消息类型.md) |
| error | 错误信息 | [当前会话-消息类型.md](当前会话-消息类型.md) |
| complete | 任务完成 | [当前会话-消息类型.md](当前会话-消息类型.md) |
| ask_user_question | 用户问答，交互式问答 | [当前会话-消息类型.md](当前会话-消息类型.md) |

### 6.6 用户问答 API

当收到 `ask_user_question` 类型消息时，前端需要调用以下 API 提交用户答案：

**API 端点**: `POST /api/task/answer`

> **详细需求**: 详见 [当前会话-答案提交.md](当前会话-答案提交.md)

### 6.7 错误处理规则

| 错误类型 | HTTP 状态码 | 用户提示 |
|----------|-------------|----------|
| prompt 为空 | 422 | "任务描述不能为空" |
| prompt 过长 | 422 | "任务描述超过最大长度限制" |
| working_dir 无效 | 400 | "工作目录不存在或无权限访问" |
| 会话不存在 | 404 | "指定的会话不存在" |
| API 认证失败 | 401 | "API 密钥无效" |
| 超过限流 | 429 | "请求过于频繁，请稍后再试" |
| 服务内部错误 | 500 | "服务内部错误，请稍后重试" |

## 7. 验收标准

### 7.1 功能验收

- [ ] 同步模式正确返回任务执行结果
- [ ] 流式模式实时推送执行过程
- [ ] 会话延续功能正常工作
- [ ] 会话恢复功能正常工作
- [ ] 任务停止功能正常工作

### 7.2 验证规则验收

- [ ] 空 prompt 被正确拒绝
- [ ] 超长 prompt 被正确拒绝
- [ ] 无效 working_dir 被正确拒绝
- [ ] 限流规则正确生效

### 7.3 安全规则验收

- [ ] 未授权请求被正确拒绝
- [ ] 目录遍历攻击被阻止
- [ ] 敏感路径访问被阻止

### 7.4 错误处理验收

- [ ] 各类型错误返回正确的 HTTP 状态码
- [ ] 错误信息对用户友好
- [ ] 敏感信息不在错误响应中泄露
