# 历史记录

## 1. 功能概述

管理 Claude Code 执行的历史会话，提供项目列表和会话列表查询功能，让用户可以查看之前的任务执行记录，并支持继续历史会话。

> **数据关联**: 历史记录的数据来源于 Claude Code 会话存储（`~/.claude/projects/`），通过解析 jsonl 文件获取会话信息。与当前会话的恢复功能（resume/continue_conversation）共用同一数据源。

## 2. 用户场景

| 场景 | 描述 |
|------|------|
| 查看项目列表 | 用户想了解有哪些项目使用过 Claude Code |
| 查看项目会话 | 用户想查看某个项目的所有历史会话 |
| 继续历史会话 | 用户想恢复某个历史会话继续对话 |
| 查询特定会话 | 用户想通过会话 ID 直接定位特定会话 |

## 3. 功能列表

### 3.1 项目列表

- 获取所有使用过 Claude Code 的项目
- 显示每个项目的基本信息：
  - 项目编码名称
  - 项目路径
  - 会话数量
  - 使用的工具列表

### 3.2 会话列表

- 获取指定项目的所有历史会话
- 显示会话基本信息：
  - 会话 ID
  - 会话标题（首条用户消息）
  - 创建时间
  - 消息数量
  - 会话数据大小
- **继续会话入口**：每个会话提供"继续此会话"按钮

#### 3.2.1 会话标题生成规则

会话标题用于在列表中快速识别会话内容，生成规则如下：

| 规则项 | 说明 |
|--------|------|
| **标题来源** | 提取 jsonl 文件中首条 user 消息的 content |
| **content 类型处理** | 字符串类型直接使用；数组类型遍历提取 text 类型内容块 |
| **标签过滤** | 解析时过滤 `<ide_selection>` 和 `<ide_opened_file>` 标签内容 |
| **长度限制** | 最大 100 字符，超出部分截断并添加 "..." |
| **空内容处理** | 如果首条用户消息被完全过滤，则使用 "无标题" 作为默认值 |

**标题提取流程**：

```
1. 读取 jsonl 文件首行（首条 user 消息）
2. 解析 content 字段：
   - 如果是字符串：直接作为标题候选
   - 如果是数组：遍历内容块，提取 type="text" 的 text 字段
3. 过滤处理：
   - 移除 <ide_selection>...</ide_selection> 标签及其内容
   - 移除 <ide_opened_file>...</ide_opened_file> 标签及其内容
4. 合并和截断：
   - 将多个 text 内容块用空格连接
   - 如果长度超过 100 字符，截断并添加 "..."
5. 空值处理：
   - 如果处理后结果为空字符串，则使用 "无标题"
```

#### 3.2.2 会话摘要生成规则

会话摘要用于在下拉列表等空间受限的场景中显示会话预览，生成规则如下：

| 规则项 | 说明 |
|--------|------|
| **摘要来源** | 与标题来源相同，提取 jsonl 文件中首条 user 消息的 content |
| **content 类型处理** | 字符串类型直接使用；数组类型遍历提取 text 类型内容块 |
| **标签过滤** | 解析时过滤 `<ide_selection>` 和 `<ide_opened_file>` 标签内容 |
| **长度限制** | 最大 30 字符，超出部分截断并添加 "..." |
| **空内容处理** | 如果首条用户消息被完全过滤，则使用 "无摘要" 作为默认值 |

**摘要提取流程**：

```
1. 读取 jsonl 文件首行（首条 user 消息）
2. 解析 content 字段：
   - 如果是字符串：直接作为摘要候选
   - 如果是数组：遍历内容块，提取 type="text" 的 text 字段
3. 过滤处理：
   - 移除 <ide_selection>...</ide_selection> 标签及其内容
   - 移除 <ide_opened_file>...</ide_opened_file> 标签及其内容
4. 合并和截断：
   - 将多个 text 内容块用空格连接
   - 如果长度超过 30 字符，截断并添加 "..."
5. 空值处理：
   - 如果处理后结果为空字符串，则使用 "无摘要"
```

**标题与摘要的区别**：

| 属性 | 标题 (title) | 摘要 (summary) |
|------|-------------|----------------|
| 用途 | 会话列表页面的主要标识 | 下拉选择、预览等空间受限场景 |
| 长度限制 | 最大 100 字符 | 最大 30 字符 |
| 空值默认 | "无标题" | "无摘要" |
| API 返回 | sessions[].title | sessions[].summary |

> **注意**：标题和摘要的来源相同，差异仅在于长度限制和空值默认文本。

### 3.3 历史记录查询 API

#### 3.3.1 项目列表 API

```
GET /api/projects
```

**响应字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| projects | array | 项目列表 |
| projects[].name | string | 项目编码名称 |
| projects[].path | string | 原始项目路径 |
| projects[].session_count | number | 会话数量 |
| projects[].tools | string[] | 使用的工具列表 |

#### 3.3.2 会话列表 API

```
GET /api/sessions
```

**查询参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| working_dir | string | - | 工作目录路径（可选，用于过滤特定项目） |
| limit | number | 20 | 返回数量上限（最大 100） |
| offset | number | 0 | 偏移量（分页） |
| sort_by | string | "timestamp" | 排序字段（timestamp/title）：<br>- `timestamp`：按会话创建时间排序<br>- `title`：按会话标题（首条用户消息内容）排序 |
| order | string | "desc" | 排序方向（asc/desc） |

**排序说明**：

| sort_by 值 | 排序行为 |
|------------|----------|
| `timestamp` | 按会话创建时间排序，依赖 jsonl 首条 user 消息的 timestamp 字段 |
| `title` | 按会话标题排序，标题生成规则见 3.2.1 节；空标题（"无标题"）排在最后 |

**分页行为**：

- `limit` 和 `offset` 配合使用实现分页
- 第一页：`offset=0, limit=20`
- 第二页：`offset=20, limit=20`
- 排序在分页之前执行，确保结果顺序稳定

**响应字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| sessions | array | 会话列表 |
| sessions[].id | string | 会话 ID（UUID） |
| sessions[].title | string | 会话标题（生成规则见 3.2.1 节） |
| sessions[].summary | string | 会话摘要（生成规则见 3.2.2 节，用于下拉选择等空间受限场景） |
| sessions[].timestamp | string | 创建时间（ISO 8601） |
| sessions[].message_count | number | 消息数量 |
| sessions[].size_bytes | number | 会话数据大小（字节） |
| total | number | 符合条件的总会话数（分页前总量，用于计算总页数） |

## 4. 继续会话功能

### 4.1 交互流程

```
用户点击"继续此会话" → 系统解析会话数据 → 回填配置 → 在新标签页打开会话
```

### 4.2 数据回填规则

| 数据项 | 来源 | 回填行为 |
|--------|------|----------|
| 会话ID | 选择的会话文件 `{sessionId}.jsonl` | 填充到会话ID输入框，设为只读 |
| 工作目录 | 首条 user 消息的 `cwd` 字段 | 填充并锁定，不可修改 |
| 历史消息 | 解析 jsonl 文件中所有消息 | 显示在输出区域 |
| 会话标题 | 首条用户消息摘要 | 显示在标签页标题 |

### 4.3 交互限制

恢复历史会话后，以下操作被限制：

| 操作 | 限制 |
|------|------|
| 继续会话复选框 | 禁用 |
| 工作目录 | 锁定，不可修改 |
| 工具权限 | 可调整 |
| 新建会话按钮 | 正常可用 |

### 4.4 消息解析规则

> 详见 [当前会话-继续会话-需求.md](当前会话-继续会话-需求.md#8-消息解析规则)

- 判断 content 是字符串还是数组类型
- 字符串类型转换为 `{"type": "text", "text": "..."}`
- 数组类型遍历解析各内容块（text/thinking/tool_use/tool_result）
- 过滤 `<ide_selection>` 和 `<ide_opened_file>` 标签内容

### 4.5 历史记录与当前会话的关联关系

历史记录功能与当前会话功能共用同一数据源，数据关联关系如下：

| 当前会话功能 | 历史记录数据源 | 说明 |
|-------------|---------------|------|
| continue_conversation | `/api/sessions?sort_by=timestamp&order=desc&limit=1` | 获取最近会话 |
| resume | `/api/sessions/{session_id}` 或直接读取 jsonl 文件 | 恢复指定会话 |
| 会话数据回填 | 解析 `{encoded-path}/{sessionId}.jsonl` | 读取会话消息历史 |
| 会话标题提取 | jsonl 首条 user 消息 content | 生成会话摘要 |
| 工作目录锁定 | jsonl 首条 user 消息 cwd 字段 | 恢复原始工作空间 |

**数据流向**：

```
~/.claude/projects/{encoded-path}/{sessionId}.jsonl
         │
         ├──→ 历史记录页面：解析文件获取项目/会话列表
         │
         └──→ 当前会话页面：解析文件恢复会话上下文
                   │
                   ├── continue_conversation: 获取最近会话 → 填充 resume
                   │
                   └── resume: 指定 session_id → 读取对应 jsonl
```
