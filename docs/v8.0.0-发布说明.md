# Claude Code Runner v8.0.0 版本发布说明

**发布日期**: 2026-03-06

---

## 版本概述

v8.0.0 是 Claude Code Runner 的重要版本，专注于提升用户体验和效率。本次版本新增了**提问历史记录**功能，让用户能够快速查看和管理之前向 Claude 提出的问题。

---

## 新增功能

### 提问历史记录

**功能简介**

添加提问历史记录功能，让用户能够快速查看之前向 Claude 提出的问题，方便重复使用类似的问题。参考历史记录功能的设计模式，实现项目列表和提问列表的二级导航。

**核心特性**

| 特性 | 说明 |
|------|------|
| 项目隔离 | 按项目维度组织提问记录 |
| 完整内容 | 提取完整的提问文本（不限长度） |
| 智能过滤 | 自动过滤 IDE 标签和敏感信息 |
| 高效缓存 | 基于文件修改时间自动缓存 |

**功能详情**

- **项目列表**：显示所有有会话记录的项目
- **提问列表**：按项目显示历史提问，支持分页
- **一键复制**：点击复制按钮复制提问内容
- **继续会话**：点击提问进入对应会话继续对话
- **文本截断**：超过 200 字符自动截断，支持展开/收起

**详细使用说明**: [提问历史记录-功能介绍.md](提问历史记录-功能介绍.md)

## 技术更新

### 后端更新

| 模块 | 更新内容 |
|------|----------|
| API | 新增 `/api/projects/{project_name}/questions` 接口 |
| 数据提取 | 实现从会话文件提取首次用户提问 |
| 缓存机制 | 复用 MetadataCache 类，扩展提问缓存字段 |
| 敏感信息 | 添加 API Keys、GitHub Tokens 等脱敏处理 |

### 前端更新

| 模块 | 更新内容 |
|------|----------|
| 视图 | 新增 `Views.QUESTIONS` 常量 |
| 组件 | 创建 `questions.js` 模块 |
| UI | 创建提问列表 HTML 模板 |
| 样式 | 添加提问列表相关 CSS |
| 导航 | 更新 navigation.js 支持提问视图切换 |

### 数据存储

- **缓存位置**: 项目目录下的 `.metadata.json` 文件
- **缓存字段**: `has_question`, `question_timestamp`
- **失效策略**: 基于文件修改时间（mtime）自动失效

## API 接口

### 新增接口

#### 获取项目提问列表

**GET** `/api/projects/{project_name}/questions`

| 参数 | 类型 | 说明 |
|------|------|------|
| project_name | path | 项目编码名称 |
| page | query | 页码（默认 1） |
| limit | query | 每页数量（默认 20，最大 100） |

**响应示例**

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "d9eb61b6-5eb1-42f7-974c-832e54859911",
        "session_id": "d9eb61b6-5eb1-42f7-974c-832e54859911",
        "project_name": "E--workspaces-2026-project",
        "question_text": "帮我实现用户认证功能",
        "timestamp": "2026-03-05T10:30:00.000Z",
        "time_display": "3 小时前"
      }
    ],
    "total": 50,
    "page": 1,
    "limit": 20,
    "pages": 3
  },
  "project_path": "E:\\workspaces_2026\\project"
}
```

## 行为变化

| 场景 | 之前 | 现在 |
|------|------|------|
| 查看历史提问 | 无此功能 | ✅ 支持按项目查看历史提问 |

---

## 注意事项

1. **缓存首次加载**: 首次访问项目提问列表时，系统需要从会话文件中提取提问内容，可能需要稍长时间。二次访问时将使用缓存，响应更快。

2. **敏感信息**: 系统会自动过滤和脱敏敏感信息（如 API Keys、密码等），但仍建议用户不要在提问中包含敏感数据。

3. **空状态**: 如果项目没有有效提问，列表将显示空状态提示。

4. **移动端支持**: 提问历史记录功能支持移动端访问，响应式布局已适配。
