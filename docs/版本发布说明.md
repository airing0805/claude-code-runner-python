## v12.0.0 版本发布说明

> **版本**: v12.0.0
> **发布日期**: 2026-03-11

---

## 版本概述

v12.0.0 是 Claude Code Runner 的重大版本更新，专注于**界面重构**。这一版本移除了多 Tab 标签页系统，简化为单会话模式，同时引入工作空间组合控件、历史记录抽屉和自动数据迁移功能，显著提升了用户体验。

---

## 主要功能

### 1. 工作空间组合控件

- **下拉选择**: 支持选择历史工作空间（最多保留 10 条）
- **手动输入**: 支持直接输入新的工作空间路径
- **自动创建会话**: 切换工作空间时自动创建新会话

### 2. 历史记录抽屉

- **侧边抽屉**: 历史记录以侧边抽屉形式展示
- **继续会话**: 点击历史记录即可继续会话
- **自动恢复**: 自动恢复工作空间路径和消息历史
- **项目分组**: 按项目分组显示会话列表

### 3. 新会话功能

- **清空表单**: 点击新会话按钮清空当前表单
- **状态重置**: 会话 ID 显示为"新会话"
- **内容检查**: 未保存内容时显示确认对话框
- **工作空间保持**: 工作空间路径保持不变

### 4. 数据迁移

- **自动检测**: 首次加载时自动检测 v11 数据
- **自动转换**: 将多 Tab 对象转换为会话数组格式
- **去重处理**: 自动处理重复数据
- **进度提示**: 显示迁移进度和错误处理

---

## 移除功能

| 功能 | 说明 |
|------|------|
| Tab 标签页 | 多会话并行显示改为单会话模式 |
| 拖拽排序 | 移除 Tab 拖拽和排序功能 |
| 继续会话复选框 | 历史记录抽屉替代此功能 |
| v11 localStorage | 自动迁移到新格式 |

---

## Breaking Changes

### 1. Tab 标签页移除

- **影响**: 从多会话并行显示改为单会话模式
- **迁移**: 历史会话数据保留，可通过历史记录抽屉访问

### 2. localStorage 结构变更

- **旧格式**: `{ tabs: [...], activeTabId: string }`
- **新格式**: 会话数组格式
- **迁移**: 首次加载时自动迁移，旧数据保留但不使用

### 3. API 无变更

- 所有后端 API 保持兼容
- 无需修改集成代码

---

## v7.1.0 版本发布说明

> **版本**: v7.1.0
> **发布日期**: 2026-03-10

---

## 版本概述

v7.1.0 是任务调度系统的重要版本更新，专注于增强**日志查看功能**。这一版本提供了独立的日志标签页（正常日志/错误日志）、实时日志推送、历史日志分页加载和日志搜索功能，显著提升了用户排查问题的效率。

---

## 主要功能

### 1. 独立日志标签页

- **正常日志标签页**: 显示标准输出 (stdout) 日志
- **错误日志标签页**: 显示标准错误 (stderr) 日志
- **标签页状态保持**: 切换标签页后返回时保持选中状态

### 2. 实时日志推送

- **SSE 连接**: 通过 Server-Sent Events 实时接收日志
- **日志类型过滤**: 支持仅显示 stdout/stderr/all
- **自动滚动**: 新日志自动滚动到底部
- **暂停/恢复**: 支持暂停自动滚动

### 3. 历史日志分页

- **分页加载**: 大日志量时自动分页加载
- **时间范围过滤**: 支持按开始/结束时间过滤
- **加载状态指示**: 显示加载进度和状态

### 4. 日志搜索

- **关键字搜索**: 快速定位日志内容
- **正则表达式**: 支持正则表达式高级搜索
- **搜索结果导航**: 上一条/下一条导航

---

## API 变更

### 新增 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/scheduler/tasks/{task_id}/logs/normal` | GET | 获取正常日志 (stdout) |
| `/api/scheduler/tasks/{task_id}/logs/error` | GET | 获取错误日志 (stderr) |
| `/api/scheduler/tasks/{task_id}/logs/stream` | GET | SSE 实时日志流 |
| `/api/scheduler/tasks/{task_id}/logs/search` | GET | 搜索日志 |
| `/api/scheduler/tasks/{task_id}/logs/count` | GET | 获取日志数量统计 |

---

## 技术实现

### 后端

- **日志分类存储**: LogsStorage 支持按类型 (stdout/stderr) 过滤
- **SSE 事件推送**: 使用 asyncio 和调度器订阅机制实现实时日志
- **分页查询**: 支持 page/limit 分页和 start_time/end_time 时间过滤
- **正则搜索**: 支持关键字和正则表达式搜索

### 前端

- **日志标签页组件**: 独立标签页容器，支持切换
- **LogViewer 模块**: 专门处理日志查看逻辑
- **虚拟滚动**: 使用虚拟滚动渲染大量日志
- **SSE 客户端**: 处理实时日志接收和断线重连

### 日志数据模型

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 日志唯一标识 |
| task_id | string | 任务 ID |
| timestamp | string | ISO 8601 时间戳 |
| type | string | 日志类型 (stdout/stderr) |
| level | string | 日志级别 (debug/info/warn/error) |
| message | string | 日志消息 |

---

## 性能优化

| 优化项 | 说明 |
|--------|------|
| 虚拟滚动 | 前端使用虚拟滚动渲染大量日志 |
| 分页加载 | 历史日志分页加载，减少内存占用 |
| 日志缓冲 | 前端缓冲刷新，提升实时日志性能 |
| SSE 心跳 | 30 秒心跳保持连接，防止超时 |

---

## 使用方法

### 1. 查看任务日志

在任务详情对话框中，点击「日志」标签页：

- 默认显示「正常日志」标签页
- 点击「错误日志」标签页查看错误日志
- 支持分页加载历史日志

### 2. 实时日志

- 任务运行时自动开启实时日志流
- 新日志自动追加到列表底部
- 点击「暂停滚动」按钮暂停自动滚动
- 点击「恢复滚动」按钮恢复自动滚动

### 3. 搜索日志

- 在搜索框输入关键字
- 点击搜索按钮或按 Enter 搜索
- 启用「正则表达式」复选框使用正则搜索
- 使用「上一条」「下一条」按钮导航搜索结果

---

## 注意事项

1. 实时日志功能仅在任务运行时可用
2. 大日志量建议使用分页加载
3. 正则表达式搜索可能影响性能
4. SSE 连接超时时间为 30 秒

# v9.0.0 版本发布说明

> Claude Code Runner v9.0.0 版本发布说明
> **版本**: v9.0.0
> **发布日期**: 2026-03-06

---

## 版本概述

v9.0.0 是 Claude Code Runner 的重大版本更新，专注于实现**当前会话功能**。这一版本带来了完整的会话管理能力，支持同步/流式任务执行、会话历史查询、会话恢复和多轮对话交互。

---

## 主要功能

### 1. 任务执行

- **同步执行**: `POST /api/task` - 执行任务并等待结果返回
- **流式执行**: `POST /api/task/stream` - 通过 SSE 流式返回执行过程
- **实时消息**: 支持 text、thinking、tool_use、tool_result、complete、error 等消息类型

### 2. 会话管理

- **会话恢复**: 支持通过 session_id 恢复历史会话继续对话
- **会话列表**: 获取所有活动会话及状态
- **新会话创建**: 创建新会话，可选择结束现有会话
- **会话状态查询**: 实时查询会话等待状态

### 3. 会话历史

- **项目列表**: 获取所有项目（分页）
- **会话历史**: 获取指定项目的会话列表
- **消息历史**: 获取会话的完整消息记录
- **元数据缓存**: 优化会话列表查询性能

### 4. 问答交互

- **问题提示**: 当 Claude 需要用户回答时，实时推送问题消息
- **答案提交**: `POST /api/task/answer` - 提交用户答案
- **多轮对话**: 支持复杂的问答交互流程

---

## API 变更

### 新增 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/task` | POST | 同步执行任务 |
| `/api/task/stream` | POST | 流式执行任务 (SSE) |
| `/api/task/answer` | POST | 提交问答答案 |
| `/api/task/session/{session_id}/status` | GET | 获取会话状态 |
| `/api/task/sessions` | GET | 获取会话列表 |
| `/api/task/new-session` | POST | 创建新会话 |
| `/api/task/session/{session_id}/exists` | GET | 检查会话是否存在 |
| `/api/session/sessions` | GET | 获取历史会话列表 |
| `/api/session/projects` | GET | 获取项目列表 |
| `/api/session/projects/{project_name}/sessions` | GET | 获取项目会话列表 |
| `/api/session/sessions/{session_id}/messages` | GET/POST | 获取/提交会话消息 |
| `/api/session/projects/{project_name}/questions` | GET | 获取项目问答记录 |

---

## 技术实现

### 后端

- **会话管理**: SessionManager 类管理内存会话和状态
- **消息解析**: 支持 JSONL 文件解析，提取会话元数据
- **元数据缓存**: MetadataCache 缓存会话元数据，提升查询性能
- **SSE 流式输出**: 使用 asyncio 和 FastAPI 实现实时消息推送

### 前端

- **消息渲染器**: 支持多种消息类型的实时渲染
- **SSE 客户端**: 管理长连接，处理重连逻辑
- **问答对话框**: 展示问题选项，收集用户答案
- **会话信息栏**: 显示会话 ID、工作目录等元信息

### 数据存储

会话数据继续存储在 Claude Code 的项目目录：

```
~/.claude/projects/<encoded-path>/*.jsonl
```

新增元数据缓存机制，提升大量会话时的查询性能。

---

## 性能优化

| 优化项 | 说明 |
|--------|------|
| 元数据缓存 | 会话列表查询使用缓存，避免重复解析 JSONL 文件 |
| 分页支持 | 项目列表和会话历史支持分页加载 |
| SSE 压缩 | 消息流式传输，减少网络开销 |

---

## 已知限制

| 限制项 | 说明 |
|--------|------|
| 无任务取消 | 流式执行过程中无法取消任务 |
| 内存会话 | 服务重启后会话状态丢失，但文件仍可恢复 |
| 单机部署 | 当前不支持分布式部署 |
