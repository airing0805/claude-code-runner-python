## 三、需求一致性审查

| 历史记录 vs 继续会话 | ✅ 一致 | 会话列表 API 参数一致 |
| 主需求 vs 会话格式需求 | ✅ 一致 | 项目路径编码规则统一 |

### 3.2 概念定义统一性

| 概念 | 定义来源 | 一致性 |
|------|----------|--------|
| 会话 ID | 主需求、继续会话需求 | ✅ 统一使用 UUID 格式 (如 c102b7ad-6e4b-4c03-b5da-21cfba48f81f) |
| 项目路径编码 | 主需求、会话格式需求、历史记录需求 | ✅ 统一使用 `C:` → `C--`, `/` → `-` 规则 |
| 消息链关系 | 主需求、会话格式需求 | ✅ 统一使用 uuid + parentUuid 构成链式结构 |
| 权限模式 | 主需求、会话格式需求 | ✅ 统一使用 default, acceptEdits, plan, bypassPermissions |
| 会话状态 | 主需求、答案提交需求 | ✅ 统一使用 idle, executing, waiting, completed, error |

### 3.3 评审发现

| 问题类型 | 严重程度 | 描述 |
|----------|----------|------|
| 注意事项 | 低 | 消息解析时需注意空内容消息过滤导致的消息链断裂问题（详见会话格式需求文档 5.7 节） |
