### v7.0.6 - 后端开发-调度器核心 - 会话完整性修复

**执行角色**: 技术经理 (tech-manager)
**新增任务**: 修复调度任务会话不完整问题

**问题说明**:
- SDK 的 `ResultMessage` 发出时间早于 Claude Code 会话记录的完整写入时间
- 当 `async with ClaudeSDKClient` 上下文退出时，会话记录可能还在写入中
- 导致会话停留在 Assistant 的 "thinking" 阶段

**修复方案**:
- 在 `client.py` 的 `run()` 方法中，收到 `COMPLETE` 消息后添加 20 秒延迟
- 确保会话记录有足够时间完成写入

**相关代码**:
- `app/claude_runner/client.py` - `run()` 方法第 296-306 行
