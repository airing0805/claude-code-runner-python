## v7.1.0 - 任务日志增强功能

### 功能概述

在 v7.0.0 核心功能基础上，增强任务日志展示能力，提供两个独立的日志标签页（正常日志/错误日志），支持实时日志滚动显示、历史日志分页加载和日志搜索功能，提升用户排查问题的效率。

**需求文档**: docs/需求文档/任务日志增强-需求.md
**技术设计**: docs/技术设计/任务日志增强-技术设计.md

#### 实现的 API 端点
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/scheduler/tasks/{id}/logs/normal` | 获取正常日志 (stdout) |
| GET | `/api/scheduler/tasks/{id}/logs/error` | 获取错误日志 (stderr) |
| GET | `/api/scheduler/tasks/{id}/logs/stream` | SSE 实时日志流 |
| GET | `/api/scheduler/tasks/{id}/logs/search` | 搜索日志 |
| GET | `/api/scheduler/tasks/{id}/logs/count` | 获取日志数量统计 |

## 风险与对策

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| SSE 连接管理复杂性 | 中 | 使用成熟的事件源库，实现断线重连 |
| 大日志量性能问题 | 中 | 使用虚拟滚动，只渲染可见区域 |
| 实时日志延迟 | 低 | 优化后端日志写入，前端缓冲刷新 |
| 搜索性能问题 | 低 | 使用全文索引或限制搜索范围 |
