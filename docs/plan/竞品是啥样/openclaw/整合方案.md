# OpenClaw 与 Claude Code Runner 整合方案

> 如何将 OpenClaw 的消息能力与 Claude Code Runner 的项目级 Agent 能力结合

---

## 1. 整合背景

### 各自优势

| 产品 | 核心优势 |
|------|----------|
| **OpenClaw** | 多渠道消息集成（20+ 渠道）、消息路由、用户交互 |
| **Claude Code Runner** | 项目级 Agent 协同、评审流程、代码生成、定时任务 |

### 整合目标

- OpenClaw 作为**用户交互层**
- Claude Code Runner 作为**任务执行层**
- 实现：消息触发 → 项目执行 → 结果通知

---

## 3. 触发方式

### 方式1: Cron 定时触发

```yaml
# OpenClaw cron job 配置
jobs:
  - name: "daily-code-review"
    cron: "0 9 * * *"
    message: "执行每日代码评审"
    channel: "telegram"
    agent: "task-trigger"

  - name: "weekly-report"
    cron: "0 18 * * 5"
    message: "生成周报"
    channel: "discord"
```

```python
# Claude Code Runner 定时任务
# 接收 Cron 触发，执行任务
@app.post("/api/task")
async def run_task(request: TaskRequest):
    # 执行任务...
    return result
```

### 方式2: Slash 命令触发

```
用户: /dev 实现用户登录功能
     ↓
OpenClaw: 解析命令，提取任务
     ↓
Claude Code Runner API: POST /api/task
     ↓
执行产品规划→技术设计→开发流程
     ↓
返回结果给 OpenClaw
     ↓
OpenClaw: 推送结果到用户
```

### 方式3: 消息关键词触发

```yaml
# OpenClaw auto-reply 配置
autoReply:
  - keyword: "实现*功能"
    action: "trigger_task"
    template: "开始执行: {message}"
    target: "claude-code-runner"
```

## 4. 实现方案

### 4.1 OpenClaw 端配置

```yaml
# openclaw.json
extensions:
  - name: "claude-code-runner"
    type: "http-api"
    config:
      baseUrl: "http://localhost:8000"
      apiKey: "${CCR_API_KEY}"

# Slash 命令
commands:
  - name: "dev"
    description: "开发功能"
    usage: "/dev [功能描述]"
    trigger:
      type: "http-post"
      url: "http://localhost:8000/api/task"
      body:
        prompt: "{args}"
        type: "feature"
        priority: "P1"
```

### 4.2 Claude Code Runner 端配置

```python
# app/main.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import Optional

app = FastAPI()

class TaskRequest(BaseModel):
    prompt: str
    type: str = "feature"  # feature / bugfix / optimization
    priority: str = "P2"   # P0 / P1 / P2 / P3
    source: Optional[str] = None  # 触发来源
    callback_url: Optional[str] = None  # 回调 URL

@app.post("/api/task")
async def create_task(request: TaskRequest):
    # 解析任务类型
    task_type = request.type

    # 调用任务管理器执行
    result = await task_manager.execute(
        prompt=request.prompt,
        task_type=task_type,
        priority=request.priority,
    )

    # 如果有回调 URL，通知结果
    if request.callback_url:
        await notify(request.callback_url, result)

    return result

@app.post("/api/task/callback")
async def task_callback(task_id: str, result: dict):
    # 接收任务完成回调
    # 推送到 OpenClaw 对应渠道
    await openclaw_notify(task_id, result)
```

## 8. 限制与注意事项

### 限制

1. **网络要求**：OpenClaw 和 CCR 需要网络互通
2. **认证机制**：需要共享 API Key 或 Token
3. **任务时长**：长时间任务需要回调通知
4. **错误处理**：网络异常时需要重试机制

### 注意事项

1. **安全**：API Key 妥善保管，使用环境变量
2. **超时**：设置合理的任务超时时间
3. **监控**：记录任务执行日志
4. **告警**：任务失败时及时通知
