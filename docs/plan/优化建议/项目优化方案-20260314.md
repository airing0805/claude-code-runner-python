## 一、项目现状总结

### 1.1 项目定位
Claude Code Runner 是基于 Claude Agent SDK 的 Web 服务封装，通过 FastAPI 提供 REST API 和 Web 界面来调用 Claude Code 执行任务。

### 1.2 当前版本
- **版本**: v0.6.0
- **技术栈**: Python 3.10+, FastAPI 0.129+, Claude Agent SDK 0.1.44+
- **核心功能**: 任务执行、会话管理、MCP服务器管理、技能系统、任务调度

### 1.3 竞品对比分析
与 OpenClaw 相比，本项目具有以下特点：
- ✅ **优势**: 轻量级、部署简单、技术栈简洁、易于二次开发、MCP协议支持
- ⚠️ **待改进**: Web UI 基础、缺少消息渠道集成、前端交互体验

### 2.1 Claude Agent SDK 集成优化 [P0 - 高优先级]

#### 2.1.1 使用 Tool Runner Beta 功能

**现状问题**: 当前项目手动处理工具调用循环，代码复杂度高。

**优化方案**: 使用官方 SDK 提供的 Tool Runner 功能，自动处理：
- 工具执行和结果管理
- 会话状态管理
- 类型安全和验证

```python
# 推荐实现方式
from anthropic import Anthropic

def execute_with_tools(client, model, messages, tools):
    with client.messages.stream(
        model=model,
        max_tokens=4096,
        messages=messages,
        tools=tools,
    ) as stream:
        for event in stream:
            yield event
```

**预期收益**:
- 减少 30% 以上工具调用相关代码
- 提升类型安全性
- 自动支持新的 SDK 特性

```python
async def stream_with_heartbeat(generator, interval=30):
    """带心跳的流式输出"""
    last_heartbeat = time.time()
    async for event in generator:
        yield event
        if time.time() - last_heartbeat > interval:
            yield ": heartbeat\n\n"
            last_heartbeat = time.time()
```

### 2.5 任务调度系统优化 [P1 - 中优先级]

#### 2.5.1 添加心跳检测机制

**现状问题**: 无法检测任务是否真正在运行。

**优化方案**:
1. 实现任务心跳检测
2. 超时自动终止
3. 失败告警通知

```python
class TaskHeartbeat:
    def __init__(self, task_id: str, timeout: int = 300):
        self.task_id = task_id
        self.timeout = timeout
        self.last_heartbeat = time.time()

    async def check(self) -> bool:
        """检查任务是否存活"""
        return time.time() - self.last_heartbeat < self.timeout

    async def update(self):
        """更新心跳"""
        self.last_heartbeat = time.time()
```

#### 2.5.2 任务优先级队列

**优化方案**:
1. 实现多级优先级队列
2. 支持任务插队
3. 资源配额管理

```python
class PriorityTaskQueue:
    def __init__(self):
        self.queues = {
            'high': asyncio.Queue(),
            'normal': asyncio.Queue(),
            'low': asyncio.Queue()
        }

    async def put(self, task: Task, priority: str = 'normal'):
        await self.queues[priority].put(task)

    async def get(self) -> Task:
        # 优先处理高优先级任务
        for priority in ['high', 'normal', 'low']:
            if not self.queues[priority].empty():
                return await self.queues[priority].get()
```

### 2.6 安全性增强 [P0 - 高优先级]

#### 2.6.1 API Key 轮换机制

**优化方案**:
1. 支持多 API Key 负载均衡
2. 自动失效检测
3. 定期轮换提醒

```python
class APIKeyManager:
    def __init__(self, keys: List[str]):
        self.keys = keys
        self.current_index = 0
        self.failure_counts = {k: 0 for k in keys}

    def get_key(self) -> str:
        """获取可用的 API Key"""
        for _ in range(len(self.keys)):
            key = self.keys[self.current_index]
            if self.failure_counts[key] < 3:
                return key
            self.current_index = (self.current_index + 1) % len(self.keys)
        raise NoValidKeyError("No valid API key available")

    def report_failure(self, key: str):
        """报告 Key 失败"""
        self.failure_counts[key] += 1
```
