# 功能规划与开发计划编写规范

---

## 文档命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 版本开发计划 | `v{版本号}开发计划.md` | `v0.6开发计划.md` |
| 归档文件 | `archive/{原文件名}.md` | `archive/v0.5开发计划.md` |

**版本号格式**：`v{主}.{次}.{补丁}`，如 `v0.6.0`、`v0.6.1`

---

## 归档规则

当目录文件超过 **20 个**时，将历史文档归档到 `archive/` 子目录。

**保留不归档**：当前开发中的版本计划、3个月内的计划

---

## 任务规范

| 状态标识 | 含义 | 优先级 | 含义 |
|----------|------|--------|------|
| ✅ 已完成 | 任务完成 | P0 | 必须立即处理 |
| ⏳ 进行中 | 执行中 | P1 | 尽快处理 |
| 📋 待开发 | 等待开始 | P2 | 按计划处理 |
| ❌ 已取消 | 任务取消 | P3 | 有时间再处理 |

---

## 任务模板

```markdown
# v0.6 版本开发计划 - {主题}

## 版本管理

| 版本 | 状态 | 发布日期 | 说明 |
|------|------|----------|------|

| v0.6.1 | ✅/📋 | YYYY-MM-DD | 补丁优化 |
| v0.6.2 | ✅/📋 | YYYY-MM-DD | 其他功能 |

---

## v0.6.0 - {子版本名称}

{简要描述}

**需求文档**: docs/需求文档/xxx-需求.md

### v0.6.0.1 -  技术设计

  - [x] 系统架构图

### v0.6.0.2 - 后端开发

- [x] 实现数据模型
- [x] 实现 API 端点

```

### 任务版本号规范

| 层级 | 格式 | 说明 |
|------|------|------|
| 主版本 | `v2` | 当前版本标识 |
| 一级分类 | `v2.x` | 阶段/功能模块 |
| 二级分类 | `v2.x.y` | 具体任务组 |
| 执行任务 | 无前缀 | 具体动作项 |
