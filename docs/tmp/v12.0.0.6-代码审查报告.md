## 三、详细问题列表

### CRITICAL (严重问题)

#### 1. [CRITICAL] XSS 漏洞风险 - HistoryDrawer 错误消息

**文件**: `web/static/components/historyDrawer.js`  
**行号**: 343-346

```javascript
showError(message) {
    if (this.emptyEl) {
        this.emptyEl.innerHTML = `
            <div class="empty-icon">❌</div>
            <div class="empty-text">${this._escapeHtml(message)}</div>
        `;
```

**问题**: `showError` 方法使用了 `_escapeHtml`，但其他地方可能遗漏转义。需要确保所有用户输入都经过转义。

**建议**: 统一使用 DOM API 创建元素，避免使用 innerHTML。

### HIGH (高优先级)

#### 2. [HIGH] 事件监听器未清理 - 内存泄漏风险

**文件**: `web/static/components/workspaceCombo.js`  
**行号**: 181-185

```javascript
// 点击外部关闭下拉
document.addEventListener('click', (e) => {
    if (!this.wrapper.contains(e.target)) {
        this.closeDropdown();
    }
});
```

**问题**: 全局 click 事件监听器在 `destroy()` 方法中未移除，会导致内存泄漏。

**建议**: 保存事件处理函数引用，在 `destroy()` 中移除：

```javascript
// 构造函数中
this._boundDocumentClick = (e) => {
    if (!this.wrapper.contains(e.target)) {
        this.closeDropdown();
    }
};
document.addEventListener('click', this._boundDocumentClick);

// destroy 中
document.removeEventListener('click', this._boundDocumentClick);
```

#### 4. [HIGH] 迁移过程中断风险

**文件**: `web/static/modules/migration.js`  
**行号**: 43-115

```javascript
async runMigration(runner = null) {
    // ... 迁移逻辑
    // 步骤 6: 备份原数据
    this.backupV11Data(v11Data);

    // 步骤 7: 更新版本标识
    this.saveDataVersion(this.VERSION_V12);
```

**问题**: 如果用户在迁移过程中刷新页面或关闭浏览器，可能导致数据不一致。备份是在转换后才创建的，如果转换失败且备份也失败，数据可能丢失。

**建议**: 
1. 在迁移开始前先创建备份
2. 添加迁移状态持久化，支持断点续传

---

### MEDIUM (中优先级)

#### 5. [MEDIUM] 路径验证逻辑不完整

**文件**: `web/static/components/workspaceCombo.js`  
**行号**: 493-514

```javascript
_validatePathFormat(path) {
    // 防止路径遍历攻击
    if (path.includes('..') || path.includes('~')) {
        return { valid: false, error: '路径包含非法字符' };
    }
```

**问题**: 
1. `~` 在某些系统是合法的家目录表示，不应一概拒绝
2. 未检查其他危险路径模式（如 `/etc/passwd`）
3. Windows UNC 路径未处理

**建议**: 完善路径验证逻辑，考虑更多边界情况。

---

#### 6. [MEDIUM] 异步竞态条件 - loadProjects

**文件**: `web/static/modules/history.js`  
**行号**: 216-279

```javascript
async loadProjects(runner, page = 1) {
    // ...
    this.isLoadingProjects = false;  // 在 try 块外重置
```

**问题**: `isLoadingProjects` 在 try 块外重置，如果请求中途失败，状态可能不正确。快速连续调用可能导致数据错乱。

**建议**: 使用请求 ID 或 AbortController 来取消旧请求。

---

#### 7. [MEDIUM] 虚拟列表未正确销毁

**文件**: `web/static/modules/history.js`  
**行号**: 102-123

```javascript
initProjectVirtualList(runner) {
```
