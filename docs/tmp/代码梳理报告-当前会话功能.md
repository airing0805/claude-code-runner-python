## 二、发现的问题清单

### 🔴 高优先级问题

#### 问题 1: AskUserQuestion 多轮对话 - 前端缺少问答对话框

**文件**: `web/static/modules/task.js`

**问题描述**:
- 后端完整实现了 AskUserQuestion 多轮对话功能 (task.py:217-587)
- 前端 task.js:688-693 只是将问答消息渲染为普通文本
- 缺少让用户选择答案的问答对话框 UI

**严重程度**: 高

**影响范围**:
- 用户无法回答 Claude 的问答问题
- 任务会卡在等待状态

**修复建议**:
1. 在 `task.js` 中添加问答对话框组件
2. 解析 `ask_user_question` 消息中的 question 数据
3. 根据问题类型（single_select, multi_select, text 等）渲染对应的表单
4. 调用 `/api/task/answer` 提交用户答案

---

#### 问题 2: 历史消息分组逻辑依赖 permissionMode 字段

**文件**: `web/static/modules/message-renderer/core.js:217-267`

**问题描述**:
- `_groupByRounds` 方法通过 `msg.permissionMode` 判断新对话轮次
- 如果历史会话数据没有该字段（可能是旧格式或外部导入），会导致分组错误
- 当前只有第一层判断依赖此字段

**严重程度**: 高

**代码位置**:
```javascript
// core.js:224-228
if (msg.permissionMode) {
    currentRound = { user: msg, assistant: [] };
    rounds.push(currentRound);
}
```

**修复建议**:
1. 添加备用分组策略：基于消息时间间隔判断（间隔 > N 分钟视为新轮次）
2. 或基于消息 role 顺序判断（user → assistant 为一轮）

### 🟡 中优先级问题

#### 问题 3: 标签页拖拽排序 Bug

**文件**: `web/static/components/tabs.js:136-141`

**问题描述**:
```javascript
// 问题代码
const newIndex = Array.from(runner.tabsBar.children).indexOf(placeholder || draggedTab);
```
- `placeholder` 在 `dragend` 事件开始时已被移除（代码:130-133）
- 这里的逻辑会导致排序计算错误

**严重程度**: 中

**修复建议**:
```javascript
// 修改为
let newIndex = draggedIndex;
const allTabs = [...runner.tabsBar.children];
allTabs.forEach((child, index) => {
    if (child.classList.contains('dragging')) return;
    if (child.classList.contains('tab-placeholder')) {
        newIndex = index;
    }
});
```

#### 问题 4: 轮次折叠状态未按标签页隔离

**文件**: `web/static/modules/message-renderer/core.js:28`

**问题描述**:
- `_collapseState` 使用 `sessionId` 作为 key
- 同一个会话可能在多个标签页打开，折叠状态会互相影响

**严重程度**: 中

**修复建议**:
```javascript
// 改为复合 key
const key = `${sessionId}_${tabId}`;
```

---

#### 问题 5: 继续会话 - 工作目录变更后缓存不同步

**文件**: `web/static/modules/continueSession.js:174-187`

**问题描述**:
- 工作目录变更时调用 `updateContinueConversationState(true)` 强制刷新
- 但本地缓存的旧会话 ID 仍然存在，可能导致混淆

**严重程度**: 中

**修复建议**:
- 在工作目录变更时同时清理旧缓存

---

### 🟢 低优先级问题

#### 问题 6: 任务状态在不同标签页切换时可能不同步

**文件**: `web/static/modules/task.js:780-805`

**问题描述**:
- `setRunning` 方法优先使用 `this._taskTabId`，但任务完成后未及时清理
- 切换标签页时可能出现按钮状态错误

**修复建议**:
- 在任务完成或停止时明确清理 `_taskTabId`

---

#### 问题 7: SSE 重连时 resume 参数可能丢失

**文件**: `web/static/modules/task.js:397-413`

**问题描述**:
```javascript
const resumeId = runner.currentSessionId || ctx.resume;
```
- 重连时优先使用 `runner.currentSessionId`
- 如果此时 `currentSessionId` 尚未设置（首次连接未收到 session_id），会导致 resume 参数丢失

**修复建议**:
- 在请求开始时立即记录 session_id（如果后端有返回）

---

#### 问题 8: 消息渲染调试日志过多

**文件**: `web/static/modules/message-renderer/core.js` 多处

**问题描述**:
- 大量 console.log 影响性能
- 包含敏感信息（如完整的消息内容）

**修复建议**:
- 生产环境移除或改为调试级别日志
