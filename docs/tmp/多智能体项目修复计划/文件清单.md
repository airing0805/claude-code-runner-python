## 快速开始

### 步骤1：准备Skills

Skills文件已创建在 `.claude/skills/` 目录下，无需额外操作。

### 步骤2：准备执行日志

将执行日志模板复制到 `docs/tmp/` 目录：

```bash
# 复制模板
cp docs/tmp/多智能体项目修复计划/执行日志模板/*.md docs/tmp/

# 重命名去掉"模板"后缀
mv docs/tmp/需求梳理-执行日志模板.md docs/tmp/需求梳理-执行日志.md
mv docs/tmp/需求梳理-评审报告模板.md docs/tmp/需求梳理-评审报告.md
# ... 其他文件类似
```

### 步骤3：配置PM2

将 `PM2配置JSON.md` 中的JSON配置添加到PM2配置文件：

```bash
# 编辑PM2配置文件
notepad ~/.claude/scheduler/scheduled-tasks.json

# 粘贴配置内容
```

### 步骤4：启动执行

```bash
# 查看任务列表
/scheduler:schedule-list

# PM2会自动根据cron表达式执行
```

## 故障排查

### Skills未生效

检查：
1. `.claude/skills/` 目录下是否有对应的SKILL.md文件
2. SKILL.md文件格式是否正确
3. 技能是否需要重新加载

### PM2任务未执行

检查：
1. `enabled` 字段是否为 `true`
2. `cron` 表达式是否正确
3. `workspace` 路径是否存在
4. 查看执行日志了解错误

### 执行日志未保存

检查：
1. `docs/tmp/` 目录是否存在
2. 文件是否有写入权限
3. 路径是否正确
