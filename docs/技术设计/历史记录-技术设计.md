# 会话管理 API 技术设计

## 概述

会话管理 API 提供项目列表和会话列表查询功能，用于管理 Claude Code 执行的历史会话。

---

## 项目列表

### GET /api/projects

获取所有项目列表（含工具汇总）。

**响应**:

```json
{
  "projects": [
    {
      "encoded_name": "E--workspaces-2026-python",
      "path": "E:\\workspaces_2026_python\\project",
      "session_count": 5,
      "tools": ["Read", "Write", "Edit", "Bash"]
    }
  ]
}
```

---

## 会话列表

### GET /api/sessions

获取会话列表，支持分页和排序。

**查询参数**:

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| working_dir | string | - | 工作目录路径（可选，用于过滤特定项目） |
| limit | number | 20 | 返回数量上限（最大 100） |
| offset | number | 0 | 偏移量（分页） |
| sort_by | string | "timestamp" | 排序字段（timestamp/title） |
| order | string | "desc" | 排序方向（asc/desc） |

**排序行为**:

| sort_by 值 | 排序行为 |
|------------|----------|
| `timestamp` | 按会话创建时间排序，依赖 jsonl 首条 user 消息的 timestamp 字段 |
| `title` | 按会话标题排序，标题生成规则见 3.2.1 节；空标题（"无标题"）排在最后 |

**分页行为**:

- `limit` 和 `offset` 配合使用实现分页
- 第一页：`offset=0, limit=20`
- 第二页：`offset=20, limit=20`
- 排序在分页之前执行，确保结果顺序稳定

**响应**:

```json
{
  "sessions": [
    {
      "id": "abc123def456",
      "title": "列出当前目录下所有 Python 文件",
      "timestamp": "2026-02-18T10:00:00",
      "message_count": 10,
      "size_bytes": 5000
    }
  ],
  "total": 100
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| sessions | array | 会话列表 |
| sessions[].id | string | 会话 ID（UUID） |
| sessions[].title | string | 会话标题（首条用户消息，过滤 ide_selection 和 ide_opened_file 标签） |
| sessions[].timestamp | string | 创建时间（ISO 8601 格式） |
| sessions[].message_count | int | 消息数量（user + assistant） |
| sessions[].size_bytes | int | 会话数据大小（字节） |
| sessions[].cwd | string | 工作目录（来自首条 user 消息的 cwd 字段） |
| sessions[].tools | string[] | 本会话使用的工具名称列表 |
| sessions[].status | string | 会话状态（见会话状态枚举） |
| total | int | 符合条件的总会话数（分页前总量，用于计算总页数） |

## 数据结构

### 会话元数据

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 会话唯一 ID（UUID） |
| title | string | 会话标题（首条用户消息，过滤 ide_selection 和 ide_opened_file 标签） |
| timestamp | string | 创建时间（ISO 8601 格式） |
| message_count | int | 消息数量（user + assistant） |
| size_bytes | int | 会话数据大小（字节） |
| cwd | string | 工作目录（来自首条 user 消息的 cwd 字段） |
| tools | string[] | 本会话使用的工具名称列表 |
| status | string | 会话状态（见会话状态枚举） |

### 会话状态枚举

| 状态值 | 说明 | 可转换目标 |
|--------|------|-----------|
| `idle` | 会话空闲，无活动 | → running |
| `running` | 任务执行中 | → waiting / completed / failed |
| `waiting` | 等待用户答案 | → running / failed |
| `completed` | 任务已完成 | → idle（清理后） |
| `failed` | 会话错误 | → idle（清理后） |

> **说明**: 状态字段用于内存中的会话管理（`SessionState`），持久化的会话文件（jsonl）不存储状态，状态在服务重启后由 `idle` 开始。

### Session 数据模型（in-memory）

```python
@dataclass
class SessionState:
    """内存中的会话状态"""
    session_id: str                          # 会话唯一标识
    client: ClaudeCodeClient                 # Claude 客户端实例
    is_waiting: bool = False                # 是否在等待用户答案
    pending_question_id: Optional[str] = None  # 待处理的问题 ID
    created_at: float = field(default_factory=time.time)  # 创建时间戳
    last_activity: float = field(default_factory=time.time)  # 最后活动时间
    cwd: Optional[str] = None               # 工作目录

@dataclass
class SessionInfo:
    """会话信息（不含客户端，用于 API 返回）"""
    session_id: str
    is_waiting: bool
    pending_question_id: Optional[str]
    created_at: float
    last_activity: float
    cwd: Optional[str]
```

### 路径编解码设计

#### 编码规则（项目路径 → 目录名）

用于将项目绝对路径编码为 `~/.claude/projects/` 下的目录名：

| 操作系统 | 原始路径 | 编码后目录名 |
|---------|---------|-------------|
| Windows | `E:\workspaces\project` | `E--workspaces-project` |
| Windows | `C:\Users\admin\code` | `C--Users-admin-code` |
| Unix | `/home/user/project` | `-home-user-project` |

**编码算法** (`get_project_dir_name`):

```
1. 获取绝对路径
2. Windows 路径处理（C: 开头）:
   - 盘符大写：E → E
   - 跳过冒号和反斜杠 "E:\" → 保留前缀 "E--"
   - 剩余路径的反斜杠、反斜杠、斜杠、下划线都替换为 "-"
3. Unix 路径处理:
   - 所有反斜杠、斜杠、下划线替换为 "-"
   - 结果以 "-" 开头
```

**示例**:

```python
# Windows
get_project_dir_name("E:\\workspaces_2026\\project")
# → "E--workspaces-2026-project"

# Unix
get_project_dir_name("/home/user/project")
# → "-home-user-project"
```

#### 解码规则（目录名 → 项目路径）

用于将目录名还原为原始项目路径：

| 编码后目录名 | 解码后路径 |
|-------------|-----------|
| `E--workspaces-project` | `E:\workspaces\project` |
| `C--Users-admin-code` | `C:\Users\admin\code` |
| `-home-user-project` | `/home/user/project` |

**解码算法** (`decode_project_name`):

```
1. 匹配正则: ^([A-Za-z])--(.+)$
   - 匹配成功 → Windows 路径
     - 盘符大写
     - 剩余部分 "-" 替换为 "\"
     - 结果: "{盘符}:\{路径}"
   - 匹配失败 → Unix 路径
     - "-" 替换为 "/"
     - 结果: "/{路径}"
```

**示例**:

```python
decode_project_name("E--workspaces-2026-python")
# → "E:\\workspaces_2026\\python"

decode_project_name("-home-user-project")
# → "/home/user/project"
```

#### 注意事项

- 编码和解码是**无损转换**
- Windows 盘符后的冒号 `:` 转换为 `--`
- 下划线 `_` 在编码时也会替换为 `-`，解码时无法还原，但不影响路径定位
- 会话文件命名：`{sessionId}.jsonl`，sessionId 为 UUID 格式

## 会话状态转换

### 状态转换图

```
                    创建会话
                       │
                       ▼
                   idle ────► running
                       ▲         │
                       │         ▼
                       │       waiting
                       │         │
                       │         ▼
                       │    (答案提交)
                       │         │
                       └─────────┘
                           │
                           ▼
                    completed / failed
                           │
                           ▼
                       (清理)
                           │
                           ▼
                         idle
```

### 状态转换条件

| 当前状态 | 事件 | 下一状态 | 说明 |
|---------|------|---------|------|
| idle | 任务启动 | running | 创建新会话或恢复会话后开始执行 |
| running | 收到 ask_user_question | waiting | AI 请求用户确认/选择 |
| waiting | 用户提交答案 | running | 继续任务执行 |
| running | 任务正常完成 | completed | 收到 complete 消息 |
| running | 执行出错 | failed | 收到 error 消息 |
| completed | 清理超时 | idle | 会话资源释放 |
| failed | 清理超时 | idle | 会话资源释放 |

### 超时配置

| 超时类型 | 默认值 | 说明 |
|---------|-------|------|
| 单次答案超时 | 300 秒 (5 分钟) | waiting 状态下用户未提交答案 |
| 会话总超时 | 3600 秒 (1 小时) | 整个会话的最大时长 |
| 活动超时 | 1800 秒 (30 分钟) | 无活动后自动结束 |
| 清理间隔 | 14400 秒 (4 小时) | 定期清理过期会话 |
