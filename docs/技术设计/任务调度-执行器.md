# 任务调度 - 任务执行器设计

> 本文档定义任务执行器的设计，包括执行流程、超时控制、状态机、重试机制和错误处理。

## 1. 执行器职责

任务执行器 (`TaskExecutor`) 负责执行单个任务，核心职责包括：

- 调用 ClaudeCodeClient 执行任务
- 超时控制机制
- 任务状态跟踪
- 失败重试机制
- 错误信息收集

## 2. 执行流程设计

### 2.1 主执行流程

```
                    ┌───────────────────┐
                    │   验证任务有效性    │
                    └──────────┬────────┘
                               │
                    ┌──────────┴──────────┐
                    │                     │
              验证失败                 验证成功
                    │                     │
                    ▼                     ▼
          返回失败结果          ┌───────────────────┐
                               │  设置执行状态      │
                               └──────────┬────────┘
                                          │
                                          ▼
                               ┌───────────────────┐
                               │  更新状态为        │
                               │  RUNNING          │
                               └──────────┬────────┘
                                          │
                                          ▼
                               ┌───────────────────┐
                               │  带超时执行任务    │
                               └──────────┬────────┘
                                          │
                    ┌─────────────────────┼─────────────────────┐
                    ▼                     ▼                     ▼
             ┌─────────────┐       ┌─────────────┐       ┌─────────────┐
             │  执行成功    │       │  执行失败    │       │  执行超时    │
             └──────┬──────┘       └──────┬──────┘       └──────┬──────┘
                    │                     │                     │
                    ▼                     ▼                     ▼
             ┌─────────────┐       ┌─────────────────────────────────┐
             │ COMPLETED   │       │         _handle_retry()          │
             └─────────────┘       └──────────────┬──────────────────┘
                                                    │
                                        ┌──────────┴──────────┐
                                        │                     │
                                  可重试               不可重试
                                        │                     │
                                        ▼                     ▼
                                 ┌─────────────┐     ┌─────────────┐
                                 │ PENDING     │     │ FAILED      │
                                 │ (重新入队)   │     │ (保存历史)   │
                                 └─────────────┘     └─────────────┘
```

## 5. 重试策略设计

### 5.1 重试条件

| 错误类型 | 可重试 |
|----------|--------|
| TRANSIENT (临时性) | 是 |
| TIMEOUT (超时) | 是 |
| RESOURCE (资源) | 是 |
| PERMANENT (永久性) | 否 |
| USER_CANCEL (用户取消) | 否 |
| VALIDATION (验证) | 否 |

### 5.2 重试限制

| 配置项 | 值 |
|--------|-----|
| MAX_RETRIES | 2 |
| BASE_DELAY | 5.0秒 |
| MAX_DELAY | 60.0秒 |
| JITTER | ±10% |

### 5.3 指数退避算法

```
delay = base_delay * 2^retry_count
delay = min(delay, max_delay)
delay = delay ± (delay * jitter * random)
```

| 重试次数 | 基础延迟 |
|----------|----------|
| 1 | ~10s |
| 2 | ~20s |
| 3 | ~40s |

## 12. 已知问题记录

### 12.1 GeneratorExit 异常处理

**问题描述**：
当外部取消 async generator 的迭代（如 SSE 连接断开、调度器取消任务）时，Python 会发送 `GeneratorExit` 异常。这会导致 async generator 立即退出，跳过了 `async with` 上下文管理器的正常清理流程。

**错误信息**：
```
RuntimeError: Attempted to exit cancel scope in a different task than it was entered in
```

**影响场景**：
- SSE 连接断开时
- 调度器取消任务时
- 客户端主动中断任务时

**根本原因**：
`ClaudeCodeClient.run_stream()` 是一个 async generator，当外部迭代被取消时：
1. Python 发送 `GeneratorExit` 异常
2. 生成器立即退出，跳过 `async with` 上下文清理
3. 当 `finally` 块尝试恢复环境变量时，它在一个不同的任务上下文中执行
4. 触发 cancel scope 错误

**解决方案**：
在 `run_stream` 方法中添加对 `GeneratorExit` 异常的专门处理：

```python
except GeneratorExit:
    # 外部取消迭代（如 SSE 连接断开、调度器取消任务）
    # 此时 async with 上下文可能已在其他任务中清理，这里不再重复清理
    logger.warning("[Client] run_stream 被外部取消 (GeneratorExit)")
    # 恢复环境变量
    if old_claudedecode is not None:
        os.environ["CLAUDECODE"] = old_claudedecode
    # 不再 yield，避免再次触发 GeneratorExit 处理
    return
```

**涉及文件**：
- `app/claude_runner/client.py` - `run_stream` 方法
