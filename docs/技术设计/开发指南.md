1. 安装 Python 3.10+
2. 安装 uv 包管理器
3. 配置环境变量

```bash

cp .env.example .env

# 编辑 .env 文件

WORKING_DIR=/path/to/your/project
```

```bash
uv sync
```

```bash
# 方式 1: 直接运行

uv run uvicorn app.main:app --reload
```

## 核心组件

### ClaudeCodeClient (`app/claude_runner/client.py`)

封装 Claude Code，提供两种执行模式：

```python
from app.claude_runner import ClaudeCodeClient

# 创建客户端
client = ClaudeCodeClient(
    working_dir="/path/to/project",
    allowed_tools=["Read", "Write", "Edit", "Bash"],
    permission_mode="acceptEdits",
    continue_conversation=False,
    resume=None
)

# 同步执行
result = await client.run("列出所有 Python 文件")
print(result.message)

# 流式执行
async for msg in client.run_stream("分析项目结构"):
    print(f"[{msg.type.value}] {msg.content}")
```

#### 关键方法

| 方法 | 说明 |
|------|------|
| `run(prompt)` | 同步执行，返回 TaskResult |
| `run_stream(prompt)` | 流式执行，异步迭代器 |

#### 数据结构

```python
# 流式消息
@dataclass
class StreamMessage:
    type: MessageType      # 消息类型
    content: str           # 内容
    timestamp: str         # 时间戳
    tool_name: str         # 工具名称（可选）
    tool_input: dict       # 工具输入（可选）
    metadata: dict         # 元数据

# 任务结果
@dataclass
class TaskResult:
    success: bool
    message: str
    session_id: str
    cost_usd: float
    duration_ms: int
    files_changed: list[str]
    tools_used: list[str]
```

## 任务调度模块 (`app/scheduler/`)

任务调度模块提供任务队列管理和定时任务功能，支持 Cron 表达式。

### 核心组件

#### 数据模型 (`models.py`)

```python
from app.scheduler.models import Task, ScheduledTask, TaskStatus

# 任务状态枚举
class TaskStatus(str, Enum):
    PENDING = "pending"      # 待执行
    RUNNING = "running"      # 运行中
    COMPLETED = "completed"  # 已完成
    FAILED = "failed"        # 失败
    CANCELLED = "cancelled"  # 已取消

# 任务数据结构
@dataclass
class Task:
    id: str                    # 任务唯一标识 (UUID)
    prompt: str                # 任务描述/提示词
    workspace: str = "."       # 工作目录
    timeout: int = 600000      # 超时时间（毫秒）
    auto_approve: bool = False # 是否自动批准工具操作
    allowed_tools: list[str]   # 允许使用的工具列表
    status: TaskStatus         # 任务状态
    result: dict               # 执行结果
    error: str                 # 错误信息
    files_changed: list[str]   # 变更的文件列表
    tools_used: list[str]      # 使用过的工具列表

# 定时任务数据结构
@dataclass
class ScheduledTask:
    id: str                    # 任务唯一标识
    name: str                  # 任务名称
    prompt: str                # 任务描述/提示词
    cron: str                  # Cron 表达式
    enabled: bool = True       # 是否启用
    last_run: str              # 上次执行时间
    next_run: str              # 下次执行时间
    run_count: int = 0         # 已执行次数
```

#### 存储层 (`storage.py`)

提供 JSON 文件存储，支持并发安全：

```python
from app.scheduler.storage import get_storage

# 获取存储实例
storage = get_storage()

# 任务队列操作
storage.queue.add(task)           # 添加任务到队列
storage.queue.get_all()           # 获取所有队列任务
storage.queue.pop()               # 弹出队首任务（FIFO）
storage.queue.remove(task_id)     # 移除指定任务
storage.queue.clear()             # 清空队列

# 定时任务操作
storage.scheduled.save(task)      # 保存或更新定时任务
storage.scheduled.get_all()       # 获取所有定时任务
storage.scheduled.get_enabled()   # 获取已启用的定时任务
storage.scheduled.delete(task_id) # 删除定时任务

# 运行中任务管理
storage.running.add(task)         # 添加运行中任务
storage.running.update(task)      # 更新运行中任务
storage.running.remove(task_id)   # 移除运行中任务

# 历史记录（分页）
storage.history.get_completed(page=1, limit=20)
storage.history.get_failed(page=1, limit=20)
```

#### Cron 解析器 (`cron.py`)

支持 5 位标准格式和 6 位扩展格式：

```python
from app.scheduler.cron import CronParser, is_due, format_next_run

parser = CronParser()

# 解析 Cron 表达式
expression = parser.parse("*/5 * * * *")  # 每 5 分钟

# 验证表达式
is_valid, error = parser.validate("0 9 * * 1-5")
# is_valid: True, error: None

# 计算下次执行时间
next_run = parser.calculate_next_run("0 9 * * 1-5")
# 返回 datetime 对象

# 获取未来 n 个执行时间
next_runs = parser.get_next_runs("0 9 * * 1-5", count=5)

# 判断是否到期
if is_due(scheduled_task.next_run):
    # 执行任务
    pass

# 格式化时间
formatted = format_next_run(next_run)  # "2026-02-26 09:00:00"
```

### Cron 表达式语法

#### 5 位标准格式

```
分 时 日 月 周
*  *  *  *  *
```

| 字段 | 范围 | 说明 |
|------|------|------|
| 分 | 0-59 | 分钟 |
| 时 | 0-23 | 小时 |
| 日 | 1-31 | 日期 |
| 月 | 1-12 | 月份 |
| 周 | 0-6 | 星期（0=周日） |

#### 6 位扩展格式

```
秒 分 时 日 月 周
*  *  *  *  *  *
```

#### 特殊字符

| 字符 | 说明 | 示例 |
|------|------|------|
| `*` | 任意值 | `* * * * *` 每分钟 |
| `/` | 步进 | `*/5 * * * *` 每5分钟 |
| `-` | 范围 | `0 9-17 * * *` 9点到17点 |
| `,` | 列表 | `0 9,12,18 * * *` 9、12、18点 |

#### 预定义别名

| 别名 | 等价于 | 说明 |
|------|--------|------|
| `@yearly` | `0 0 1 1 *` | 每年 |
| `@monthly` | `0 0 1 * *` | 每月 |
| `@weekly` | `0 0 * * 0` | 每周 |
| `@daily` | `0 0 * * *` | 每天 |
| `@hourly` | `0 * * * *` | 每小时 |

#### 常用示例

```bash
# 每 5 分钟执行
*/5 * * * *

# 每天早上 9 点
0 9 * * *

# 工作日早上 9 点
0 9 * * 1-5

# 每小时整点
0 * * * *

# 每周一早上 9 点
0 9 * * 1

# 每月 1 号零点
0 0 1 * *

# 每 30 秒（6位格式）
*/30 * * * * *
```

## 常见问题

### Q: 任务执行时看不到输出？

检查 SSE 连接是否正常，确保使用了 `-N` 参数（curl）或正确处理流式响应。

### Q: 会话 ID 无法恢复？

确认会话 ID 正确，检查 `~/.claude/projects/` 目录下是否存在对应的会话文件。

### Q: 工具被拒绝？

检查 `allowed_tools` 参数，确保包含需要的工具。也可以设置 `permission_mode="bypassPermissions"`。

### 关键设计决策

#### 1. 超时单位使用毫秒

```python
timeout: int = 600000  # 10 分钟，单位毫秒
```

**原因**：与 JavaScript 前端保持一致，避免单位转换错误。

#### 2. TaskStatus 继承 str 和 Enum

```python
class TaskStatus(str, Enum):
    PENDING = "pending"
```

**原因**：便于 JSON 序列化，`status.value` 直接返回字符串。

#### 3. 时间字段使用 ISO 格式字符串

```python
created_at: str = field(default_factory=lambda: datetime.now().isoformat())
```

**原因**：datetime 对象无法直接 JSON 序列化，ISO 字符串通用性强。

#### 4. 文件锁使用原子文件创建

```python
fd = os.open(lock_path, os.O_CREAT | os.O_EXCL | os.O_WRONLY)
```

**原因**：Unix 文件锁在 Windows 上不可用，原子创建跨平台兼容。

### 遇到的问题与解决方案

#### 问题 1：状态枚举序列化

- **问题**：枚举直接 JSON 序列化失败
- **解决**：继承 `str` 和 `Enum`，使用 `.value` 序列化

#### 问题 2：Windows 文件锁兼容性

- **问题**：`fcntl.flock()` 仅限 Unix 系统
- **解决**：使用 `os.O_CREAT | os.O_EXCL` 原子创建锁文件

#### 问题 3：历史记录无限增长

- **问题**：已完成/失败任务会持续累积
- **解决**：设置 `MAX_HISTORY = 1000`，自动截断旧记录

### 存储类结构

```
BaseStorage              # 基类：读写、加锁
    ├── QueueStorage     # 任务队列：FIFO 操作
    ├── ScheduledStorage # 定时任务：CRUD + 启用过滤
    ├── RunningStorage   # 运行中：状态跟踪
    └── HistoryStorage   # 历史记录：分页 + 限制数量
```

## 调度器配置说明

> 详细的调度器配置参数和使用指南。

### 配置文件位置

调度器配置定义在 `app/scheduler/config.py` 中：

```python
from app.scheduler.config import (
    # 数据目录
    DATA_DIR,           # Path("data")

    # 数据文件
    QUEUE_FILE,         # data/queue.json
    SCHEDULED_FILE,     # data/scheduled.json
    RUNNING_FILE,       # data/running.json
    COMPLETED_FILE,     # data/completed.json
    FAILED_FILE,        # data/failed.json

    # 超时配置
    DEFAULT_TIMEOUT,    # 600000 (10分钟)
    MIN_TIMEOUT,        # 1000 (1秒)
    MAX_TIMEOUT,        # 3600000 (1小时)

    # 调度配置
    POLL_INTERVAL,      # 10 (秒)
    MAX_RETRIES,        # 2
    MAX_HISTORY,        # 1000

    # 分页配置
    DEFAULT_PAGE,       # 1
    DEFAULT_LIMIT,      # 20
    MAX_LIMIT,          # 100

    # 锁配置
    LOCK_TIMEOUT,       # 5 (秒)
    LOCK_RETRY_INTERVAL,# 0.1 (秒)
    MAX_LOCK_RETRIES,   # 50

    # 状态枚举
    SchedulerStatus,
)
```

### 核心配置详解

#### 超时配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `DEFAULT_TIMEOUT` | 600000 (10分钟) | 新任务的默认超时时间（毫秒） |
| `MIN_TIMEOUT` | 1000 (1秒) | 允许的最小超时时间 |
| `MAX_TIMEOUT` | 3600000 (1小时) | 允许的最大超时时间 |

**修改方式**：直接修改 `config.py` 或在创建任务时指定 `timeout` 参数。

```python
# 创建任务时指定超时（单位：毫秒）
task = Task(
    id=str(uuid.uuid4()),
    prompt="执行任务",
    timeout=300000,  # 5 分钟
)
```

#### 调度配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `POLL_INTERVAL` | 10 | 调度器轮询间隔（秒） |
| `MAX_RETRIES` | 2 | 任务失败后最大重试次数 |
| `MAX_HISTORY` | 1000 | 历史记录最大保留数量 |

**调整轮询间隔**：

```python
from app.scheduler import Scheduler

# 创建调度器时指定轮询间隔
```

### 执行器配置

执行器（`TaskExecutor`）负责实际执行任务，配置参数包括：

#### 重试延迟配置

在 `app/scheduler/executor.py` 中定义：

```python
BASE_DELAY = 5.0       # 基础延迟（秒）
MAX_DELAY = 60.0       # 最大延迟（秒）
JITTER = 0.1           # 抖动因子（±10%）
```

**指数退避算法**：

```
delay = base_delay × 2^retry_count + jitter
delay = min(delay, max_delay)
```

| 重试次数 | 基础延迟 | 实际延迟范围 |
|---------|---------|-------------|
| 1 | 5 秒 | 4.5 ~ 5.5 秒 |
| 2 | 10 秒 | 9 ~ 11 秒 |
| 3+ | 20 秒（限制） | 18 ~ 22 秒 |

#### 错误分类

执行器将错误分为以下类型：

| 类型 | 可重试 | 说明 |
|------|--------|------|
| `TRANSIENT` | ✅ | 临时性错误 |
| `TIMEOUT` | ✅ | 超时错误 |
| `RESOURCE` | ✅ | 资源错误（网络、限流） |
| `PERMANENT` | ❌ | 永久性错误 |
| `USER_CANCEL` | ❌ | 用户取消 |
| `VALIDATION` | ❌ | 验证错误 |

### 已知问题和限制

> 记录任务调度系统的已知问题、功能限制和未来改进方向。
> **最后更新**: 2026-02-27

#### 功能限制

| 限制项 | 当前值 | 说明 |
|--------|--------|------|
| 单机部署 | - | 当前 JSON 文件存储方案不支持分布式部署，无法多实例共享数据 |
| 无内存缓存 | - | 每次操作都直接读写文件，频繁操作可能影响性能 |
| 无任务优先级 | - | 任务按 FIFO（先进先出）顺序执行，不支持优先级调度 |
| 无任务取消 | - | 后端 API 不支持取消正在运行的任务 |
| 历史记录上限 | 1000 条 | 已完成和失败任务各最多保留 1000 条，超出自动截断 |
| 超时时间范围 | 1秒 - 1小时 | 任务超时时间必须在 `MIN_TIMEOUT` 和 `MAX_TIMEOUT` 之间 |
| 最大重试次数 | 2 次 | 任务失败后最多重试 2 次，之后标记为失败 |
| 无持久化重试状态 | - | 重试延迟期间服务重启会丢失重试状态 |

#### API 限制

| 参数 | 限制 | 说明 |
|------|------|------|
| prompt 长度 | ≤ 10000 字符 | 任务描述超过限制会返回 400 错误 |
| name 长度 | ≤ 100 字符 | 定时任务名称超过限制会返回 400 错误 |
| 分页 limit | ≤ 100 | 单次查询最多返回 100 条记录 |
| 工作目录 | 必须在允许范围内 | 路径遍历攻击防护，工作目录必须在 `WORKING_DIR` 配置的目录内 |

#### Cron 表达式限制

| 限制项 | 说明 |
|--------|------|
| 不支持预定义别名 | `@yearly`、`@monthly` 等别名暂不支持 |
| 不支持 `L`（最后一天） | 月末日期（如 `L` 表示月末）暂不支持 |
| 不支持 `W`（工作日） | 最近工作日计算暂不支持 |
| 不支持 `#`（第N个周几） | 如 `0 9 * * 1#3`（第三个周一）暂不支持 |

#### 前端限制

| 限制项 | 说明 |
|--------|------|
| 无任务取消按钮 | 由于后端 API 不支持，运行中任务列表没有取消按钮 |
| 轮询更新 | 使用 5 秒轮询刷新数据，非实时 WebSocket 推送 |
| 无离线支持 | 网络断开后无法查看缓存数据 |
| 响应式设计 | < 768px 小屏幕下部分表格列可能被隐藏 |

#### 已知问题

1. **Windows 文件锁性能** - Windows 平台使用原子文件创建实现锁机制，高并发场景下可能出现锁等待超时
2. **大量历史数据分页** - 历史记录超过 1000 条时，分页查询可能变慢
3. **高频轮询开销** - 默认 10 秒轮询间隔，大量定时任务检查可能有性能开销
4. **跨平台路径处理** - Windows 和 Unix 路径分隔符差异需注意
5. **时区处理** - 所有时间使用本地时区，不支持指定时区
6. **重试延迟不持久化** - 服务重启后重试延迟状态丢失

#### 安全措施

| 措施 | 说明 |
|------|------|
| 路径遍历防护 | `validate_workspace()` 防止访问允许目录外的路径 |
| 输入验证 | prompt、name、timeout、tools 等字段严格验证 |
| XSS 防护 | 前端使用 `textContent` 转义用户输入 |
| CSRF 防护 | FastAPI 内置 CSRF 防护 |
| 可选认证 | 支持 OAuth2 可选认证 |

```python
class SchedulerCache:
    """调度器内存缓存"""
    def __init__(self, ttl_seconds: int = 30):
        self._cache: dict[str, CacheEntry] = {}
        self._ttl = ttl_seconds

    def get(self, key: str) -> Optional[Any]:
        entry = self._cache.get(key)
        if entry and datetime.now() < entry.expires_at:
            return entry.data
        return None
```

```python
def validate_workspace_enhanced(workspace: str) -> str:
    workspace = validate_workspace(workspace)
    if ALLOWED_WORKSPACES:
        workspace_path = Path(workspace).resolve()
        for allowed_dir in ALLOWED_WORKSPACES:
```
