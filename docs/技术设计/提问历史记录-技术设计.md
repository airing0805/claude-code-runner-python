## 1. 系统架构设计

### 1.2 技术选型

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 后端框架 | FastAPI | 现有框架，异步支持 |

## 2. 模块职责划分

### 2.1 后端模块

#### 2.1.1 session.py (Router)

**职责**:
- 提供 REST API 端点
- 处理请求参数验证
- 协调数据提取和缓存
- 格式化响应数据

**API 端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/projects` | 获取项目列表 (复用) |
| GET | `/api/projects/{project_name}/questions` | 获取项目提问列表 |
| GET | `/api/projects/{project_name}/sessions` | 获取会话列表 (复用) |
| GET | `/api/sessions/{session_id}/messages` | 获取会话消息 (复用) |

#### 2.1.2 questions.py (Service)

**职责**:
- 从会话文件中提取首次用户提问
- 过滤 IDE 相关标签
- 脱敏敏感信息

**核心函数**:

```python
# 提取提问
def extract_question(filepath: Path) -> dict[str, Any] | None

# 过滤 IDE 标签
def _filter_ide_tags(text: str) -> str

# 敏感信息脱敏
def mask_sensitive_info(text: str) -> str

# 数据模型
@dataclass
class QuestionItem:
    id: str
    project_name: str
    question_text: str
    session_id: str
    timestamp: str
```

#### 2.1.3 MetadataCache (缓存)

**职责**:
- 缓存会话元数据
- 基于文件 mtime 自动失效

**扩展字段**:

```python
{
    "session_uuid.jsonl": {
        "mtime": 1708123456.789,
        "title": "会话标题",
        "message_count": 10,
        "tools": ["Read", "Write"],
        "has_question": true,           # 新增: 是否存在有效提问
        "question_timestamp": "..."      # 新增: 提问时间戳
    }
}
```

### 2.2 前端模块

#### 2.2.1 questions.js

**职责**:
- 管理提问历史记录的展示
- 处理用户交互 (点击、复制、展开/收起)
- 协调虚拟滚动
- 与 History 模块交互继续会话

**核心功能**:

| 功能 | 说明 |
|------|------|
| 项目列表加载 | 分页加载项目列表 |
| 提问列表加载 | 分页加载项目提问列表 |
| 虚拟滚动 | 优化长列表性能 |
| 文本截断 | 超过 200 字符自动截断 |
| 展开/收起 | 点击按钮查看完整内容 |
| 复制功能 | 一键复制提问到剪贴板 |
| 继续会话 | 点击提问进入对应会话 |

#### 2.2.2 navigation.js

**职责**:
- 视图切换管理
- 添加提问历史记录视图 (Views.QUESTIONS)

#### 2.2.3 History 模块复用

继续会话功能复用现有 History 模块的 `continueSession` 方法。

## 5. 缓存机制设计

### 5.1 缓存架构

采用文件级缓存，复用现有的 MetadataCache 类，扩展支持提问缓存字段。

### 5.2 缓存数据结构

```json
// .metadata.json
{
    "session_uuid.jsonl": {
        "mtime": 1708123456.789,
        "title": "会话标题",
        "message_count": 10,
        "tools": ["Read", "Write"],
        "has_question": true,
        "question_timestamp": "2026-02-18T09:45:09.554Z",
        "question_text": "完整提问文本"  // 可选，仅在需要时存储
    }
}
```

### 5.3 缓存策略

| 策略 | 配置 | 说明 |
|------|------|------|
| 缓存位置 | `.metadata.json` | 项目目录下 |
| 失效条件 | mtime 变化 | 文件修改时缓存失效 |
| 最大条目 | 无限制 | 按项目隔离存储 |
| 缓存标志 | has_question | 是否存在有效提问 |
| 时间戳 | question_timestamp | 提问时间用于排序 |

### 5.4 缓存流程

```
请求 /api/projects/{project_name}/questions
        │
        ▼
┌───────────────────┐
│ 检查项目目录是否存在 │
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│ 创建 MetadataCache │
└─────────┬─────────┘
          │
          ▼
遍历所有 .jsonl 文件
          │
          ▼
┌───────────────────┐
│ 检查缓存是否有效   │
│ (mtime 比较)      │
└─────────┬─────────┘
          │
    ┌─────┴─────┐
    │ 缓存命中   │ 缓存失效
    ▼           ▼
┌────────┐  ┌────────────────┐
│ 返回缓存 │  │ 读取文件提取提问 │
└────────┘  │ 更新缓存       │
            └────────┬───────┘
                     ▼
            ┌────────────────┐
            │ 按时间戳排序    │
            └────────┬───────┘
                     ▼
            ┌────────────────┐
            │ 分页返回结果   │
            └────────────────┘
```

### 5.5 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 缓存命中率 | > 95% | 二次访问时优先命中缓存 |
| 缓存读取时间 | < 10ms | 纯内存操作 |
| 首次提取时间 | < 500ms/文件 | 文件 IO + JSON 解析 |

## 7. 数据提取规则

### 7.1 提取流程

```
读取 .jsonl 文件
    │
    ▼
遍历每行 JSON
    │
    ▼
检查 type == "user"
    │
    ├─ 是: 提取内容
    │
    └─ 否: 跳过
    │
    ▼
从 message.content 数组中提取 text 类型
    │
    ▼
过滤 IDE 标签 (ide_selection, ide_opened_file)
    │
    ▼
脱敏敏感信息 (API keys, tokens, passwords)
    │
    ▼
返回 QuestionItem
```

### 7.2 过滤规则

**需要过滤的标签**:

| 标签 | 说明 |
|------|------|
| `<ide_selection>...</ide_selection>` | IDE 选中的代码 |
| `<ide_opened_file>...</ide_opened_file>` | IDE 打开的文件 |

**敏感信息模式**:

| 模式类型 | 正则示例 | 替换为 |
|----------|----------|--------|
| API Keys | `sk-[a-zA-Z0-9]{20,}` | `[API_KEY]` |
| GitHub Tokens | `ghp_[a-zA-Z0-9]{36}` | `[GITHUB_TOKEN]` |
| Passwords | `password['"]?\s*[:=]\s*['"]...` | `[PASSWORD]` |
| AWS Keys | `AKIA[0-9A-Z]{16}` | `[AWS_ACCESS_KEY]` |

| 会话不存在 | 显示错误信息并返回列表 |
