# 任务调度 - 测试用例文档

## 1. 测试概述

### 1.1 测试目标
验证任务调度模块的功能正确性、性能表现、安全性和用户体验，确保系统稳定可靠。

### 1.2 测试范围
- 任务队列管理功能
- 定时任务管理功能  
- 任务状态跟踪功能
- 调度器控制功能
- 任务详情与日志功能
- 前端用户界面交互
- API接口正确性
- 系统性能和安全性

### 1.3 测试策略
采用分层测试策略：
- **单元测试**: 验证核心逻辑和数据模型
- **集成测试**: 验证组件间交互和API功能
- **E2E测试**: 验证完整用户交互流程
- **性能测试**: 验证系统在负载下的表现
- **安全测试**: 验证安全防护措施有效性

## 2. 功能测试用例

### 2.1 任务队列管理

#### TC-001: 添加任务到队列
- **前置条件**: 调度器已启动
- **测试步骤**:
  1. 打开任务调度页面
  2. 点击"添加任务"按钮
  3. 输入任务描述"测试任务"
  4. 选择工作目录"/test"
  5. 设置超时时间300秒
  6. 选择工具["Read", "Write"]
  7. 启用自动批准
  8. 点击"确认添加"
- **预期结果**:
  - 任务成功添加到队列
  - 显示成功通知"任务已添加到队列"
  - 队列列表显示新任务
  - 任务ID、描述、配置参数正确

#### TC-002: 队列任务验证
- **前置条件**: 队列中有任务
- **测试步骤**:
  1. 查看队列列表
  2. 验证任务显示字段完整性
  3. 验证任务状态为"待执行"
  4. 验证任务来源为"手动"
- **预期结果**:
  - 显示任务ID、描述、工作目录、超时时间、工具列表、自动批准状态
  - 状态显示为"待执行"
  - 来源显示为"手动"

#### TC-003: 删除队列任务
- **前置条件**: 队列中有任务
- **测试步骤**:
  1. 在队列列表中找到目标任务
  2. 点击"删除"按钮
  3. 在确认对话框中点击"确定"
- **预期结果**:
  - 显示确认对话框
  - 任务从队列中移除
  - 显示成功通知"任务已从队列中删除"

#### TC-004: 清空任务队列
- **前置条件**: 队列中有多个任务
- **测试步骤**:
  1. 点击"清空队列"按钮
  2. 在确认对话框中点击"确定"
- **预期结果**:
  - 显示确认对话框
  - 所有任务从队列中移除
  - 队列统计数量更新为0
  - 显示成功通知"队列已清空"

### 2.2 定时任务管理

#### TC-005: 创建定时任务
- **前置条件**: 调度器已启动
- **测试步骤**:
  1. 切换到"定时任务"标签页
  2. 点击"添加定时任务"按钮
  3. 输入任务名称"每日备份"
  4. 输入任务描述"执行每日数据备份"
  5. 输入Cron表达式"0 2 * * *"
  6. 选择工作目录"/backup"
  7. 设置超时时间600秒
  8. 选择工具["Bash", "Glob"]
  9. 启用自动批准
  10. 保持启用状态
  11. 点击"保存"
- **预期结果**:
  - Cron表达式验证通过
  - 显示下次执行时间预览
  - 定时任务成功创建
  - 显示成功通知"定时任务已创建"
  - 定时任务列表显示新任务

#### TC-006: Cron表达式验证
- **前置条件**: 打开添加定时任务对话框
- **测试步骤**:
  1. 输入有效Cron表达式"*/5 * * * *"
  2. 观察验证结果
  3. 输入无效Cron表达式"invalid"
  4. 观察验证结果
- **预期结果**:
  - 有效表达式显示绿色验证通过，显示下次执行时间
  - 无效表达式显示红色错误提示"无效的Cron表达式"

#### TC-007: 编辑定时任务
- **前置条件**: 存在定时任务
- **测试步骤**:
  1. 在定时任务列表中找到目标任务
  2. 点击"编辑"按钮
  3. 修改任务名称为"每周备份"
  4. 修改Cron表达式为"0 2 * * 0"
  5. 点击"保存"
- **预期结果**:
  - 编辑对话框显示当前任务配置
  - 任务信息成功更新
  - 显示成功通知"定时任务已更新"
  - 列表显示更新后的任务信息

#### TC-008: 启用/禁用定时任务
- **前置条件**: 存在启用的定时任务
- **测试步骤**:
  1. 在定时任务列表中找到目标任务
  2. 点击"禁用"按钮
  3. 观察任务状态变化
  4. 再次点击"启用"按钮
- **预期结果**:
  - 禁用后任务状态变为"已禁用"，下次执行时间清空
  - 启用后任务状态变为"已启用"，重新计算下次执行时间
  - 显示相应状态通知

#### TC-009: 立即执行定时任务
- **前置条件**: 存在启用的定时任务
- **测试步骤**:
  1. 在定时任务列表中找到目标任务
  2. 点击"立即执行"按钮
- **预期结果**:
  - 任务被添加到执行队列
  - 显示成功通知"任务已添加到队列"
  - 任务来源标记为"立即执行"

### 2.3 任务状态跟踪

#### TC-010: 任务状态流转
- **前置条件**: 调度器正在运行
- **测试步骤**:
  1. 添加一个简单任务到队列
  2. 观察任务状态变化
  3. 等待任务执行完成
- **预期结果**:
  - 任务状态从"待执行" → "运行中" → "已完成"
  - 运行中状态显示执行时长实时更新
  - 完成后显示执行结果、文件变更、工具使用记录

#### TC-011: 失败任务处理
- **前置条件**: 调度器正在运行
- **测试步骤**:
  1. 添加一个会导致失败的任务（如不存在的文件路径）
  2. 观察任务执行过程
- **预期结果**:
  - 任务状态从"待执行" → "运行中" → "失败"
  - 失败任务移动到失败列表
  - 显示错误信息

#### TC-012: 分页查询
- **前置条件**: 已完成任务超过20条
- **测试步骤**:
  1. 切换到"已完成"标签页
  2. 验证默认显示第一页（20条）
  3. 点击下一页
  4. 验证显示第二页内容
  5. 跳转到最后一页
- **预期结果**:
  - 正确分页显示任务列表
  - 页码导航正常工作
  - 总数量和页数计算正确

## 3. API测试用例

### 3.1 任务队列API

#### TC-API-001: POST /api/scheduler/tasks
- **请求**:
  ```json
  {
    "prompt": "测试任务",
    "workspace": "/test",
    "timeout": 300000,
    "allowed_tools": ["Read", "Write"],
    "auto_approve": true
  }
  ```
- **预期响应**:
  ```json
  {
    "success": true,
    "data": {
      "id": "uuid",
      "prompt": "测试任务",
      "workspace": "/test",
      "timeout": 300000,
      "allowed_tools": ["Read", "Write"],
      "auto_approve": true,
      "status": "pending",
      "source": "manual"
    },
    "message": "任务已添加到队列"
  }
  ```

#### TC-API-002: GET /api/scheduler/tasks
- **预期响应**:

- **测试条件**: 队列中有1000个任务
- **测试步骤**: 查询队列列表
- **预期结果**: 响应时间 ≤ 500ms

### 4.2 并发测试

#### TC-PERF-003: 并发任务执行
- **测试条件**: 调度器运行中
- **测试步骤**: 同时添加50个任务到队列
- **预期结果**: 
  - 所有任务正确排队
  - 无数据丢失或重复
  - 系统资源使用合理

### 4.3 负载测试

#### TC-PERF-004: 长时间运行稳定性
- **测试条件**: 调度器持续运行24小时
- **测试步骤**: 每小时添加10个任务
- **预期结果**:
  - 系统无内存泄漏

  - 响应时间保持稳定
